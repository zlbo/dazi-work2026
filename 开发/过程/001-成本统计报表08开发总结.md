## 1. 业务目标

将管理端上传的 Excel 设计稿「**成本统计报表08**」落地为 DRAP **中国式复杂报表**，在子应用中通过 `ComplexReport` 渲染：

- 多级表头（基本信息 / 成本项 / 直接成本子组 / 成本比较）
- 左侧维度列（组织、年度、月度、项目）横向冻结
- 组织、年度纵向合并（`rowMerge`）
- 项目/月度/组织「总计」行强调
- 开发阶段使用静态样例数据；上线可切换 SQL / ontology

## 5. 关键实现

### 5.1 三级表头 `headerCells`

平台 RLIR 中 `headerCells` 为空，且 `columnGroups` 为**三层嵌套**（成本项 → 直接成本 → 叶子列）。仅用 `columnGroups` 回退渲染无法正确表达 Excel 第 2–4 行表头。

**处理**：在同步到 `reportLayout.json` 时手工补全 `headerCells`（20 个锚点单元格）+ `headerRowCount: 3`，使 `ComplexReport` 走矩阵表头路径（与 Excel 合并格一致）。

### 5.2 `normalizeReportLayout()`

`src/fixtures/reportLayout.ts` 中：

- 过滤 `leaf: false` 父列（本例 16 列均为叶子，但保留规范）
- **组织、年度** 开启 `rowMerge`
- 维度列左对齐；数值列 `format: "number"`、右对齐
- 总计行：`rowEmphasis` 匹配 project/organization/month 含「总计」

### 5.3 左侧四列冻结

`src/styles/template.css` 按列 id 设置 `position: sticky`：

- `col_organization` → `col_year` → `col_month` → `col_project` 逐级 `left` 偏移

### 5.4 静态样例数据

从 `原生解析.md` 抽取 **15 行**（公司1/公司2 各月项目明细 + 各级总计）写入：

- `src/fixtures/generated/reportRows.json`
- `manifest.data_sources[].data.rows`

### 5.5 SDK 升级

从 `app_应用0519` 同步增强版 `ComplexReport.tsx` 至 `app_项目应用0520/sdk/app-sdk-ui`：

- 支持 `rowMerge` 纵向合并
- 列 CSS 类名（`drap-complex-report__cell--col_*`）供冻结列样式使用

## 6. 问题与修复

### 6.1 表头正确但表体无数据

**现象**：页面布局正常，数据区为空。

**原因**：

1. `manifest.data_sources[].data` 使用了字段名 `data`，而 SDK `resolveStaticDatasetFromManifest` 只读取 `rows`
2. 宿主未注入 datasets 时，`useReportDataset` 未回退到 manifest 内联数据

**修复**：

| 文件                                    | 改动                                |
| --------------------------------------- | ----------------------------------- |
| `manifest.json`                         | `data.data` → `data.rows`           |
| `resolveStaticDataset.ts`               | 同时兼容 `rows` 与 `data`           |
| `useDataset.ts` / `useReportDataset.ts` | 无宿主注入时从 manifest 解析 static |

## 11. 后续同类报表建议

1. **先看 RLIR**：若 `columnGroups` 三层嵌套且 `headerCells` 为空，优先补 `headerCells`，不要仅依赖 columnGroups 回退。
2. **static 数据**：manifest 内联务必用 `data.rows`，与 SDK 一致。
3. **样例数据**：优先从 `原生解析.md` 或 Excel 第 5 行起抽取，保证总计行结构完整。
4. **复用实践文档**：通用流程仍以 [`主要财务指标复杂报表开发实践.md`](../../资源/docs/app/主要财务指标复杂报表开发实践.md) 为准；本文记录本报表特有差异。

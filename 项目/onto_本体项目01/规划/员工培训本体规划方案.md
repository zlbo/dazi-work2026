## 二、需求分析

### 2.1 业务背景

围绕**员工培训**域构建本体，将员工信息、培训课程、培训记录、培训评估等数据统一为可查询、可分析、可对话的业务对象，支撑：

- 员工培训档案管理与查询
- 培训课程体系管理与分析
- 培训完成率、合格率等核心指标汇总
- 员工能力画像与培训需求分析
- 培训效果评估与改进建议

### 2.2 业务域与边界

| 范围内                         | 范围外（本期不做）     |
| ------------------------------ | ---------------------- |
| 员工主数据（基本信息/岗位）    | 员工薪资与绩效明细     |
| 培训课程体系（课程/类别/讲师） | 招聘与入职流程         |
| 培训记录（报名/出勤/成绩）    | 员工晋升与轮岗         |
| 培训评估（满意度/能力提升）   | 员工离职与考勤         |
| 培训预算与资源分配           | 财务报销与审批流       |

### 2.3 核心业务问题

| 问题               | 说明                           |
| ------------------ | ------------------------------ |
| 培训体系有哪些课程 | 课程分类、层级、覆盖范围       |
| 谁参加了培训       | 员工培训记录、完成情况         |
| 培训效果如何       | 合格率、满意度、能力提升       |
| 培训资源如何分配   | 讲师负荷、场地利用率、预算使用 |
| 谁需要培训         | 培训需求分析、能力缺口识别     |
| 如何优化培训方案   | 高价值课程、薄弱环节识别       |

### 3.2 表结构

#### 3.2.1 维度表：employee_master（员工表）

**用途**：存储员工基本信息，支撑培训对象分析。

| 字段名           | 类型     | 说明           | 主键/索引 |
| ---------------- | -------- | -------------- | --------- |
| employee_id      | String   | 员工 ID        | PK        |
| employee_code    | String   | 员工编号       |           |
| employee_name    | String   | 员工姓名       |           |
| department       | String   | 部门           |           |
| position         | String   | 岗位           |           |
| level            | String   | 职级           |           |
| hire_date        | Date     | 入职日期       |           |
| status           | String   | 在职状态       |           |
| created_at       | DateTime | 创建时间       |           |
| updated_at       | DateTime | 更新时间       |           |

**排序键**：`ORDER BY (employee_id)`  
**引擎**：`MergeTree()`

#### 3.2.2 维度表：course_master（课程表）

**用途**：存储培训课程信息，支撑课程体系管理。

| 字段名           | 类型     | 说明               | 主键/索引 |
| ---------------- | -------- | ------------------ | --------- |
| course_id        | String   | 课程 ID           | PK        |
| course_code      | String   | 课程编码           |           |
| course_name      | String   | 课程名称           |           |
| course_category  | String   | 课程类别（技术/管理/通用） |      |
| course_level     | String   | 课程级别（初级/中级/高级） |    |
| duration_hours   | Float64  | 课程时长（小时）   |           |
| required         | Boolean  | 是否必修课         |           |
| trainer_id       | String   | 主讲讲师 ID       |           |
| capacity         | Int32    | 容纳人数         |           |
| status           | String   | 课程状态（启用/停用） |      |
| created_at       | DateTime | 创建时间           |           |

**排序键**：`ORDER BY (course_id)`

#### 3.2.3 维度表：trainer_master（讲师表）

**用途**：存储讲师信息，支撑师资管理与负荷分析。

| 字段名           | 类型     | 说明           | 主键/索引 |
| ---------------- | -------- | -------------- | --------- |
| trainer_id       | String   | 讲师 ID        | PK        |
| trainer_name     | String   | 讲师姓名       |           |
| department       | String   | 所属部门       |           |
| expertise        | String   | 专业领域       |           |
| qualification    | String   | 资质等级       |           |
| status           | String   | 状态（在职/兼职/离职） |    |

**排序键**：`ORDER BY (trainer_id)`

#### 3.2.4 事实表：training_record（培训记录表）

**用途**：存储员工参加培训的记录，一行对应「某员工 × 某课程」的培训记录。

| 字段名           | 类型     | 说明           | 主键/索引 |
| ---------------- | -------- | -------------- | --------- |
| record_id        | String   | 记录 ID        | PK        |
| employee_id      | String   | 员工 ID        |           |
| course_id        | String   | 课程 ID        |           |
| training_date    | Date     | 培训日期       | 分区/排序 |
| attendance       | String   | 出勤状态（出勤/缺勤/迟到） |      |
| completion       | String   | 完成状态（完成/未完成） |    |
| score            | Float64  | 考核成绩（0-100） |        |
| certificate_no   | String   | 证书编号（如有） |        |
| created_at       | DateTime | 创建时间       |           |

**排序键**：`ORDER BY (training_date, employee_id, course_id)`  
**引擎**：`MergeTree()`  
**分区建议**：`PARTITION BY toYYYYMM(training_date)`（按月分区）

**业务规则（物理层约束建议）**：
- `score` 取值范围：0-100
- `completion = '完成'` 时，`attendance = '出勤'`

#### 3.2.5 事实表：training_evaluation（培训评估表）

**用途**：存储员工对培训的评估反馈，用于课程质量监控。

| 字段名           | 类型     | 说明           | 主键/索引 |
| ---------------- | -------- | -------------- | --------- |
| evaluation_id    | String   | 评估 ID        | PK        |
| record_id        | String   | 关联培训记录 ID |           |
| employee_id      | String   | 员工 ID        |           |
| course_id        | String   | 课程 ID        |           |
| trainer_score    | Float64  | 讲师评分（1-5） |        |
| content_score    | Float64  | 内容评分（1-5） |        |
| environment_score| Float64  | 环境评分（1-5） |        |

| suggestion       | String   | 改进建议       |           |

**排序键**：`ORDER BY (record_id)`

## 四、Cube 层设计

### 4.1 TrainingCube（培训分析主 Cube）

**目的**：全局培训分析，支持多维度切片。

**数据源**：`training_record`

**维度**：

| 维度名           | 列名             | 类型   | 说明     |
| ---------------- | ---------------- | ------ | -------- |
| employee_id      | employee_id      | string | 员工 ID  |
| department       | department       | string | 部门     |
| position         | position         | string | 岗位     |
| course_id        | course_id        | string | 课程 ID  |
| course_category  | course_category  | string | 课程类别 |
| training_date    | training_date    | date   | 培训日期 |

**度量**：

| 度量名       | 列名          | 聚合 | 说明         |
| ------------ | ------------- | ---- | ---------- |
| total_records | record_id     | count | 培训记录数 |
| attendance_rate | -          | -    | 出勤率（派生） |
| completion_rate | -          | -    | 完成率（派生） |
| avg_score    | score         | avg  | 平均成绩   |
| qualified_count | -          | -    | 合格人数（派生） |

**派生度量**：

| 度量名            | 表达式                                              | 说明       |
| ----------------- | --------------------------------------------------- | ---------- |
| attendance_rate   | sum(attendance='出勤') / total_records * 100        | 出勤率(%)  |
| completion_rate   | sum(completion='完成') / total_records * 100        | 完成率(%)  |
| qualified_count   | sum(score >= 60)                                    | 合格人数   |
| qualified_rate    | qualified_count / total_records * 100               | 合格率(%)  |

### 4.2 EmployeeTrainingCube（员工培训 Cube）

**目的**：按员工维度聚合培训记录。

**维度**：`employee_id`, `department`, `position`, `level`  
**度量**：`total_records`, `completion_rate`, `avg_score`, `qualified_rate`

### 4.3 CourseTrainingCube（课程培训 Cube）

**目的**：按课程维度聚合培训效果。

**维度**：`course_id`, `course_category`, `course_level`  
**度量**：`total_records`, `completion_rate`, `avg_score`, `qualified_rate`

### 4.4 TrainerPerformanceCube（讲师绩效 Cube）

**目的**：讲师授课效果评估。

**数据源**：`training_evaluation` 关联 `trainer_master`

**维度**：`trainer_id`, `department`, `expertise`  
**度量**：`trainer_score`, `content_score`, `overall_score`（平均值）

### 4.5 TimeTrainingCube（时间维度 Cube）

**目的**：培训趋势分析。

**维度**：

| 维度名       | 列名           | 类型   | 说明 |
| ---------- | ---------- | ------ | ---- |
| training_date | training_date | date   | 日期 |
| year_month | training_date | string | 年月 |
| quarter    | training_date | string | 季度 |
| year       | training_date | date   | 年份 |

**度量**：`total_records`, `completion_rate`, `avg_score`, `qualified_count`

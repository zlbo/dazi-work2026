## 二、需求分析

### 2.1 业务背景

围绕**产品销售**域构建本体，将产品主数据与销售事实数据统一为可查询、可分析、可对话的业务对象，支撑：

- 按产品、品类、客户、区域、渠道、时间等多维度销售分析
- 销售额、销量、客单价、订单数等核心指标汇总与排行
- 同比、环比趋势与异常识别
- 产品与客户的销售贡献归因

### 2.2 业务域与边界

| 范围内                         | 范围外（本期不做）     |
| ------------------------------ | ---------------------- |
| 产品主数据（SKU/品类/定价）    | 库存与仓储明细         |
| 销售订单/出库事实              | 采购、生产排程         |
| 客户主数据与区域               | CRM 跟进、合同审批流   |
| 销售渠道（线上/线下/经销等）   | 财务总账、应收核销     |
| 销售指标聚合与排行、趋势分析   | 利润/成本精细核算      |

### 2.3 核心业务问题

| 问题               | 说明                           |
| ------------------ | ------------------------------ |
| 卖了多少           | 销售额、销量、订单数总览       |
| 什么产品卖得好     | 产品/品类销售排行与结构       |
| 谁在买             | 客户贡献、区域分布、客户分层   |
| 通过什么渠道卖     | 渠道销售占比与效率             |
| 销售趋势如何       | 日/月/季同比、环比             |
| 如何优化销售策略   | 高潜产品、重点客户、薄弱区域   |

## 三、物理层设计

### 3.1 表清单

| 表名                 | 类型   | 说明               |
| -------------------- | ------ | ------------------ |
| `product_master`     | 维度表 | 产品主数据         |
| `customer_dimension` | 维度表 | 客户主数据         |
| `channel_dimension`  | 维度表 | 销售渠道主数据     |
| `sales_order_fact`   | 事实表 | 销售订单行事实     |

> 命名与字段可与数据空间侧栏 **数据资源 → 数据空间0519** 中实际建表对齐；下表为推荐初始结构。

### 3.2 表结构

#### 3.2.1 维度表：product_master（产品表）

**用途**：存储产品 SKU 及品类、定价等主数据。

| 字段名           | 类型     | 说明           | 主键/索引 |
| ---------------- | -------- | -------------- | --------- |
| product_id       | String   | 产品 ID        | PK        |
| product_code     | String   | 产品编码       |           |
| product_name     | String   | 产品名称       |           |
| product_category | String   | 产品大类       |           |
| product_subcategory | String | 产品子类     |           |
| brand            | String   | 品牌           |           |
| unit             | String   | 计量单位       |           |
| list_price       | Float64  | 挂牌价         |           |
| cost_price       | Float64  | 成本价（可选） |           |
| status           | String   | 状态（在售/停售） |        |
| created_at       | DateTime | 创建时间       |           |
| updated_at       | DateTime | 更新时间       |           |

**排序键**：`ORDER BY (product_id)`  
**引擎**：`MergeTree()`

#### 3.2.2 维度表：customer_dimension（客户表）

**用途**：存储客户主数据，支撑区域与客户类型分析。

| 字段名          | 类型     | 说明                     |
| --------------- | -------- | ------------------------ |
| customer_id     | String   | 客户 ID                  |
| customer_code   | String   | 客户编码                 |
| customer_name   | String   | 客户名称                 |
| customer_region | String   | 销售区域（华东/华北等）  |
| customer_type   | String   | 客户类型（战略/VIP/普通）|
| industry        | String   | 所属行业                 |
| created_at      | DateTime | 创建时间                 |

**排序键**：`ORDER BY (customer_id)`

#### 3.2.3 维度表：channel_dimension（销售渠道表）

**用途**：渠道主数据，用于渠道占比分析。

| 字段名       | 类型   | 说明                         |
| ------------ | ------ | ---------------------------- |
| channel_id   | String | 渠道 ID                      |
| channel_code | String | 渠道编码                     |
| channel_name | String | 渠道名称（直营/电商/经销等） |
| channel_type | String | 渠道类型                     |

**排序键**：`ORDER BY (channel_id)`

#### 3.2.4 事实表：sales_order_fact（销售数据表）

**用途**：存储订单行级销售事实，一行对应「某订单 × 某产品」的销售记录。

| 字段名           | 类型    | 说明           | 主键/索引 |
| ---------------- | ------- | -------------- | --------- |
| order_id         | String  | 订单 ID        |           |
| order_line_id    | String  | 订单行 ID      | PK        |
| order_date       | Date    | 订单日期       | 分区/排序 |
| product_id       | String  | 产品 ID        |           |
| product_category | String  | 产品大类（冗余）|          |
| customer_id      | String  | 客户 ID        |           |
| customer_region  | String  | 客户区域（冗余）|          |
| channel_id       | String  | 渠道 ID        |           |
| quantity         | Int32   | 销售数量       |           |
| unit_price       | Float64 | 成交单价       |           |
| discount_amount  | Float64 | 折扣金额       |           |
| sales_amount     | Float64 | 销售金额       |           |
| currency         | String  | 币种           |           |
| order_status     | String  | 订单状态       |           |

**排序键**：`ORDER BY (order_date, order_id, order_line_id)`  
**引擎**：`MergeTree()`  
**分区建议**：`PARTITION BY toYYYYMM(order_date)`（按月分区，便于时间范围查询）

**业务规则（物理层约束建议）**：

- `sales_amount` ≈ `quantity * unit_price - discount_amount`（允许舍入误差）
- 仅 `order_status IN ('已完成','已发货')` 的行计入销售分析（可在 Cube 或函数层过滤）

| --------------- | ---------------------------------------------- |

## 四、Cube 层设计

### 4.1 SalesCube（销售分析主 Cube）

**目的**：全局销售分析，支持多维度切片。

**数据源**：`sales_order_fact`（过滤有效订单状态）

**维度**：

| 维度名           | 列名             | 类型   | 说明     |
| ---------------- | ---------------- | ------ | -------- |
| order_id         | order_id         | string | 订单 ID  |
| order_line_id    | order_line_id    | string | 订单行   |
| order_date       | order_date       | date   | 订单日期 |
| product_id       | product_id       | string | 产品 ID  |
| product_category | product_category | string | 产品大类 |
| customer_id      | customer_id      | string | 客户 ID  |
| customer_region  | customer_region  | string | 销售区域 |
| channel_id       | channel_id       | string | 渠道 ID  |

**度量**：

| 度量名       | 列名          | 聚合 | 说明     |
| ------------ | ------------- | ---- | -------- |
| quantity     | quantity      | sum  | 销售数量 |
| sales_amount | sales_amount  | sum  | 销售金额 |
| discount     | discount_amount | sum | 折扣合计 |
| order_count  | order_id      | uniq | 订单数   |
| line_count   | order_line_id | uniq | 订单行数 |

**派生度量**：

| 度量名            | 表达式                                              | 说明       |
| ----------------- | --------------------------------------------------- | ---------- |
| avg_unit_price    | if(quantity > 0, sales_amount / quantity, 0)        | 平均单价   |
| avg_order_value   | if(order_count > 0, sales_amount / order_count, 0)  | 客单价     |

### 4.2 ProductSalesCube（产品销售 Cube）

**目的**：按产品/品类聚合销售。

**维度**：`product_id`, `product_category`  
**度量**：`quantity`, `sales_amount`, `order_count`（同上）  
**派生度量**：`avg_unit_price`

> 可与 `product_master` 做维表关联，在查询层补充 `product_name`、`brand` 等展示字段。

### 4.3 CustomerSalesCube（客户销售 Cube）

**目的**：按客户/区域聚合销售。

**维度**：`customer_id`, `customer_region`, `customer_type`（来自维表 join 或事实表冗余）  
**度量**：`quantity`, `sales_amount`, `order_count`  
**派生度量**：`avg_order_value`

### 4.4 ChannelSalesCube（渠道销售 Cube）

**目的**：渠道销售占比与对比。

**维度**：`channel_id`  
**度量**：`quantity`, `sales_amount`, `order_count`

### 4.5 TimeSalesCube（时间维度 Cube）

**目的**：同比、环比、趋势分析。

**维度**：

| 维度名     | 列名       | 类型   | 说明 |
| ---------- | ---------- | ------ | ---- |
| order_date | order_date | date   | 日期 |
| year_month | order_date | string | 年月 |
| quarter    | order_date | string | 季度 |
| year       | order_date | date   | 年份 |

**度量**：`sales_amount`, `quantity`, `order_count`

### 6.2 函数详细设计（节选）

#### 6.2.1 sales.fn.get_summary（销售总览）

**返回值**：

| 字段名           | 类型    | 说明     |
| ---------------- | ------- | -------- |
| total_sales      | Float64 | 总销售额 |
| total_quantity   | Int64   | 总销量   |
| order_count      | Int     | 订单数   |
| avg_order_value  | Float64 | 客单价   |

**参数**：`limit`（默认 10）、`metric`（`sales_amount` / `quantity`）、`start_date`、`end_date`

与利润分析示例类似，指标替换为 `sales_amount`、`quantity`、`order_count`，增长率字段为 `yoy_growth` / `mom_growth`。

### 9.1 风险识别

| 风险等级 | 风险描述                     | 缓解措施                           |
| -------- | ---------------------------- | ---------------------------------- |
| 中       | 事实表与维表 ID 不一致       | 导入前校验 product_id / customer_id |
| 中       | 订单状态口径不统一           | Cube 层统一过滤条件并文档化        |
| 低       | 冗余字段（品类、区域）漂移   | ETL 从维表同步或定期校验           |
| 低       | 与生产 KPI 表语义混淆        | 本体命名空间分离，不做跨域链接     |

### 9.2 验收标准

| 验收项   | 验收标准                                   |
| -------- | ------------------------------------------ |
| 物理表   | 4 张表可查询，样例数据可关联               |
| Cube     | 5 个 Cube 维度/度量/派生度量完整           |
| 对象类型 | 5 种对象类型与属性映射正确                 |
| 链接类型 | 8 种链接可在本体图中表达                   |
| 本体函数 | 至少 5 个核心函数可执行并返回合理结果      |
| 业务问答 | 能回答总览、Top 产品、区域与趋势类问题     |

## 一、模拟数据概述

### 1.1 模拟数据的必要性

由于以下原因，部分数据需要通过模拟方式生成：

| 原因 | 说明 |
|------|------|
| 数据源缺失 | Excel台账为空模板，无法导入数据 |
| 数据源不可用 | 部分属性在现有数据库表中无来源 |
| 完整性要求 | 演示和测试需要完整的业务场景数据 |

### 1.2 模拟数据的使用场景

| 场景 | 说明 |
|------|------|
| 开发测试 | 在真实数据到位前进行功能开发 |
| 原型演示 | 展示系统能力给业务人员 |
| 规则验证 | 验证数据勾稽关系和计算逻辑 |
| 性能测试 | 评估系统在大数据量下的性能 |

```

│                    模拟数据分层策略                           │
```

## 三、模拟数据规格

### 3.1 Project（项目基础信息）

#### 模拟规则

| 字段 | 模拟方式 | 规则 | 参考来源 |
|------|---------|------|---------|
| project_id | 直接使用现有值 | 从tb_project_major_cost提取唯一值 | 现有数据 |
| project_name | 模拟+格式 | "项目-" + project_id后4位 | 参考行业命名 |
| project_code | 模拟+格式 | "PJ-" + project_id | 参考公司编码规则 |
| project_status | 随机选择 | 在建:60%, 竣工:30%, 结算:10% | 行业分布 |
| monthly_budget_approved | 逻辑推断 | 竣工项目为true，在建项目按比例 | - |

### 3.2 ProjectOutput（项目产值）

#### 模拟规则

| 字段 | 模拟方式 | 规则 | 参考来源 |
|------|---------|------|---------|
| project_id | 关联Project | 使用Project表的project_id | 现有数据 |
| report_period | 时间序列 | 最近12个月 | - |
| confirmed_output | 范围随机 | 100万-5000万 | 行业产值范围 |
| unconfirmed_output | 比例计算 | confirmed_output的0-30% | 行业经验 |
| total_output | 计算 | confirmed + unconfirmed | - |

#### 模拟计算逻辑

```yaml
# 产值模拟计算

规则1：confirmed_output范围
  - 最小值: 1,000,000 (100万)
  - 最大值: 50,000,000 (5000万)
  - 分布: 正态分布，集中在500万-3000万

规则2：unconfirmed_output比例
  - 最小值: confirmed_output * 0%
  - 最大值: confirmed_output * 30%
  - 平均值: confirmed_output * 10%

规则3：total_output计算
  - total_output = confirmed_output + unconfirmed_output

规则4：时间序列
  - report_period: 最近12个月（YYYY-MM格式）
  - 每个项目有12条记录
```

#### 勾稽关系

```yaml
# ProjectOutput勾稽关系

关系1：project_id必须存在于Project表
  - 约束: FK(project_id) → Project.project_id
  - 检查: SELECT COUNT(*) WHERE project_id NOT IN Project = 0

关系2：时间连续性
  - 每个项目应该有连续12个月的记录
  - 检查: 每个项目的report_period覆盖最近12个月

关系3：数值合理性
  - confirmed_output > 0
  - unconfirmed_output >= 0
  - unconfirmed_output <= confirmed_output * 0.5
  - total_output = confirmed_output + unconfirmed_output
```

### 3.3 ProjectCost（已合并数据）

#### 合并后数据规格

| 字段 | 来源 | 说明 |

| project_id | tb_project_major_cost | 直接使用 |

| incurred_cost | tb_project_cost_pay_detail | 直接使用（需关联） |
| unconfirmed_cost | tb_project_cost_unconfirmed_analyse | 直接使用（需关联） |
| total_cost | 计算 | incurred + unconfirmed |

#### 勾稽关系

```yaml
# ProjectCost勾稽关系

关系1：project_id必须存在于Project表
  - 约束: FK(project_id) → Project.project_id

关系2：数值合理性
  - target_cost > 0
  - incurred_cost >= 0
  - unconfirmed_cost >= 0
  - incurred_cost <= target_cost * 1.2 (允许超支20%)

关系3：total_cost计算
  - total_cost = incurred_cost + unconfirmed_cost
  - 误差容许: < 0.01
```

#### alert_level转换规则

```yaml
# risk_value → alert_level 转换

转换规则:
  - risk_value < 80: alert_level = "green"
  - risk_value >= 80 AND risk_value < 120: alert_level = "yellow"
  - risk_value >= 120: alert_level = "red"

示例:
  - risk_value: 65 → alert_level: "green" (正常)
  - risk_value: 95 → alert_level: "yellow" (关注)
  - risk_value: 135 → alert_level: "red" (预警)
```

## 四、模拟数据生成计划

### 4.1 生成顺序

```
Step 1: Project (项目基础信息)
    ↓
Step 2: ProjectOutput (产值数据)
    ↓
Step 3: ProjectCost (成本数据 - 合并)
    ↓
Step 4: RiskAlert (风险预警)
    ↓
Step 5: ProjectIndicator (指标数据 - 确认后)
```

### 4.2 数据量规划

| 表名 | 记录数 | 说明 |
|------|--------|------|
| Project | 10 | 基于现有project_id数量 |
| ProjectOutput | 120 | 每个项目12个月 |
| ProjectCost | 50 | 每个项目约5条成本记录 |
| RiskAlert | 30 | 每个项目约3条风险记录 |

### 4.3 生成脚本模板

```python
# 模拟数据生成脚本模板

class MockDataGenerator:
    def __init__(self, config):
        self.config = config
    
    def generate_project(self):
        """生成Project数据"""
        # 使用现有project_id
        project_ids = self.fetch_existing_project_ids()
        
        projects = []
        for pid in project_ids:
            project = {
                'project_id': pid,
                'project_name': f'项目-{pid[-4:]}',
                'project_code': f'PJ-{pid}',
                'project_status': self.random_status(),
                'monthly_budget_approved': self.random_approved()
            }
            projects.append(project)
        return projects
    
    def generate_project_output(self, projects):
        """生成ProjectOutput数据"""
        outputs = []
        report_periods = self.generate_report_periods(12)
        
        for project in projects:
            for period in report_periods:
                confirmed = self.random_output()
                unconfirmed = confirmed * self.random_ratio(0, 0.3)
                
                output = {
                    'project_id': project['project_id'],
                    'report_period': period,
                    'confirmed_output': confirmed,
                    'unconfirmed_output': unconfirmed,
                    'total_output': confirmed + unconfirmed
                }
                outputs.append(output)
        return outputs
    
    def validate_references(self, data, reference_table, field):
        """验证外键引用"""
        ref_ids = set([r[field] for r in reference_table])
        data_ids = set([d[field] for d in data])
        orphan_ids = data_ids - ref_ids
        return len(orphan_ids) == 0, orphan_ids
```

## 一、ETL转换逻辑概述

```
┌─────────────────────────────────────────────────────┐
│                   转换流程                           │
│                                                     │
│  数据源 → [字段映射] → [类型转换] → [业务规则] → 目标表 │
│         ↓           ↓           ↓                  │
│    映射表       转换规则      派生计算               │
```

## 三、派生字段计算规则

### 3.1 date_key计算

```python
def calculate_date_key(report_period: str) -> int:
    """
    根据报告期间计算date_key
    格式：YYYYMMDD（最后一天）
    """
    year = int(report_period[:4])
    month = int(report_period[5:7])
    
    # 获取当月最后一天
    if month in [1, 3, 5, 7, 8, 10, 12]:
        day = 31
    elif month == 2:
        if (year % 4 == 0 and year % 100 != 0) or year % 400 == 0:
            day = 29
        else:
            day = 28
    else:
        day = 30
    
    return year * 10000 + month * 100 + day
```

## 五、数据质量校验规则

### 5.1 通用校验规则

| 规则ID | 规则描述 | 校验逻辑 | 错误处理 |
|--------|---------|---------|---------|
| DQ001 | project_id不能为空 | len(project_id) > 0 | 拒绝记录 |
| DQ002 | report_period格式验证 | 正则匹配^\d{4}-\d{2}$ | 拒绝记录 |
| DQ003 | 数值字段非负 | value >= 0 | 拒绝记录 |
| DQ004 | date_key格式验证 | 正则匹配^\d{8}$ | 重新计算 |
| DQ005 | 日期逻辑校验 | due_date > payment_date | 警告记录 |

### 5.2 业务特定校验规则

| 规则ID | 规则描述 | 适用表 | 校验逻辑 |
|--------|---------|-------|---------|
| DQ006 | 保证金金额校验 | fact_bond | bond_amount = returned_amount + unreturned_amount |
| DQ007 | 罚款金额校验 | fact_penalty | penalty_amount > 0 |
| DQ008 | 保险日期校验 | fact_insurance | expiry_date > purchase_date |
| DQ009 | 成本层级校验 | fact_project_cost | cost_level IN ('L1', 'L2', 'L3') |
| DQ010 | 预警级别校验 | fact_project_risk | warning_level IN ('green', 'yellow', 'red') |

## 六、转换脚本结构

### 6.1 脚本目录结构

```
scripts/
├── etl/
│   ├── extract/           # 数据抽取模块
│   ├── transform/         # 数据转换模块
│   │   ├── field_mapping.py    # 字段映射
│   │   ├── type_conversion.py  # 类型转换
│   │   └── business_rules.py   # 业务规则
│   ├── load/              # 数据加载模块
│   └── validate/          # 数据验证模块
├── config/
│   └── mapping_config.yml # 映射配置文件
└── run_etl.py             # ETL执行入口
```

### 6.2 配置文件格式

```yaml
# mapping_config.yml
tables:
  fact_project_cost:
    mappings:
      - source: id
        target: cost_id
        type: string
      - source: projectId
        target: project_id
        type: string
        required: true
      - source: reportPeriod
        target: report_period
        type: string
        required: true
        validator: report_period_validator
    derived_fields:
      - name: date_key
        expression: calculate_date_key(report_period)
```

## 七、异常处理

### 7.1 异常类型

| 异常类型 | 处理策略 |
|---------|---------|
| 数据格式错误 | 记录错误日志，跳过记录 |
| 数据验证失败 | 记录错误日志，跳过记录 |
| 数据库连接失败 | 重试3次，失败则终止 |
| 文件读取失败 | 记录错误日志，终止执行 |
| 网络超时 | 重试3次，失败则跳过 |

### 7.2 错误日志格式

```
[时间戳] [错误级别] [表名] [记录ID] [错误描述]
示例：
2026-06-10 10:30:00 ERROR fact_project_cost COST000001 report_period格式错误: '2025/01'
```

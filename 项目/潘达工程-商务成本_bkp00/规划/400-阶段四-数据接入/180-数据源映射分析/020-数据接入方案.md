## 四、数据接入策略

### 4.1 总体策略

```
数据源 → 抽取 → 转换 → 加载 → 验证
     ↓
   映射表 → 转换规则 → 校验规则
```

### 4.2 数据抽取策略

| 数据源类型 | 抽取方式 | 频率 |
|-----------|---------|------|
| 数据库表 | JDBC连接 | 每日增量 |
| Excel文件 | 文件读取 | 按需导入 |

### 4.3 数据转换策略

基于差异分析报告，设计以下转换规则：

#### 4.3.1 字段映射规则

| 设计字段名 | 实际字段名 | 转换说明 |
|-----------|-----------|---------|
| id | xxx_id | 按表名生成主键名 |
| projectId | project_id | 下划线命名转换 |
| companyId | company_id | 下划线命名转换 |
| reportPeriod | report_period | 下划线命名转换 |

#### 4.3.2 数据类型转换规则

| 设计类型 | 实际类型 | 转换规则 |
|---------|---------|---------|
| DECIMAL(18,2) | Float64 | 保留2位小数 |
| DECIMAL(18,4) | Float64 | 保留4位小数 |
| DATE | Date | 格式转换 |
| INTEGER | Int32 | 直接转换 |

#### 4.3.3 约束处理规则

| 设计约束 | 实际约束 | 处理规则 |
|---------|---------|---------|
| NOT NULL | 可空 | 接入时强制校验 |
| 枚举值约束 | 可空 | 接入时校验枚举值 |

### 4.4 数据加载策略

| 加载类型 | 适用场景 | 说明 |
|---------|---------|------|
| 全量加载 | 首次接入 | 清空现有数据，重新导入 |
| 增量加载 | 日常同步 | 仅加载新增/变更数据 |
| 追加加载 | 历史数据 | 追加历史数据 |

## 五、ETL设计要点

### 5.1 ETL架构

```
┌─────────────────────────────────────────────────────┐
│                    数据源层                          │
│  [数据库表] [Excel文件] [API接口]                     │
└─────────────────────────────────────────────────────┘
                          ↓
┌─────────────────────────────────────────────────────┐
│                    抽取层                            │
│  [数据抽取] [格式转换] [编码处理]                     │
└─────────────────────────────────────────────────────┘
                          ↓
┌─────────────────────────────────────────────────────┐
│                    转换层                            │
│  [字段映射] [类型转换] [业务规则处理]                 │
└─────────────────────────────────────────────────────┘
                          ↓
┌─────────────────────────────────────────────────────┐
│                    加载层                            │
│  [数据写入] [索引创建] [统计更新]                     │
└─────────────────────────────────────────────────────┘
                          ↓
┌─────────────────────────────────────────────────────┐
│                    验证层                            │
│  [数据质量检查] [完整性校验] [一致性验证]             │
└─────────────────────────────────────────────────────┘
```

### 5.2 关键转换逻辑

#### 5.2.1 日期键生成

```
date_key = YEAR(report_period) * 10000 + MONTH(report_period) * 100 + LAST_DAY(report_period)
```

#### 5.2.2 项目状态映射

| 原始状态 | 目标状态 |
|---------|---------|
| 进行中 | 在建 |
| 已完成 | 竣工 |
| 结算中 | 结算 |
| 已归档 | 归档 |

#### 5.2.3 预警级别映射

| 设计值 | 实际值 |
|-------|-------|
| 绿 | green |
| 黄 | yellow |
| 红 | red |

### 5.3 数据质量规则

| 规则ID | 规则描述 | 适用表 |
|--------|---------|-------|
| DQ001 | project_id不能为空 | 所有表 |
| DQ002 | report_period格式为YYYY-MM | 所有事实表 |
| DQ003 | 数值字段不能为负 | 所有数值字段 |
| DQ004 | date_key格式为YYYYMMDD | 所有事实表 |
| DQ005 | warning_level IN ('green', 'yellow', 'red') | fact_project_risk |

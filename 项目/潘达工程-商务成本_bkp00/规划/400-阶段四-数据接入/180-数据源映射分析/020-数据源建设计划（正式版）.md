## 一、计划概述

### 1.1 背景
基于《数据源映射分析报告（正式版）》，制定本数据源建设计划，明确各项任务的执行顺序、技术路径和资源需求，确保理想表能够按时建设完成，为本体落地提供数据支撑。

### 1.2 核心策略
本计划遵循**相对理想型策略**：
- **理想化**：按照业务需求建设理想表，支撑完整的分析需求
- **相对**：基于现实可获得的数据源，采用合理的建设策略
- **平衡**：在理想设计与现实约束之间寻求平衡

### 1.3 建设目标
完成6个理想表的数据源建设，为本体设计和数据对接提供实际的数据支撑。

### 1.4 建设范围

| 优先级 | 理想表名 | 对应Object Type | 来源类型 | 建设策略 |
|--------|---------|----------------|---------|----------|
| P0 | tb_project_indicator | ProjectIndicator | A（直接使用） | 确认表存在，验证数据质量 |
| P0 | tb_project_cost | ProjectCost | B（合并转换） | 合并多个成本表 |
| P0 | tb_project_output | ProjectOutput | C（Excel导入） | 从Excel导入产值数据 |
| P1 | tb_project | Project | B（合并转换） | 合并转换，补充项目基础信息 |
| P1 | tb_project_risk | RiskAlert | B（合并转换） | 转换风险数据，完善关联关系 |
| P2 | tb_project_cost_detail | ProjectCostDetail | D（新建表） | 设计表结构，逐步填充数据 |

### 3.2 Flow流程设计

#### 流程1：Excel产值导入流程

```
Start → Excel读取节点 → 数据转换节点 → 项目ID匹配节点 → 派生字段计算节点 → 数据写入节点 → End
```

**节点说明**：
- **Excel读取节点**：读取Excel台账（表3.1、表3.1.1）
- **数据转换节点**：列名映射、数据类型转换
- **项目ID匹配节点**：通过工程名称匹配项目ID
- **派生字段计算节点**：计算outputTotal等派生字段
- **数据写入节点**：写入tb_project_output表

---

#### 流程2：成本数据合并流程

```
Start → SQL查询节点（tb_project_major_cost）
       → SQL查询节点（tb_project_cost_pay_detail）
       → SQL查询节点（tb_project_cost_unconfirmed_analyse）
       → Python合并节点 → 数据写入节点 → End
```

**节点说明**：
- **SQL查询节点**：分别查询三个成本表
- **Python合并节点**：按项目ID汇总，合并数据
- **数据写入节点**：写入tb_project_cost表

---

#### 流程3：项目基础信息合并流程

```
Start → SQL查询节点（提取project_id）
       → Excel读取节点（补充项目信息）
       → Python合并节点 → 数据写入节点 → End
```

**节点说明**：
- **SQL查询节点**：从现有表提取project_id
- **Excel读取节点**：从Excel台账补充项目名称等信息
- **Python合并节点**：合并数据
- **数据写入节点**：写入tb_project表

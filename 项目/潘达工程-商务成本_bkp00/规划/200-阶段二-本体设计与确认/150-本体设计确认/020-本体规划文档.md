## 一、本体设计概述

### 1.1 设计原则

| 原则 | 说明 | 平台规约要求 |
|------|------|-------------|
| **业务导向** | Object Type面向业务人员，而非技术实现 | 使用稳定的业务英文标识 |
| **平台对齐** | 遵循Palantir本体体系规范 | 参考《本体规划指南》 |
| **知识集成** | 整合阶段一的显性/隐性关系和行业诀窍 | 关联知识标注（E001-E061） |
| **映射一致** | 与阶段二的字段映射矩阵保持一致 | bind_source配置正确 |
| **可扩展性** | 支持后续业务扩展 | 命名空间规范 |
| **企业专识** | 整合企业业务专识（产值双轨制、三级成本分类等） | ER001-ER006 |

### 1.2 核心实体映射概览（61个实体）

| 实体类别 | 实体数量 | Object Type映射 | 优先级 |
|---------|---------|----------------|--------|
| **管理对象** | 19个 | Project, Contract, Material, Equipment, Bond等 | P0 |
| **业务活动** | 17个 | ProjectOutput, ProjectCost, Payment, Receipt等 | P0 |
| **参与方** | 14个 | Employer, Contractor, Supplier, Subcontractor等 | P0 |
| **管理要素** | 11个 | Indicator, Risk, ApprovalFlow, Metric等 | P1 |
| **新增实体** | 11个 | Bond, Penalty, Compensation, Insurance等 | P0 |

### 1.3 平台规范要求

| 规范项 | 要求 | 本项目实现 |
|--------|------|-----------|
| **Object Type命名** | 使用稳定的业务英文标识 | Project、ProjectOutput等 |
| **Link Type定义** | 两端为对象类型code，非Cube名 | belongsTo、contains、hasPart |
| **bind_source配置** | 每个Object Type必须绑定Cube | 见本章4节 |
| **qualified_name格式** | CubeName.member | ProjectCube.project_name |
| **脚本目录结构** | 项目/xxx/脚本/setup/functions/actions | 按规范创建 |

### 3.2 Link Type详细定义

#### 3.2.1 belongsTo（属于关系）

| 属性 | 值 |
|------|------|
| 语义 | 表示一个对象属于另一个对象 |
| 基数 | 多对一（Many-to-One） |
| 方向 | 从子对象指向父对象 |
| 示例 | ProjectOutput → Project（产值属于项目） |

#### 3.2.2 contains（包含关系）

| 属性 | 值 |
|------|------|
| 语义 | 表示一个对象包含另一个对象 |
| 基数 | 一对多（One-to-Many） |
| 方向 | 从父对象指向子对象 |

#### 3.2.3 hasPart（组成关系）

| 属性 | 值 |
|------|------|
| 语义 | 表示一个对象由多个部分组成 |
| 基数 | 一对多（One-to-Many） |
| 方向 | 从整体指向部分 |

#### 3.2.4 signs（签订关系）

| 属性 | 值 |
|------|------|

| 方向 | 从合同指向签订方 |

#### 3.2.5 affects（影响关系）

| 属性 | 值 |
|------|------|
| 语义 | 表示一个对象对另一个对象的影响 |

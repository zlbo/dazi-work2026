## 一、Action概述

### 1.1 设计目标

基于本体设计和业务需求，开发可执行的Action，支持预警通知、审批流程、自动决策等业务场景，实现从分析到执行的闭环。

### 1.2 Action分类

| 分类 | 说明 | 示例 |
|------|------|------|
| **通知类Action** | 发送预警通知、提醒消息 | SendRiskAlert |
| **审批类Action** | 发起审批、审批流转 | ApprovePayment |
| **数据类Action** | 数据更新、状态变更 | UpdateRiskStatus |
| **流程类Action** | 触发业务流程 | TriggerWorkflow |

## 二、通知类Action

### 2.1 SendRiskAlert

**功能**：发送风险预警通知

**触发条件**：
- 风险等级达到"高"或"严重"
- 指标超出预警阈值

**参数**：

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| projectId | STRING | 是 | 项目ID |
| riskType | STRING | 是 | 风险类型 |
| riskLevel | STRING | 是 | 风险等级 |
| riskDescription | STRING | 是 | 风险描述 |
| recipients | LIST | 是 | 接收人列表 |
| priority | STRING | 否 | 优先级（高/中/低） |

**执行动作**：
1. 生成预警消息内容
2. 发送邮件通知
3. 发送钉钉/企业微信通知
4. 记录通知日志

**AI问答映射**：
- "发送风险预警通知"
- "通知相关人员风险情况"

---

### 2.2 SendIndicatorAlert

**功能**：发送指标异常提醒

**触发条件**：
- 指标值超出阈值范围
- 指标变化率超过设定阈值

**参数**：

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| projectId | STRING | 是 | 项目ID |
| indicatorCode | STRING | 是 | 指标编码 |
| currentValue | DECIMAL | 是 | 当前值 |
| targetValue | DECIMAL | 是 | 目标值 |
| varianceRatio | DECIMAL | 是 | 偏差率 |
| recipients | LIST | 是 | 接收人列表 |

**执行动作**：
1. 生成指标异常消息
2. 发送通知给相关责任人
3. 记录通知日志

**AI问答映射**：
- "指标异常，发送提醒"
- "通知指标异常情况"

---

## 三、审批类Action

### 3.1 ApprovePayment

**功能**：发起付款审批

**触发条件**：
- 付款金额超过审批阈值
- 付款类型为大额付款

**参数**：

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| paymentId | STRING | 是 | 付款记录ID |
| projectId | STRING | 是 | 项目ID |
| contractId | STRING | 是 | 合同ID |
| amount | DECIMAL | 是 | 付款金额 |
| paymentType | STRING | 是 | 付款类型 |
| approver | STRING | 是 | 审批人 |
| reason | STRING | 否 | 付款理由 |

**执行动作**：
1. 创建审批工单
2. 通知审批人
3. 记录审批流程

**审批流程**：
```
提交申请 → 部门经理审批 → 财务审批 → 完成
```

**AI问答映射**：
- "发起付款审批"
- "审批这笔付款"

---

### 3.2 ApproveCostAdjustment

**功能**：审批成本调整

**触发条件**：
- 成本调整金额超过阈值
- 成本偏差率超过设定值

**参数**：

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| adjustmentId | STRING | 是 | 调整记录ID |
| projectId | STRING | 是 | 项目ID |
| adjustmentAmount | DECIMAL | 是 | 调整金额 |
| adjustmentReason | STRING | 是 | 调整原因 |
| approver | STRING | 是 | 审批人 |

**执行动作**：
1. 创建成本调整审批工单
2. 通知审批人
3. 记录审批日志

**AI问答映射**：
- "审批成本调整"
- "确认成本调整申请"

**功能**：更新风险状态

**触发条件**：
- 风险处理完成

**参数**：

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| riskId | STRING | 是 | 风险记录ID |
| newStatus | STRING | 是 | 新状态 |
| handledBy | STRING | 是 | 处理人 |
| handleResult | STRING | 是 | 处理结果 |

**执行动作**：
1. 更新风险状态
2. 记录处理日志
3. 通知相关人员

**状态流转**：
```
待处理 → 处理中 → 已处理/已关闭
```

**AI问答映射**：
- "更新风险状态为已处理"
- "关闭风险记录"

| indicatorId | STRING | 是 | 指标记录ID |
| projectId | STRING | 是 | 项目ID |
| newTargetValue | DECIMAL | 是 | 新目标值 |
| reason | STRING | 是 | 调整原因 |

**执行动作**：
1. 更新指标目标值

**AI问答映射**：
- "更新指标目标值"
- "调整利润率目标"

## 五、流程类Action

### 5.1 TriggerMonthlyReport

**功能**：触发月度报告生成

**触发条件**：
- 每月固定时间触发
- 手动触发

**参数**：

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| reportMonth | STRING | 是 | 报告月份 |
| recipients | LIST | 是 | 接收人列表 |
| reportType | STRING | 否 | 报告类型（完整/简版） |

**执行动作**：
1. 收集月度数据
2. 生成报告内容
3. 发送报告给相关人员
4. 归档报告

**AI问答映射**：
- "生成月度报告"
- "发送本月商务报告"

---

### 5.2 TriggerQuarterlyReview

**功能**：触发季度评审流程

**触发条件**：
- 每季度末触发
- 手动触发

**参数**：

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| quarter | STRING | 是 | 季度（Q1/Q2/Q3/Q4） |
| year | INTEGER | 是 | 年份 |

**执行动作**：
1. 收集季度数据
2. 生成评审材料
3. 发起评审流程
4. 记录评审结果

**AI问答映射**：
- "触发季度评审"
- "开始Q2评审流程"

## 六、Action执行流程

### 6.1 Action触发方式

| 触发方式 | 说明 | 示例 |
|---------|------|------|
| **自动触发** | 根据预设条件自动执行 | 风险预警自动通知 |
| **手动触发** | 用户手动执行 | 发起审批 |
| **定时触发** | 按时间表执行 | 月度报告生成 |
| **事件触发** | 响应业务事件 | 付款完成后触发 |

### 6.2 Action调用流程

```
触发条件满足 → 参数校验 → 权限验证 → 执行动作 → 记录日志 → 通知相关人员
```

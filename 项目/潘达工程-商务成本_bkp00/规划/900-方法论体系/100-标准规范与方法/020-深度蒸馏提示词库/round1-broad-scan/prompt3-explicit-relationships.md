# 提示词3：显性关系梳理

## 角色设定
你是一名建筑工程行业的数据架构专家，擅长梳理业务实体间的关系。

## 任务说明
请描述建筑工程商务成本管理中核心实体之间的显性业务关系。

## 关系类型定义
- **包含关系**：整体与部分的关系（如项目包含合同）
- **归属关系**：从属关系（如合同归属项目）
- **关联关系**：业务上的关联（如供应商提供资源）
- **计算关系**：派生或计算关系（如利润=产值-成本）
- **影响关系**：因果或影响关系（如变更影响成本）

## 输出要求
- 每个关系必须说明具体的业务含义
- 明确关系的基数（1:1、1:N、N:M）
- 识别建筑工程行业的特有关系
- 数量要求：不少于30条关系
- 确保关系可映射到平台的Link Type

## 输出格式
请按以下格式输出：

【关系编号】关系名称
- 关系类型：（包含/归属/关联/计算/影响）
- 源实体：（实体名称）
- 目标实体：（实体名称）
- 基数：（1:1 / 1:N / N:M）
- 业务含义：（具体说明该关系的业务意义）
- 行业特点：（说明是否为建筑工程行业特有）
- 映射建议：（建议映射到平台的Link Type类型）

示例：
【R001】项目包含成本
- 关系类型：包含
- 源实体：项目
- 目标实体：成本
- 基数：1:N
- 业务含义：一个项目包含多个成本项
- 行业特点：通用
- 映射建议：Link Type（包含关系）

【R002】成本归属合同项
- 关系类型：归属
- 源实体：成本
- 目标实体：合同项
- 基数：1:N
- 业务含义：一个成本项可归属到多个合同项
- 行业特点：通用
- 映射建议：Link Type（归属关系）

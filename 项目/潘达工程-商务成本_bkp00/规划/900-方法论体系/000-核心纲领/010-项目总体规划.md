## 二、项目背景

### 2.1 业务背景
潘达建工集团需要建立一套**本体驱动的智能决策体系**，用于商务成本管理分析。该体系将：
- 替代传统Excel台账，提供Web版报表展示
- 支持AI助手问答，实现指标查询、规律探寻、根因分析
- 提供可视化图表看板和驾驶舱
- 生成分析报告和决策建议

### 2.2 平台架构说明
**重要说明**：本项目基于**本体驱动的智能决策平台**构建，该平台内置了以下核心能力：

| 平台能力         | 说明                                    | 对本项目的影响                |
| ------------ | ------------------------------------- | ---------------------- |
| **自动Cube生成** | 平台可将数据表自动转换为Cube分析视图                  | 无需手动设计Cube，阶段一输出转换规则即可 |
| **本体层管理**    | 支持Object Type、Link Type、Action Type定义 | 阶段二专注于本体语义设计           |
| **函数引擎**     | 支持Function和Action的开发和注册               | 阶段三专注于业务逻辑封装           |
| **智能问答**     | 支持基于本体的自然语言问答                         | 最终交付的AI能力              |

**四层架构**：

```
┌──────────────────────────────────────────────────────────────────────┐
│                         应用层（面向用户）                              │
│  AI助手问答 │ Web报表 │ 可视化看板 │ 驾驶舱 │ 分析报告                  │
```

## 三、核心策略：相对理想型

**这是本项目的核心指导思想，必须贯穿所有阶段**：

| 维度      | 说明                                      |
| ------- | --------------------------------------- |
| **理想化** | 本体架构和数据架构应该支撑企业商务成本分析的**完整、科学、面向未来**的状态 |
| **相对**  | 理想架构必须基于**现实可获得的数据源**，不能脱离实际            |
| **平衡**  | 在理想设计与现实约束之间寻求平衡                        |

### 3.1 方法优化：预研机制

> **优化说明**：为平衡理想设计与现实落地，增加**预研机制**作为相对理想型策略的补充

**预研机制核心内容**：
1. **预研时机**：在阶段二本体设计期间，可同步启动数据源预研
2. **预研内容**：数据源盘点、差距分析、可行性评估
3. **预研成果**：作为参考资料，不影响理想设计的科学性
4. **预研价值**：提前识别风险，缩短后续落地周期

**预研约束**：
- 预研成果不得作为本体设计的直接依据
- 理想设计必须保持独立性和科学性
- 预研仅作为落地风险的预判和准备

**当前预研成果**：已完成《数据源映射分析报告（预研版）》，存放在 `200-阶段二-本体设计与确认\160-数据源映射分析（预研）` 目录下

## 八、本体建模标准化操作规范（阶段三必读）

### 8.1 规范背景

**问题**：之前本体建模阶段未严格遵循标准化流程，导致字段显示名不完整、分类挂载不规范等问题。

**解决方案**：制定阶段三标准化操作规范，明确操作步骤和质量标准。

### 8.2 快速启动文档定位

> **重要说明**：`本体/ontos/商务成本/快速启动_商务成本.md` 是**本项目专属**的本体建模启动文档，不是跨项目的公共模板。

**文档性质**：
- 由架构师预设，提供标准化操作流程
- **本项目专属性**：仅适用于 `space__panda_construction` 数据空间
- **可复制性**：可作为新项目的模板，但需根据新项目情况修改 space_id、示例路径等

**模板复制流程**：
1. 复制 `快速启动_商务成本.md` → 新项目的 `快速启动_<项目名>.md`
2. 替换文档中的：
   - `space__panda_construction` → 新项目的 space_id
   - `潘达工程-商务成本` → 新项目名称
   - 参照示例路径 → 新项目对应的示例

### 8.3 快速启动文档的局限性

**已知局限**：快速启动文档由架构师预设，不了解项目经理自建的规划体系。

- 未明确要求读取阶段一、阶段二的正式成果

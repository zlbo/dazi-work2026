## 三、数据库表设计

### 3.1 tb_project_output_unconfirmed(待确认产值分析表)

这个表的结构在表3.1的映射文档中已经定义过,这里补充完整字段:

| 字段名 | 类型 | 说明 | 示例 |
|-------|------|------|------|
| id | bigint | 主键 | 自增 |
| project_id | bigint | 项目ID | 外键 |
| year | int | 年份 | 2024 |
| month | int | 月份 | 6 |
| code | varchar(50) | 编码 | '10','20','30','40' |
| description | varchar(500) | 实际形象进度描述 | 实际形象进度的详细描述 |
| contract_summary | varchar(500) | 合同付款摘要 | 合同付款条款的摘要说明 |
| reason | varchar(500) | 待确主要原因 | 待确认产值的主要原因 |
| amount | decimal(18,2) | 待确金额 | 200000.00 |
| remark | varchar(500) | 备注 | 补充说明 |
| created_by | varchar(50) | 创建人 | - |
| created_time | datetime | 创建时间 | - |

**索引设计**:
- PRIMARY KEY (id)
- UNIQUE KEY (project_id, year, month, code)
- INDEX (project_id, year, month)

**表间关系**:
```
tb_project_output_unconfirmed
    ↓ 被引用
表3.1 (通过!N6='表3.1.1'!C8)
    ↓ 被引用
表3
    ↓ 被引用
表1、表2
```

## 六、特殊情况处理

### 6.1 数据校验

| 校验项 | 校验规则 | 说明 |
|-------|---------|------|
| 序号连续 | A5:A8 = 1,2,3,4 | 序号必须连续 |
| 编码规范 | B5:B8 = [10],[20],[30],[40] | 编码必须符合规范 |
| 金额非负 | C8 >= 0 | 待确金额不能为负数 |
| 描述非空 | C5,C6,C7不能为空 | 必填字段不能为空 |

### 6.2 附件要求

| 要求项 | 说明 | 重要性 |

## 七、SQL查询示例

### 7.1 查询待确产值分析

```sql
SELECT 
    code as 编号,
    description as 实际形象进度,
    contract_summary as 合同付款摘要,
    reason as 待确主要原因,
    amount as 待确金额,
    remark as 备注
FROM tb_project_output_unconfirmed
WHERE project_id = ? 
  AND year = ? 
  AND month = ?
ORDER BY code;
```

### 7.2 查询待确金额汇总

```sql
SELECT 
    SUM(amount) as 待确金额汇总
FROM tb_project_output_unconfirmed
WHERE project_id = ? 
  AND year = ? 
  AND month = ?
  AND code = '40';
```

### 7.3 查询待确原因明细

```sql
SELECT 
    reason as 待确原因,
    amount as 金额,
    remark as 备注
FROM tb_project_output_unconfirmed
WHERE project_id = ? 
  AND year = ? 
  AND month = ?
  AND code IN ('10', '20', '30');
```

## 八、数据同步方案

### 8.1 同步流程

```

    ↓
3. 表3.1引用表3.1.1的C8

    ↓
```

# 快速启动 · 商务成本

本文件对应本体实现 **`商务成本`**（`项目/潘达工程-商务成本_bkp00/本体/ontos/商务成本/`）。由扩展动态生成，更新脚手架后会刷新带 `<!-- dazi-onto-scaffold -->` 标记的本文件。

## 0. 任务模式（智能体 / TRAE 必读）

| 模式 | 何时使用 | 允许 | 禁止 |
| --- | --- | --- | --- |
| **独立规划**（**默认**） | 新建本体、能力评估、用户要求「设计方案/规划」 | 在 `plans/` **独立撰写** Markdown；**只读**对照规划示例学结构 | 复制 `资源/examples/` 或 `项目/*/本体/ontos/*` 下 init/seed/函数到本实现；把规划示例全文粘贴为 `plans/` |
| **对照实施** | `plans/` 已定稿且通过自检清单 | 参考示例 init 的 **API 写法**；逐行对照 plans 实现 `setup/`、`functions/` | 无 plans 先写脚本；表/Cube/对象与 plans 不一致 |
| **快速演示** | 用户**明确**要求 demo/跑通示例 | 复制示例改 `space_id`（须在 README 标注 demo） | 冒充已完成的业务规划与实施 |

**默认路径**：§2 规划 → 自检定稿 → §5 实施。**未确认模式时按「独立规划」执行。**

智能体规划阶段提示词：**`onto/planning-design`**（`dazi prompt sync` 后位于 `资源/prompts/onto/`）。

- **数据空间 ID** 权威来源：同目录 `README.md`（与上表同步）。

## 3. 阶段门禁

## 4. 执行脚本前必读（强制）

在终端执行 **`dazi onto script publish` / `script run` / `function run` / `save-test-arguments`** 之前，须通读：

- **[脚本运行常见错误处理](../../../../../资源/docs/onto/脚本运行常见错误处理.md)** — 文档 ID `onto/script-run-troubleshooting` · `dazi docs show onto/script-run-troubleshooting`

| 要点 | 说明 |
| --- | --- |
| setup API | 勿用 `sync_metrics`、`s.cubes.upsert`；Cube 用 `register_cube`；链接参数为 `*_object_type_code` |
| SQL 聚合 | 多列聚合用 `query()` 取 `rows[0]`，勿对 `query_one()` 结果调用 `.get()` |
| 函数输出 | **`return _ontology_fn_body(p)`** + `p.function_result(...)`；**禁止** `output.print_json()` |
| `function run` | 无 `--params` 时**自动回退** `test_arguments` / 脚本 `TEST_ARGUMENTS` |
| `save-test-arguments` | 推荐 **`--function-id domain.fn.name`**；亦可用 `ofn_xxx` |
| init 首次 | 对照 §1.2 **init 检查表**（dim_date、bind_source、define_property） |
| 域成员 | `category_mount` **之后** `s.domain.apply_registry`（§5）；仅 category 时域树 kind 灰色 |

同步：`dazi docs sync` → `资源/docs/onto/`。实施阶段提示词：**`onto/script-publish-run`**（须先完成 §2–§3）。

## 7. 侧栏与 onto-assets

- **搭子 → Onto 本体**：展开业务项目 → 本实现 → 对象/函数/活动/规则
- **`本体/onto-assets/`**：平台资源缓存；改函数代码后可在 Explorer 对 `.py` **提交**
- 脚本**真理源**在本实现的 `setup/`、`functions/`（非 onto-assets）

## 8. 推荐步骤

1. 阅读 §0 任务模式；规划任务使用提示词 **onto/planning-design**
2. 在 `plans/` **独立撰写**本体规划并完成自检（§2、§3）
3. 通读 §4 [脚本运行常见错误处理](../../../../../资源/docs/onto/脚本运行常见错误处理.md)
4. 在 `setup/` 按 plans 实现 init / seed 并发布执行（§5–§6）
5. 在 `functions/` 实现并 **全部** publish（`--register-function-id`）→ `function list` 门禁
6. 最后 run `setup/*_category_mount.py`（平台分类 + **末尾域成员** `s.domain.apply_registry`）
7. `function run` → `save-test-arguments`；管理端验证 **本体域** 各 kind count > 0
8. 侧栏 **Onto** 运维 `onto-assets`

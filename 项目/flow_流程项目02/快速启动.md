# 快速启动

欢迎开始 **数据流程项目** 开发（`flow_流程项目02`）。

## 1. 流程目录结构

每个流程是 `流程/<流程名>/` 目录：

| 路径 | 说明 |
|------|------|
| `flow.json` | 画布真理源（节点配置 + 连线，代码已剥离） |
| `flow.meta.json` | flowId、node_uuid 映射（CLI 维护，勿手改） |
| `节点/<名>/code.*` | SQL / Python 节点代码 |
| `变量/<名>.json` | 调试 Run 变量预览（只读，pull/sync 生成） |
| `_run/` | 测试/运行产物（`*.last-error.md` 等） |

> 流程**不绑定数据空间**；`connectionId` 仅为数据库节点的配置字段。

## 2. 新建 / 拉取流程

资源管理器（推荐 **表单**）：

| 操作 | 菜单 |
|------|------|
| 新建流程 | 右键 **`流程/`** → **新建流程** |
| 拉取平台流程 | 右键 **`流程/`** 或项目根 → **拉取平台流程** |

拉取表单可从平台 Flow 列表选择，或手动填写 Flow ID；新建表单填写名称与说明后自动创建并拉取。

命令行（`dazi-work` 根）：

```powershell
.\scripts\dazi.ps1 flow project pull --flow <flowId> --dir "项目\flow_流程项目02\流程\<流程名>"
```

拉取后每个流程目录会生成 **`快速启动.md`**（含 flowId 与本流程常用命令）。

## 3. 开发与提交

1. 编辑 `节点/<名>/code.sql` 或 `code.py`
2. 右键 **测试运行节点** 或 `flow run node-exec --node <uuid>`
3. 右键 **提交节点** / **提交流程** 或 `flow node push` / `flow project push --canvas`
4. 查看表变量：设计器 **output_variable_name** 旁 📊，或 `flow variable pull --name <名>`

命令前缀：`.\scripts\dazi.ps1 flow`（在 **dazi-work 根**；进入流程目录后可用 `flow` 子命令省略 `--dir`）

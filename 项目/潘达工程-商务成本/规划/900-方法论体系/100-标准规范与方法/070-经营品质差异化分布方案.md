> **版本**：V1.0  
> **创建日期**：2026年6月12日  
> **适用范围**：潘达工程-商务成本智能决策体系

> **核心定位**：本方案定义模拟数据经营品质差异化分布的具体策略，确保模拟数据覆盖从"卓越"到"危机"的完整经营品质光谱，支撑多元决策分析场景。

## 一、方案概述

### 1.1 核心理念

> **核心洞见**：数据质量 ≠ 经营品质
>
> - **数据质量**：数据本身的技术正确性（业务规则、勾稽关系、取值范围）
> - **经营品质**：数据所反映的业务运营状态（健康度、风险水平、问题严重程度）
>
> **真实业务世界**：即使数据质量100%合格，企业经营仍然呈现从卓越到危机的差异化分布。
>
> **模拟数据目标**：同时满足数据质量要求和经营品质差异化分布。

### 1.2 差异化分布的意义

| 分析场景 | 需要的经营品质 | 传统方法的问题 |
|---------|--------------|--------------|
| 标杆学习 | A级（卓越） | 数据全为优秀，无法学习 |
| 经验借鉴 | B级（优良） | 数据过于完美，缺乏参考 |
| 常规管理 | C级（一般） | 数据问题太少，不真实 |
| 风险管控 | D级（预警） | 缺乏风险数据，无法演练 |
| 危机处理 | E级（危机） | 没有危机场景，无法训练 |

### 1.3 分布目标

**五级分布目标**：

| 等级 | 名称 | 占比 | 项目数量 | 说明 |
|------|------|------|---------|------|
| A级 | 卓越 | 10% | 10个 | 各项指标优秀 |
| B级 | 优良 | 25% | 25个 | 指标良好，少量问题 |
| C级 | 一般 | 35% | 35个 | 指标正常，常规问题 |
| D级 | 预警 | 20% | 20个 | 多项异常，较大风险 |
| E级 | 危机 | 10% | 10个 | 严重问题，面临困难 |

## 二、经营品质分级模型

### 2.1 分级维度

**五维度评分模型**：

| 维度 | 权重 | 衡量指标 | 数据来源 |
|------|------|---------|---------|
| **盈利能力** | 25% | 利润率、成本利润率 | fact_project_indicator |
| **运营效率** | 20% | 工期偏差率、成本偏差率 | fact_project_indicator |
| **资金状况** | 25% | 收款率、现金流、逾期率 | fact_receivable, fact_cash_flow |
| **风险状况** | 15% | 风险数量、高风险占比 | fact_risk |
| **成长性** | 15% | 营收增长率、新签合同增长率 | fact_project_output |

### 2.2 综合评分计算

**评分公式**：

```
综合评分 = 盈利能力得分 × 25% 
         + 运营效率得分 × 20% 
         + 资金状况得分 × 25% 
         + 风险状况得分 × 15% 
         + 成长性得分 × 15%
```

**等级划分**：

| 等级 | 综合评分 | 说明 |
|------|---------|------|
| A级 | ≥90分 | 卓越 |
| B级 | 80-89分 | 优良 |
| C级 | 70-79分 | 一般 |
| D级 | 60-69分 | 预警 |
| E级 | <60分 | 危机 |

### 2.3 各等级详细特征

#### 2.3.1 A级（卓越）特征

**财务指标**：

| 指标 | 数值 | 说明 |
|------|------|------|
| 利润率 | ≥15% | 优秀盈利水平 |
| 成本偏差率 | ≤0% | 成本节约或持平 |
| 工期偏差率 | ≤2% | 提前或按时完成 |
| 收款率 | ≥95% | 几乎全额收款 |
| 应收账款逾期率 | <5% | 极少逾期 |

**业务特征**：

| 特征 | 说明 |
|------|------|
| 项目数量 | 占总数10% |
| 项目类型 | 大型房建为主 |
| 业务模式 | 直营为主 |
| 风险事件 | 无或极少 |
| 业主资质 | 优质业主 |

#### 2.3.2 E级（危机）特征

**财务指标**：

| 指标 | 数值 | 说明 |
|------|------|------|
| 利润率 | <0% | 亏损状态 |
| 成本偏差率 | >15% | 严重超支 |
| 工期偏差率 | >20% | 严重延误 |
| 收款率 | <50% | 大量未收 |
| 应收账款逾期率 | >50% | 大量逾期 |

**业务特征**：

| 特征 | 说明 |
|------|------|
| 项目数量 | 占总数10% |
| 项目类型 | 各种类型均有 |
| 业务模式 | 合作/挂靠为主 |
| 风险事件 | 多发高危风险 |
| 业主资质 | 问题业主 |

## 三、经营品质注入规则

### 3.1 注入原则

| 原则 | 说明 |
|------|------|
| **差异化原则** | 每个等级的数据特征必须可区分 |
| **真实性原则** | 注入的问题必须符合业务实际 |
| **关联性原则** | 问题之间有关联（如逾期+现金流紧张） |
| **隐蔽性原则** | C级问题不易察觉，需深入分析 |

### 3.2 各等级注入规则

#### 3.2.1 A级注入规则（无问题或极少）

```yaml
问题注入清单:
  - 安全事故: 0次
  - 工期延误: 0-5%偏差
  - 成本超支: 0%或负偏差（节约）
  - 应收账款逾期: 0-5%, 逾期天数<30
  - 变更签证: 0-2个,金额<2%
  - 索赔纠纷: 0-1个
  - 法律诉讼: 0
  - 现金流: 净流入
```

#### 3.2.2 B级注入规则（少量小问题）

```yaml
问题注入清单:
  - 安全事故: 0次
  - 工期延误: 5-10%偏差
  - 成本超支: 0-3%超支
  - 应收账款逾期: 5-15%, 逾期天数30-60
  - 变更签证: 2-5个,金额2-5%
  - 索赔纠纷: 1-2个
  - 法律诉讼: 0-1个
  - 现金流: 基本平衡
```

#### 3.2.3 C级注入规则（常规问题）

```yaml
问题注入清单:
  - 安全事故: 0-1次
  - 工期延误: 10-15%偏差
  - 成本超支: 3-8%超支
  - 应收账款逾期: 15-30%, 逾期天数60-90
  - 变更签证: 5-10个,金额5-10%
  - 索赔纠纷: 2-3个
  - 法律诉讼: 1-2个
  - 现金流: 阶段性紧张
```

#### 3.2.4 D级注入规则（较大风险）

```yaml
问题注入清单:
  - 安全事故: 1-2次
  - 工期延误: 15-25%偏差
  - 成本超支: 8-15%超支
  - 应收账款逾期: 30-50%, 逾期天数90-180
  - 变更签证: 10-20个,金额10-20%
  - 索赔纠纷: 3-5个
  - 法律诉讼: 2-3个
  - 现金流: 经常紧张
  - 高风险预警: 2-4个
```

#### 3.2.5 E级注入规则（危机状态）

```yaml
问题注入清单:
  - 安全事故: ≥2次
  - 工期延误: >25%偏差或停工
  - 成本超支: >15%超支
  - 应收账款逾期: >50%, 逾期天数>180
  - 变更签证: >20个,金额>20%
  - 索赔纠纷: >5个
  - 法律诉讼: ≥3个
  - 现金流: 持续净流出
  - 高风险预警: ≥5个
  - 债务违约风险: 存在
```

## 四、问题类型与注入策略

### 4.1 问题类型分类

| 安全事故 | P004 | 高 | 停工整改、声誉损失 |
| 质量缺陷 | P005 | 中 | 返工成本、业主扣款 |
| 合同纠纷 | P006 | 中 | 法律诉讼、声誉损失 |
| 资金断裂 | P007 | 极高 | 项目停工、债务违约 |

### 4.2 问题关联关系

**问题传导链**：

```
应收账款逾期（P001）
    ↓
现金流紧张
    ↓
资金断裂（P007）
    ↓
项目停工
    ↓
工期延误（P002） + 成本超支（P003）
    ↓
利润下降/亏损
    ↓
安全投入减少
    ↓
安全事故（P004）
```

## 五、经营品质时间演化

### 5.1 时间演化模型

**三年周期演化**：

```
2020年（基准年）
    ↓
2021年（成长年）
    ├── A级企业：继续卓越（80%维持，20%略降）
    ├── B级企业：部分晋升A（30%），部分降至C（20%）
    ├── C级企业：分化（30%晋升B，30%降至D）
    ├── D级企业：部分恶化至E（40%），部分改善至C（30%）
    └── E级企业：多数崩溃（60%退出或破产）
    ↓
2022年（震荡年）
    ├── A级企业：保持（90%维持）
    ├── B级企业：稳定（70%维持）
    ├── C级企业：继续分化
    ├── D级企业：高风险期（行业下行）
    └── E级企业：危机爆发期
    ↓
2023-2024年（恢复年）
    ├── A级企业：强者恒强
    ├── B/C级：缓慢恢复
```

## 七、验证方法

### 7.1 分布验证

```sql
-- 验证1：经营品质分布
SELECT 
    quality_level,
    COUNT(*) as project_count,
    ROUND(COUNT(*) * 100.0 / SUM(COUNT(*)) OVER(), 2) as percentage
FROM (
    SELECT 
        project_key,
        CASE 
            WHEN indicator_score >= 90 THEN 'A级'
            WHEN indicator_score >= 80 THEN 'B级'
            WHEN indicator_score >= 70 THEN 'C级'
            WHEN indicator_score >= 60 THEN 'D级'
            ELSE 'E级'
        END as quality_level
    FROM fact_project_indicator
    WHERE date_key = 202406  -- 最近一期
) t
GROUP BY quality_level
ORDER BY quality_level;

-- 期望结果：A级10%, B级25%, C级35%, D级20%, E级10%
```

### 7.2 问题覆盖率验证

```sql
-- 验证2：问题覆盖率
SELECT 
    problem_type,
    affected_projects,
    total_projects,
    ROUND(affected_projects * 100.0 / total_projects, 2) as coverage_rate
FROM (
    SELECT 
        '应收账款逾期' as problem_type,
        COUNT(DISTINCT project_key) as affected_projects
    FROM fact_receivable
    WHERE overdue_days > 90
) cross join (
    SELECT COUNT(*) as total_projects FROM dim_project
);

-- 期望：D/E级项目问题覆盖率≥80%
```

### 7.3 等级特征验证

```sql
-- 验证3：A级项目特征
SELECT 
    'A级项目特征' as check_type,
    AVG(profit_rate) as avg_profit_rate,
    AVG(cost_deviation_rate) as avg_cost_deviation,
    AVG(schedule_deviation_rate) as avg_schedule_deviation,
    AVG(collection_ratio) as avg_collection_ratio
FROM fact_project_indicator
WHERE quality_level = 'A级';

-- 期望：利润率≥15%, 成本偏差≤0%, 工期偏差≤2%, 收款率≥95%

-- 验证4：E级项目特征
SELECT 
    'E级项目特征' as check_type,
    AVG(profit_rate) as avg_profit_rate,
    AVG(cost_deviation_rate) as avg_cost_deviation,
    AVG(schedule_deviation_rate) as avg_schedule_deviation,
    AVG(collection_ratio) as avg_collection_ratio
FROM fact_project_indicator
WHERE quality_level = 'E级';

-- 期望：利润率<0%, 成本偏差>15%, 工期偏差>20%, 收款率<50%
```

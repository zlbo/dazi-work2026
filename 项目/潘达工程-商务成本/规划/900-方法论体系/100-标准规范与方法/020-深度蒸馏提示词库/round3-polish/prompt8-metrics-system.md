# 提示词8：指标体系构建

## 角色设定
你是一名建筑工程行业的绩效指标专家，擅长构建完整的指标体系。

## 任务说明
请为建筑工程商务成本管理构建一套完整的指标体系，包括指标名称、计算规则、预警阈值和数据来源。

## 指标类别
1. **产值类指标**：确认产值、待确产值、累计产值、产值完成率、产值增长率
2. **成本类指标**：实际成本、目标成本、偏差成本、成本偏差率、成本节约率
3. **利润类指标**：利润总额、净利润、利润率、成本利润率
4. **资金类指标**：应收金额、已收金额、未收金额、收款率、资金周转率
5. **风险类指标**：风险发生率、风险损失率、预警次数、已解决次数
6. **综合类指标**：项目健康度、综合成本指标

## 输出要求
- 每个指标必须有明确的计算规则
- 定义预警阈值和预警逻辑
- 说明数据来源和计算依赖
- 确保指标可映射到平台的Cube度量

## 输出格式
请按以下格式输出：

【指标编号】指标名称
- 指标类别：（产值类/成本类/利润类/资金类/风险类/综合类）
- 业务定义：（指标的业务含义）
- 计算公式：（具体计算方法）
- 参数说明：（公式中各参数的含义）
- 预警阈值：（触发预警的数值）
- 预警逻辑：（如何判断是否需要预警）
- 数据来源：（所需数据的物理表或Cube）
- 平台映射：（建议映射到平台的Cube度量）
- 行业基准：（行业平均水平或参考值）

示例：
【M001】成本偏差率
- 指标类别：成本类
- 业务定义：实际成本与预算成本的偏差程度
- 计算公式：(实际成本 - 预算成本) / 预算成本 × 100%
- 参数说明：实际成本来自成本台账，预算成本来自预算表
- 预警阈值：> 5%
- 预警逻辑：当成本偏差率连续3个月超过5%时触发预警
- 数据来源：fact_project_cost、fact_budget
- 平台映射：Cube度量（ProjectCostCube）
- 行业基准：通常控制在±3%以内

【M002】资金周转率
- 指标类别：资金类
- 业务定义：资金周转的效率
- 计算公式：主营业务收入 / 平均应收账款
- 参数说明：主营业务收入来自产值确认表，平均应收账款来自财务报表
- 预警阈值：< 3次/年
- 预警逻辑：资金周转率低于行业平均水平时触发预警
- 数据来源：fact_project_output、dim_finance
- 平台映射：Cube度量（FinanceCube）
- 行业基准：建筑行业平均约4-6次/年

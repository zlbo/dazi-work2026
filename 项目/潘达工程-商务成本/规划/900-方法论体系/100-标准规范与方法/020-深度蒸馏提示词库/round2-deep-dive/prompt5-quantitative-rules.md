# 提示词5：量化规则提取

## 角色设定
你是一名建筑工程行业的数据分析专家，擅长将业务知识转化为量化规则。

## 任务说明
基于隐性关系清单，请尝试量化这些关系，提取可用于计算和预警的规则。

## 输出要求
- 对于每条隐性关系，评估是否可以量化
- 如果可量化，给出具体的计算公式和阈值
- 说明如何将量化规则应用于预警系统
- 确保量化规则可映射到平台的派生属性或度量

## 输出格式
请按以下格式输出：

【规则编号】规则名称
- 对应隐性关系：（关联的隐性关系编号）
- 量化可能性：（可量化/部分可量化/难以量化）
- 量化公式：（如果可量化，给出具体公式）
  - 公式表达式：（数学表达式）
  - 参数定义：（公式中各参数的含义）
- 阈值定义：（预警阈值或临界点）
- 预警逻辑：（如何触发预警）
- 数据来源：（所需数据的来源）
- 平台映射：（建议映射到平台的派生属性或度量）

示例：
【QR001】成本超支预警规则
- 对应隐性关系：IR001
- 量化可能性：可量化
- 量化公式：
  - 公式表达式：成本偏差率 = (实际成本 - 预算成本) / 预算成本 × 100%
  - 参数定义：实际成本来自成本台账，预算成本来自预算表
- 阈值定义：成本偏差率 > 5% 时触发预警
- 预警逻辑：当成本偏差率连续3个月超过5%时发送预警通知
- 数据来源：fact_project_cost、fact_budget
- 平台映射：派生属性（成本偏差率）+ Action Type（预警通知）

【QR002】资金周转预警规则
- 对应隐性关系：IR002
- 量化可能性：可量化
- 量化公式：
  - 公式表达式：资金周转率 = 主营业务收入 / 平均应收账款
  - 参数定义：主营业务收入来自产值确认表，平均应收账款来自财务报表
- 阈值定义：资金周转率 < 3次/年时触发预警
- 预警逻辑：资金周转率低于行业平均水平时发送预警
- 数据来源：fact_project_output、dim_finance
- 平台映射：派生属性（资金周转率）+ Rule（预警规则）

# 目标企业选择与素材采集方案

> **版本**：V1.0  
> **创建日期**：2026年6月12日  
> **适用范围**：潘达工程-商务成本智能决策体系  
> **关联文档**：《050-模拟数据方法.md》（方法论）、《080-模拟数据设计方案.md》（执行方案）  
> **核心定位**：本方案定义目标企业的选择标准、类型、以及素材采集的具体方法，确保模拟数据素材真实、可信、有代表性。

---

## 一、方案概述

### 1.1 核心原则

> **核心原则**：选取真实企业作为素材来源，而非凭空杜撰。工作量小、仿真度高、可追溯。

**与传统数据模拟的区别**：

| 对比维度 | 传统方法（凭空杜撰） | 本方法（目标企业参考） |
|---------|---------------------|---------------------|
| **工作量** | 大 | 小（直接参考真实数据） |
| **仿真度** | 低（容易失真） | 高（基于真实业务） |
| **可追溯性** | 差 | 好（可追溯数据来源） |
| **数据质量** | 难以保证 | 有行业标准参照 |
| **问题场景** | 难以模拟 | 可从真实事件中提炼 |

### 1.2 选择目标

| 目标 | 说明 |
|------|------|
| **数量** | 选取3-5家目标企业 |
| **类型覆盖** | 涵盖行业龙头、区域龙头、问题企业、新兴企业 |
| **业务匹配** | 与本项目业务范围（房建、市政、安装、基础设施）高度匹配 |
| **体量匹配** | 年营收10亿+企业，确保数据丰富度 |

## 四、素材采集方法

| 资料类型 | 采集渠道 | 数据价值 | 采集优先级 |

| **年报** | 上交所/深交所披露平台 | 营收、成本、利润、项目信息 | P0 |
| **半年报** | 交易所披露平台 | 中期经营数据 | P0 |
| **季报** | 交易所披露平台 | 季度经营数据 | P1 |
| **招标公告** | 全国公共资源交易平台 | 项目信息、合同金额 | P0 |
| **中标公告** | 各地公共资源交易中心 | 中标信息、中标金额 | P0 |
| **信用评级报告** | 评级机构（东方金诚、中诚信等） | 经营状况、风险评估 | P1 |
| **司法文书** | 裁判文书网 | 法律纠纷、违约情况 | P2 |
| **执行信息** | 中国执行信息公开网 | 被执行人、失信人 | P2 |
| **新闻报道** | 行业媒体（建筑时报、中国建设报等） | 重大事件、风险事件 | P2 |
| **上市问询函** | 交易所官网 | 监管关注点、经营问题 | P2 |

### 4.2 采集优先级定义

| 优先级 | 资料类型 | 采集内容 | 用途 |
|--------|---------|---------|------|

## 五、素材加工方法

### 5.1 数据清洗规则

| 清洗类型 | 规则 | 说明 |
|---------|------|------|
| **去标识化** | 将真实企业名替换为模拟企业名 | 保护商业秘密 |
| **数值归一化** | 将绝对值转为相对值（占比、比率） | 提取业务规律 |
| **时间标准化** | 统一时间格式和粒度 | 便于分析 |
| **异常值处理** | 识别和处理不合理数据 | 保证数据质量 |
| **缺失值填补** | 使用行业均值填补缺失数据 | 保证完整性 |

### 5.2 数据蒸馏方法

**三层蒸馏模型**：

| 层级 | 输入 | 处理方法 | 输出 |
|------|------|---------|------|
| **表层特征** | 年报数据 | 数值提取、结构化 | 项目表、财务表 |
| **业务特征** | 表层数据 | 业务分类、规律挖掘 | 业务模式、成本结构 |
| **知识特征** | 业务特征 | 知识抽象、规则提炼 | 业务规则、经营品质 |

**示例：项目数据蒸馏**

```
原始数据（年报）：
"2024年新签合同约300亿元，其中房建项目约180亿元，市政项目约80亿元..."

蒸馏处理：
1. 提取：房建180亿、市政80亿
2. 分类：房建30个项目（平均6亿）、市政20个项目（平均4亿）
3. 映射：30个项目 → dim_project + fact_project_output
         比例：房建:市政 = 6:4
```

### 5.3 数据混合规则

**多源混合原则**：

| 原则 | 说明 | 示例 |
|------|------|------|
| **比例分配** | 按目标企业类型分配数据比例 | 行业龙头40%、区域龙头35%、问题企业25% |
| **类型匹配** | 将企业类型与经营品质关联 | 问题企业 → D/E级项目 |
| **业务匹配** | 将企业业务结构与项目类型关联 | 上海建工 → 房建60%、市政30% |
| **时间错位** | 避免数据时间过于集中 | 混合不同年份数据 |

## 八、数据溯源管理

### 8.1 溯源体系

**四层溯源结构**：

| 层级 | 溯源内容 | 溯源粒度 |
|------|---------|---------|
| **企业层** | 数据来源企业 | 企业名称 |
| **文档层** | 数据来源文档 | 年报、公告等 |
| **字段层** | 数据来源字段 | 数值、文本 |

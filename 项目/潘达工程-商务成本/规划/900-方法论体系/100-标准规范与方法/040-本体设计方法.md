## 二、本体设计概述

**具体目标**：
1. 物理表设计（fact_*/dim_*表结构、表间关系）
2. Cube层设计（维度、度量、派生度量、平台分类）

## 三、物理表设计

### 3.1 物理表设计规则

**规则**：
- 表前缀：fact_（事实表）、dim_（维度表）、bridge_（桥接表）
- 命名格式：snake_case
- 时间键：date_key（Int32类型）
- 主键命名：{entity}_key或{entity}_id
- 外键命名：与关联表主键同名

**示例**：
```yaml
物理表设计示例:
  - table_name: fact_project_cost
    display_name: 项目成本事实表
    table_type: fact
    primary_key: cost_id
    foreign_keys:
      - name: date_key
        reference: dim_date.date_key
      - name: project_key
        reference: dim_project.project_key
      - name: cost_subject_key
        reference: dim_cost_subject.cost_subject_key
    fields:
      - name: cost_id
        data_type: string
        description: 成本记录唯一标识
        is_primary_key: true
      - name: date_key
        data_type: int32
        description: 日期键
        is_sort_key: true
      - name: project_key
        data_type: string
        description: 项目唯一标识
      - name: cost_subject_key
        data_type: string
        description: 成本科目标识
      - name: cost_amount
        data_type: decimal(18,2)
        description: 成本金额
        value_range: [0, ∞)

  - table_name: dim_cost_subject
    display_name: 成本科目维度表
    table_type: dim
    primary_key: cost_subject_key
    fields:
      - name: cost_subject_key
        data_type: string
        description: 成本科目唯一标识
        is_primary_key: true
      - name: cost_subject_code
        data_type: string
        description: 成本科目编码
      - name: cost_subject_name
        data_type: string
        description: 成本科目名称
      - name: parent_key
        data_type: string
        description: 父级科目标识
        is_foreign_key: true
        reference: dim_cost_subject.cost_subject_key
```

### 3.2 物理表分类

| 分类 | 说明 | 示例 |
|------|------|------|
| **事实表** | 存储业务事实数据 | `fact_project_cost`、`fact_project_output` |
| **维度表** | 存储业务维度数据 | `dim_cost_subject`、`dim_project_status` |
| **桥接表** | 处理多对多关系 | `bridge_project_supplier` |

### 3.3 表间关系定义

**关系类型**：
| 关系类型 | 说明 | 示例 |
|---------|------|------|
| **时间关联** | 事实表关联时间维度 | `fact_project_cost.date_key → dim_date.date_key` |
| **主数据关联** | 事实表关联主数据 | `fact_project_cost.project_key → dim_project.project_key` |
| **维度关联** | 事实表关联维度表 | `fact_project_cost.cost_subject_key → dim_cost_subject.cost_subject_key` |
| **层级自关联** | 维度表层级关系 | `dim_cost_subject.parent_key → dim_cost_subject.cost_subject_key` |

## 四、Cube层设计

### 4.1 Cube设计规则

**规则**：
- 命名格式：PascalCase + Cube后缀
- 平台分类：流程型、主体型、对比型、快照型
- 维度定义：时间维（date_key、year、quarter、month、year_month）+ 业务维
- 度量定义：基础度量 + 派生度量
- 事实源：关联物理事实表

**示例**：
```yaml
Cube设计示例:
  - cube_name: ProjectCostCube
    display_name: 项目成本Cube
    platform_category: 流程型
    fact_source: fact_project_cost
    dimensions:
      - name: date_key
        data_type: int32
        description: 日期键
      - name: year
        data_type: int32
        description: 年份
      - name: quarter
        data_type: int32
        description: 季度
      - name: month
        data_type: int32
        description: 月份
      - name: year_month
        data_type: string
        description: 年月
      - name: project_key
        data_type: string
        description: 项目标识
      - name: cost_subject_key
        data_type: string
        description: 成本科目标识
    measures:
      - name: cost_amount
        data_type: decimal(18,2)
        aggregation: sum
        description: 成本金额
      - name: cost_count
        data_type: int32
        aggregation: count
        description: 成本记录数
    derived_metrics:
      - name: cost_amount_avg
        formula: cost_amount / cost_count
        data_type: decimal(18,2)
        description: 平均成本金额
      - name: cost_ratio_by_subject
        formula: cost_amount / total_cost_by_project
        data_type: decimal(5,2)
        description: 成本科目占比
```

### 4.2 Cube平台分类

| 分类 | 说明 | 示例 |
|------|------|------|
| **流程型** | 分析业务流程 | `ProjectCostCube`、`PaymentCube` |
| **主体型** | 分析主体指标 | `ProjectIndicatorCube` |
| **对比型** | 对比分析 | `BudgetVsActualCube` |
| **快照型** | 快照分析 | `ProjectSnapshotCube` |

### 4.3 Cube维度定义

**维度类型**：
| 维度类型 | 说明 | 示例 |
|---------|------|------|
| **时间维** | 日期、年、季、月、年月 | `date_key`、`year`、`quarter`、`month`、`year_month` |
| **业务维** | 项目、合同、科目等 | `project_key`、`contract_key`、`cost_subject_key` |
| **组织维** | 部门、组织 | `organization_key` |
| **地理维** | 地区、区域 | `region_key` |

### 4.4 Cube度量定义

**度量类型**：
| 度量类型 | 说明 | 示例 |
|---------|------|------|
| **基础度量** | 直接从事实表获取 | `cost_amount`、`cost_count` |
| **派生度量** | 通过计算得出 | `cost_amount_avg`、`cost_ratio_by_subject` |

## 五、bind_source映射设计

### 5.1 bind_source设计规则

**规则**：
- 明确Object Type到Cube的绑定关系
- 区分直接绑定和间接绑定
- 说明属性映射关系

**示例**：
```yaml
bind_source映射示例:
  - object_type: ProjectCost
    cube: ProjectCostCube
    binding_method: 直接绑定
    description: 属性直接映射到Cube维度和度量
    attribute_mapping:
      - object_attribute: cost_amount
        cube_measure: cost_amount
      - object_attribute: date_key
        cube_dimension: date_key
      - object_attribute: project_key
        cube_dimension: project_key

  - object_type: Project
    cube: ProjectIndicatorCube
    binding_method: 间接绑定
    description: 通过project_key关联，聚合维度和度量
    attribute_mapping:
      - object_attribute: project_key
        cube_dimension: project_key
      - object_attribute: total_output
        cube_measure: total_output
      - object_attribute: total_cost
        cube_measure: total_cost
```

### 5.2 绑定方式分类

| 绑定方式 | 说明 | 示例 |
|---------|------|------|
| **直接绑定** | Object Type属性直接映射到Cube | `ProjectCost → ProjectCostCube` |
| **间接绑定** | 通过主键关联，聚合维度和度量 | `Project → ProjectIndicatorCube` |

## 六、Object Type 设计

### 6.1 Object Type 设计规则

**规则**：
- 命名格式：PascalCase，无前缀
- 长度限制：不超过50个字符
- 命名原则：反映业务实体的本质
- 属性定义：包含核心属性和派生属性

**示例**：
```yaml
Object Type 定义示例:
  - code: Project
    display_name: 项目
    description: 建筑工程项目
    attributes:
      - name: project_id
        data_type: string
        description: 项目唯一标识
        is_primary_key: true
      - name: project_name
        data_type: string
        description: 项目名称
      - name: project_status
        data_type: string
        description: 项目状态
        value_range: [在建, 竣工, 结算, 归档]
      - name: start_date
        data_type: date
        description: 项目开始日期
      - name: end_date
        data_type: date
        description: 项目结束日期
    classification: Master
    source_type: 行业通用知识

  - code: ProjectCost
    display_name: 项目成本
    description: 项目成本数据
    attributes:
      - name: project_cost_id
        data_type: string
        description: 成本记录唯一标识
        is_primary_key: true
      - name: project_id
        data_type: string
        description: 项目唯一标识
        is_foreign_key: true
        reference: Project.project_id
      - name: cost_subject_key
        data_type: string
        description: 成本科目维度表主键
        is_foreign_key: true
        reference: dim_cost_subject.cost_subject_key
      - name: cost_amount
        data_type: decimal(18,2)
        description: 成本金额
        value_range: [0, ∞)
      - name: cost_level
        data_type: int
        description: 成本层级（1/2/3）
        value_range: [1, 2, 3]
    classification: Transaction
    source_type: 混合（行业通识+企业专识）
```

### 6.2 Object Type 分类

| 分类 | 说明 | 示例 |
|------|------|------|
| **Master** | 主数据，相对稳定 | `Project`、`Contract`、`Supplier` |
| **Transaction** | 交易数据，频繁变化 | `ProjectCost`、`ProjectOutput`、`Payment` |
| **Analytical** | 分析实体，用于分析 | `ProjectIndicator`、`RiskAlert` |

### 6.3 Object Type 属性定义

**属性类型**：
| 属性类型 | 说明 | 示例 |
|---------|------|------|
| **核心属性** | 业务实体的核心属性 | `project_id`、`project_name` |
| **外键属性** | 关联其他实体的属性 | `project_id`（关联 Project） |
| **派生属性** | 通过计算得出的属性 | `profit_rate`（利润率） |
| **维度属性** | 关联维度表的属性 | `cost_subject_key`（关联 dim_cost_subject） |

**属性定义要求**：
- 必须定义数据类型
- 必须定义业务含义
- 必须定义取值范围（如适用）
- 必须定义外键关系（如适用）
- 必须标记来源类型

## 七、Link Type 设计

### 7.1 Link Type 设计规则

**规则**：
- 命名格式：PascalCase，无前缀
- 长度限制：不超过50个字符
- 命名原则：反映关系的语义
- 基数定义：明确关系的基数（1:1、1:N、N:M）

**示例**：
```yaml
Link Type 定义示例:
  - code: belongsTo
    display_name: 归属于
    description: 实体归属于另一个实体
    source_type: 行业通用知识
    relationships:
      - source: Contract
        target: Project
        cardinality: 1:N
        description: 合同归属于项目
      - source: ProjectCost
        target: Project
        cardinality: 1:N
        description: 成本归属于项目

  - code: contains
    display_name: 包含
    description: 实体包含另一个实体
    source_type: 行业通用知识
    relationships:
      - source: Project
        target: Contract
        cardinality: 1:N
        description: 项目包含合同
      - source: ProjectCost
        target: CostItem
        cardinality: 1:N
        description: 成本包含费用项

  - code: calculates
    display_name: 计算
    description: 实体通过计算得出
    source_type: 行业通用知识
    relationships:
      - source: Profit
        target: ProjectOutput
        target: ProjectCost
        cardinality: 1:1:1
        description: 利润通过产值和成本计算
        calculation_rule: profit = output - cost
```

### 7.2 Link Type 分类

| 分类 | 说明 | 示例 |
|------|------|------|
| **包含关系** | 整体与部分 | `Project` 包含 `Contract` |
| **归属关系** | 从属关系 | `Contract` 归属于 `Project` |
| **关联关系** | 业务关联 | `Supplier` 提供 `Material` |
| **计算关系** | 派生计算 | `Profit` 通过 `Output` 和 `Cost` 计算 |
| **影响关系** | 因果关系 | `Change` 影响 `Cost` |

### 7.3 Link Type 基数定义

**基数类型**：
| 基数类型 | 说明 | 示例 |
|---------|------|------|
| **1:1** | 一对一关系 | `Project` 与 `ProjectStatus` |
| **1:N** | 一对多关系 | `Project` 与 `Contract` |
| **N:M** | 多对多关系 | `Project` 与 `Supplier` |

## 八、Action Type 设计

### 8.1 Action Type 设计规则

**规则**：
- 命名格式：PascalCase，无前缀
- 长度限制：不超过50个字符
- 命名原则：反映业务动作的语义
- 触发条件：明确动作的触发条件
- 执行主体：明确动作的执行主体

**示例**：
```yaml
Action Type 定义示例:
  - code: ApproveBudget
    display_name: 批准预算
    description: 批准项目预算
    source_type: 企业业务素材
    trigger_condition: 预算申请提交
    executor: 项目经理
    input:
      - project_id
      - budget_amount
      - budget_period
    output:
      - approval_status
      - approval_date
      - approver
    business_rules:
      - rule_id: BR001
        description: 预算金额不能超过目标成本的110%
        constraint: budget_amount ≤ target_cost * 1.1

  - code: ConfirmOutput
    display_name: 确认产值
    description: 确认项目产值
    source_type: 企业业务素材
    trigger_condition: 产值确认申请提交
    executor: 商务经理
    input:
      - project_id
      - output_amount
      - output_type
      - output_period
    output:
      - confirmation_status
      - confirmation_date
      - confirmer
    business_rules:
      - rule_id: BR002
        description: 产值确认需要业主签字
        constraint: confirmation_status = 'confirmed' → owner_signature_required = true
```

### 8.2 Action Type 分类

| 分类 | 说明 | 示例 |
|------|------|------|
| **审批动作** | 审批流程中的动作 | `ApproveBudget`、`ApproveContract` |
| **确认动作** | 确认业务数据 | `ConfirmOutput`、`ConfirmCost` |
| **计算动作** | 计算派生数据 | `CalculateProfit`、`CalculateIndicator` |
| **预警动作** | 触发预警 | `TriggerRiskAlert`、`TriggerBudgetAlert` |

## 九、业务规则定义

> **核心要求**：本体设计必须包含详细的业务规则定义，包括计算规则、勾稽关系、约束条件、数据标准，为后续模拟数据提供基础。

### 9.1 计算规则定义

**定义要求**：
- 明确计算公式
- 明确计算顺序
- 明确数据来源
- 明确计算结果的数据类型

**示例**：
```yaml
计算规则定义示例:
  - rule_id: CR001
    name: 利润率计算
    formula: profit_rate = (total_output - total_cost) / total_output
    description: 项目利润率计算公式
    data_sources:
      - total_output: fact_project_output.output_amount
      - total_cost: fact_project_cost.cost_amount
    result_data_type: decimal(5,2)
    value_range: (-∞, 1]
    source_type: 行业通用知识

  - rule_id: CR002
    name: 成本偏差率计算
    formula: cost_variance_rate = (actual_cost - target_cost) / target_cost
    description: 成本偏差率计算公式
    data_sources:
      - actual_cost: fact_project_cost.cost_amount
      - target_cost: dim_project.target_cost
    result_data_type: decimal(5,2)
    value_range: (-∞, ∞)
    source_type: 行业通用知识

  - rule_id: CR003
    name: 产值合计计算
    formula: total_output = confirmed_output + unconfirmed_output
    description: 产值合计计算公式
    data_sources:
      - confirmed_output: fact_project_output.confirmed_amount
      - unconfirmed_output: fact_project_output.unconfirmed_amount
    result_data_type: decimal(18,2)
```

**示例**：

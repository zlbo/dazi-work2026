## 一、规划概述

### 1.1 目标定位

本规划旨在建立一套**面向年产值500-600亿规模央国企工程类公司**的商务成本管理量化规则体系，实现以下目标：

| 目标维度 | 具体目标 |
|---------|---------|
| **数据模拟** | 提供数据生成的约束条件和取值范围，确保模拟数据符合业务常识 |
| **数据质量** | 作为数据质量校验规则，保障业务数据的准确性和一致性 |
| **智能决策** | 为智能决策引擎提供核心规则支撑 |
| **知识沉淀** | 将业务经验转化为可执行的量化规则 |

### 2.2 规则编码规范

```
规则编号 = 规则类别前缀 + 3位数字序号

示例：
- QR001：业务运算规则第1条
- DR001：数据取值规则第1条
- CR001：数据勾稽规则第1条
- FR001：数据格式规则第1条
- IR001：数据完整性规则第1条
- BR001：业务逻辑规则第1条
```

### 2.3 规则描述模板

```markdown
【规则编号】QR001
【规则名称】成本偏差率计算规则
【规则类别】业务运算规则 → 成本类规则
【适用实体】ProjectCost, Budget
【适用场景】成本分析、成本预警
【量化公式】成本偏差率 = (实际成本 - 预算成本) / 预算成本 × 100%
【参数定义】
  - 实际成本：来自成本台账 fact_project_cost
  - 预算成本：来自预算表 fact_budget
【阈值定义】成本偏差率 > 5% 时触发预警
【预警级别】黄色预警
【业务依据】行业成本控制标准
【企业规模适配】年产值500-600亿企业适用
【数据来源】fact_project_cost, fact_budget
【平台映射】派生属性（成本偏差率）+ Action Type（预警通知）
```

## 二、编码分类与优先级

> **引用**：《000-顶层设计.md》，编码标准规范（核心定义）。

### 2.1 编码分类

| 编码类型 | 说明 | 示例 | 优先级 |
|---------|------|------|-------|
| **结构编码** | Object Type、Link Type、维度表结构等框架性编码 | `Project`、`ProjectCost`、`dim_project` | 通识优先 |
| **业务编码** | 科目编码、指标编码等业务分类编码 | 成本科目：`01-人工成本`、`02-材料成本` | 通识优先，参考专识 |
| **属性值** | 具体的科目名称、指标名称等业务术语 | `钢筋`、`商品混凝土` | 专识优先 |

### 2.2 优先级规则

| 编码类型 | 优先级 | 说明 |
|---------|-------|------|
| **结构编码** | 通识优先 | 采用行业标准编码，确保行业通用性 |
| **业务编码** | 通识优先，参考专识 | 采用行业标准编码，企业编码仅作参考 |
| **属性值** | 专识优先 | 采用企业实际业务术语，确保业务准确性 |

## 三、结构编码规则

### 3.1 Object Type 编码规则

**规则**：
- 命名格式：PascalCase，无前缀
- 长度限制：不超过50个字符
- 命名原则：反映业务实体的本质

**示例**：
```yaml
Object Type 编码示例:
  - Project: 项目
  - ProjectCost: 项目成本
  - ProjectOutput: 项目产值
  - Contract: 合同
  - Supplier: 供应商
  - Risk: 风险
  - Indicator: 指标
```

### 3.2 Link Type 编码规则

**规则**：
- 命名格式：PascalCase，无前缀
- 长度限制：不超过50个字符
- 命名原则：反映关系的语义

**示例**：
```yaml
Link Type 编码示例:
  - belongsTo: 归属于
  - contains: 包含
  - relatesTo: 关联
  - calculates: 计算
  - affects: 影响
```

### 3.3 物理表编码规则

**规则**：
- 事实表：`fact_*` 前缀 + 业务名称（snake_case）
- 维度表：`dim_*` 前缀 + 业务名称（snake_case）
- 长度限制：不超过50个字符（含前缀）

**示例**：
```yaml
物理表编码示例:
  - fact_project_cost: 项目成本事实表
  - fact_project_output: 项目产值事实表
  - fact_project_balance: 项目收支事实表
  - dim_project: 项目维度表
  - dim_cost_subject: 成本科目维度表
  - dim_subject: 收支科目维度表
```

### 3.4 Cube 编码规则

**规则**：
- 命名格式：PascalCase + `Cube` 后缀
- 长度限制：不超过50个字符
- 命名原则：反映分析主题

**示例**：
```yaml
Cube 编码示例:
  - ProjectCostCube: 项目成本分析Cube
  - ProjectOutputCube: 项目产值分析Cube
  - ProjectBalanceCube: 项目收支分析Cube
  - ProjectRiskCube: 项目风险分析Cube
```

## 四、业务编码规则

### 4.1 层级编码规则

**规则**：
- 采用数字层级结构（如 L1-L2-L3）
- 编码格式：`[层级编码]-[业务含义]`
- 编码长度：统一为2位或4位数字

**示例**：
```yaml
成本科目编码（通识优先）:
  L1: 01-人工成本, 02-材料成本, 03-机械成本, 04-管理费用
  L2: 0101-基本工资, 0102-绩效工资, 0201-主材, 0202-辅材
  L3: 020101-钢筋, 020102-混凝土, 020103-水泥

收支科目编码（通识优先）:
  L1: 01-收入, 02-支出
  L2: 0101-工程款收入, 0102-其他收入, 0201-材料费, 0202-人工费
```

### 4.2 指标编码规则

**规则**：
- 采用字母+数字组合
- 编码格式：`[指标类别缩写]-[序号]`
- 编码长度：不超过10个字符

**示例**：
```yaml
指标编码示例:
  - PRO-001: 项目毛利率（Profit Rate）
  - COS-001: 成本偏差率（Cost Variance Rate）
  - OUT-001: 产值完成率（Output Completion Rate）
  - BAL-001: 资金结余率（Balance Rate）
```

### 4.3 风险类型编码规则

**规则**：
- 采用字母+数字组合
- 编码格式：`[风险类别缩写]-[序号]`
- 编码长度：不超过10个字符

**示例**：
```yaml
风险类型编码示例:
  - FIN-001: 资金风险（Financial Risk）
  - COS-001: 成本风险（Cost Risk）
  - SAF-001: 安全风险（Safety Risk）
  - LEG-001: 法律风险（Legal Risk）
```

## 五、属性值规则

### 5.1 属性值优先级

**规则**：
- 属性值采用企业实际业务术语
- 企业术语不清晰时，参考行业标准术语

**示例**：
```yaml
成本科目属性值（专识优先）:
  020101 → "钢筋"（企业术语）
  020102 → "商品混凝土"（企业术语，非通用术语"商砼"）
  020103 → "水泥"（企业术语）

收支科目属性值（专识优先）:
  0101 → "工程款收入"（企业术语）
  0102 → "其他收入"（企业术语）
  0201 → "材料费"（企业术语）
  0202 → "人工费"（企业术语）
```

### 5.2 属性值冲突解决

**规则**：
- 企业术语优先
- 企业术语不清晰时，参考行业标准术语

**示例**：
```yaml
属性值冲突解决示例:
  场景: 企业术语"商砼"与行业标准术语"商品混凝土"冲突
  解决: 采用企业术语"商砼"，但增加备注说明对应行业标准术语"商品混凝土"

  场景: 企业术语"人工费"与行业标准术语"人工成本"冲突
  解决: 采用企业术语"人工费"，但增加备注说明对应行业标准术语"人工成本"
```

## 六、编码冲突解决策略

### 6.1 冲突类型

| 冲突类型 | 说明 | 示例 |
|---------|------|------|
| **结构编码冲突** | 企业编码与行业标准编码冲突 | 企业用 `ProjectCost`，行业标准用 `Project_Cost` |
| **业务编码冲突** | 企业编码与行业标准编码冲突 | 企业用 `A01`，行业标准用 `01` |
| **属性值冲突** | 企业术语与行业标准术语冲突 | 企业用"商砼"，行业标准用"商品混凝土" |

### 6.2 解决策略

| 冲突类型 | 解决策略 | 说明 |
|---------|---------|------|
| **结构编码冲突** | 采用行业标准编码 | 确保行业通用性 |
| **业务编码冲突** | 采用行业标准编码，企业编码仅作参考 | 确保行业通用性 |
| **属性值冲突** | 采用企业实际业务术语 | 确保业务准确性 |

## 八、编码管理

### 8.1 编码文档化

**要求**：
- 所有编码必须文档化
- 编码文档必须包含编码规则、示例、说明
- 编码文档必须保持更新

**示例**：
```yaml
成本科目编码文档:
  - 编码类型: 业务编码
  - 编码规则: L1(2位) + L2(2位) + L3(3位)
  - 编码示例:
    - L1: 01-人工成本, 02-材料成本, 03-机械成本, 04-管理费用
    - L2: 0101-基本工资, 0102-绩效工资, 0201-主材, 0202-辅材
    - L3: 020101-钢筋, 020102-混凝土, 020103-水泥
  - 编码说明: 采用建筑工程行业成本科目编码标准
  - 属性值: 采用企业实际业务术语
  - 冲突解决: 企业编码与行业标准编码冲突时，采用行业标准编码
```

### 8.2 编码变更管理

**要求**：
- 编码变更必须遵循变更流程
- 编码变更必须更新所有相关文档

## 九、总结

### 9.1 核心原则

| 原则 | 说明 |
|------|------|
| **结构编码通识优先** | 结构编码采用行业标准编码 |
| **业务编码通识优先，参考专识** | 业务编码采用行业标准编码，企业编码仅作参考 |
| **属性值专识优先** | 属性值采用企业实际业务术语 |
| **编码冲突解决** | 明确的编码冲突解决策略 |
| **编码文档化** | 所有编码必须文档化 |

### 9.2 关键产出

| 产出 | 说明 |
|------|------|
| **编码规则** | 结构编码、业务编码、属性值的编码规则 |
| **编码示例** | 各类编码的示例 |
| **冲突解决策略** | 编码冲突的解决策略 |
| **编码实施方法** | 编码设计的流程和验证清单 |

### 9.3 下一步

编码标准规范完成后，进入本体设计阶段，将编码规则应用到 Object Type、Link Type 和维度表的设计中。

## 二、关键问题与解决方案

### 2.1 数据源连接问题（已解决）

**问题描述**：
- 文档说明使用 `connectionId`（字符串），但测试时遇到「数据连接不存在」错误
- 曾尝试使用 `dataSourceId`（数字ID），仍失败

**解决方案**：
- **正确配置**：使用 `connectionId`（字符串类型）
- **正确值**：`"duckdb__建工成本01"`（必须与数据源文档一致）

**配置示例**：
```json
{
"type": "database-source",
"connectionId": "duckdb__建工成本01",
"queryConfig": {
    "tableName": "tb_project_income_outcome_summary",
    "selectedFields": [...]
}
}
```

### 2.2 数据类型转换问题（已解决）

**问题描述**：
- ClickHouse 写入失败：`object of type 'numpy.float64' has no len()`

**解决方案**：
- 使用 `tolist()` 将 numpy 类型转换为 Python 原生类型
- 在数据转换节点和质量检查节点都需要处理

**代码示例**：
```python
result_df['amount'] = df['amount'].fillna(0.0).tolist()
```

### 2.3 字段映射问题（已解决）

**问题描述**：
- 写入时出现 `Unrecognized column 'year' in table fact_project_output`

**解决方案**：
- 输出字段必须与目标表完全一致
- 只输出目标表中存在的字段

### 2.4 数据落地问题（已解决）

**问题描述**：
- 流程运行成功但数据未写入本体模型

**解决方案**：
- 必须添加 `dataspace-sink` 节点
- 配置正确的 `spaceId` 和 `tableName`

**配置示例**：
```json
{
"type": "dataspace-sink",
"spaceId": "space__panda_construction",
"tableName": "fact_project_output",
"writeMode": "append",
"input_variable_name": "validated_output_data"
}
```

## 五、已验证的流程结构

```
开始 → database-source → python-script(转换) → python-script(质量检查) → dataspace-sink → 结束
       读取数据源          字段映射转换            数据质量验证              写入本体模型
```

### 5.1 节点配置要点

**database-source 节点**：
- `connectionId`: `"duckdb__建工成本01"`
- `output_variable_name`: `"output_raw_data"`
- `queryConfig.tableName`: 源表名

**python-script 转换节点**：
- 输入变量：`get_variable("output_raw_data")`
- 输出变量：`result_df`（必须）
- 字段映射、类型转换、缺失值处理

**python-script 质量检查节点**：
- 输入变量：`get_variable("project_output_data")`
- 输出变量：`result_df`（必须）
- 必填字段检查、格式验证

**dataspace-sink 节点**：
- `spaceId`: `"space__panda_construction"`
- `tableName`: 目标表名（如 `"fact_project_output"`）
- `writeMode`: `"append"`
- `input_variable_name`: `"validated_output_data"`

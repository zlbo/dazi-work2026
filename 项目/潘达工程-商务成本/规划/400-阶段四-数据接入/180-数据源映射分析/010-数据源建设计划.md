## 一、计划概述

### 1.1 背景
基于《数据源映射分析报告》，制定本数据源建设计划，明确各项任务的执行顺序、技术路径和资源需求。

### 1.2 目标
完成本体建模表的数据源建设，为本体设计和数据对接提供实际的数据支撑。

### 1.3 范围
- P0核心表：fact_project_output, fact_project_cost, fact_project_indicator, dim_project
- P1扩展表：fact_project_risk, fact_project_payment, fact_project_balance

## 三、技术路径

### 3.1 数据源连接
- **数据源ID**：duckdb__建工成本01
- **连接方式**：平台内置DuckDB连接器

### 3.2 Flow流程设计

#### 流程1：Excel产值导入流程

```
Start → Excel读取节点 → 数据转换节点 → 数据写入节点 → End
```

#### 流程2：成本数据合并流程

```
Start → SQL查询节点（tb_project_major_cost）
       → SQL查询节点（tb_project_cost_pay_detail）
       → SQL查询节点（tb_project_cost_unconfirmed_analyse）
       → Python合并节点 → 数据写入节点 → End
```

### 3.3 数据质量检查规则

| 规则ID | 检查内容 | 检查方式 |
|--------|---------|---------|
| DQ001 | project_id非空 | NOT NULL约束 |
| DQ002 | report_period格式正确 | 正则表达式校验 |
| DQ003 | 数值字段非负 | 范围检查 |
| DQ004 | 关联字段存在 | 外键约束检查 |

## 六、风险与缓解措施

### 6.1 风险清单

| 风险 | 可能性 | 影响 | 缓解措施 |
|------|--------|------|---------|
| Excel格式变化 | 中 | 数据提取失败 | 建立Excel模板版本控制 |
| 数据质量问题 | 高 | 影响业务分析 | 建立数据质量检查机制 |
| 业务人员配合不足 | 中 | 进度延迟 | 提前沟通，明确需求 |
| 技术难题 | 低 | 进度延迟 | 预留技术攻关时间 |

### 6.2 应急预案

| 场景 | 预案 |
|------|------|
| Excel数据提取失败 | 使用备用数据提取工具或手动处理 |
| 数据质量问题严重 | 先使用部分可用数据，后续逐步完善 |
| 进度延迟 | 调整计划优先级，优先保证P0核心表 |

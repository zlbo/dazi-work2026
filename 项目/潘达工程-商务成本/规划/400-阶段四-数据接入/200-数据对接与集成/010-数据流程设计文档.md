## 四、详细流程设计

### 4.1 Flow-001：产值数据同步

**目的**：将tb_project_output数据同步到ProjectOutput Cube

```
┌─────────┐      ┌─────────────┐      ┌─────────────┐      ┌─────────┐
│  开始   │─────→│ SQL-查询产值│─────→│ 数据转换    │─────→│ 结束    │
└─────────┘      └─────────────┘      └─────────────┘      └─────────┘
```

| 节点ID | 节点名称 | 类型 | 说明 |
|--------|---------|------|------|
| start | 开始 | start | 入口 |
| sql-output | SQL-查询产值 | sql-query | 查询tb_project_output |
| py-transform | 数据转换 | python-script | 数据格式转换 |
| end | 结束 | end | 出口 |

**SQL查询**：
```sql
SELECT 
    project_id,
    report_period,
    output_confirmed,
    output_unconfirmed,
    output_confirmed + output_unconfirmed AS output_total,
    output_last_year_confirmed,
    output_last_year_unconfirmed
FROM tb_project_output
WHERE report_period = '{{report_period}}';
```

### 4.5 Flow-005：指标计算

**目的**：基于同步数据计算派生指标（L2-L4指标）

```
┌─────────┐      ┌─────────────┐      ┌─────────────┐      ┌─────────────┐      ┌─────────┐
│  开始   │─────→│ 获取产值数据│─────→│ 获取成本数据│─────→│ 计算派生指标│─────→│ 结束    │
└─────────┘      └─────────────┘      └─────────────┘      └─────────────┘      └─────────┘
                                                      ↓
                                              ┌─────────────┐
                                              │ 获取付款数据 │
                                              └─────────────┘
```

| 节点ID | 节点名称 | 类型 | 说明 |
|--------|---------|------|------|
| start | 开始 | start | 入口 |
| get-output | 获取产值数据 | variable | 从Flow-001获取 |
| get-cost | 获取成本数据 | variable | 从Flow-002获取 |
| get-payment | 获取付款数据 | variable | 从Flow-003获取 |
| calc-indicator | 计算派生指标 | python-script | 计算L2-L4指标 |
| end | 结束 | end | 出口 |

**派生指标计算**：

| 指标名称 | 公式 | 指标层级 | 说明 |
|---------|------|---------|------|
| 利润率 | (产值 - 成本) / 产值 | L2 | 基础效益指标 |
| 成本率 | 成本 / 产值 | L2 | 成本控制指标 |
| 回款率 | 已收款 / 应收款 | L2 | 资金回收指标 |
| 毛利率 | (产值 - 完全成本) / 产值 | L3 | 综合效益指标 |
| 成本偏差率 | (实际成本 - 预算成本) / 预算成本 | L3 | 成本偏差指标 |
| 项目健康度 | 加权综合评分 | L4 | 决策支持指标 |

### 4.6 Flow-006：风险检测

**目的**：基于同步数据检测异常风险

```
┌─────────┐      ┌─────────────┐      ┌─────────────┐      ┌─────────────┐      ┌─────────┐
│  开始   │─────→│ 获取指标数据│─────→│ 风险规则匹配│─────→│ 风险预警生成│─────→│ 结束    │
└─────────┘      └─────────────┘      └─────────────┘      └─────────────┘      └─────────┘
```

| 节点ID | 节点名称 | 类型 | 说明 |
|--------|---------|------|------|
| start | 开始 | start | 入口 |
| get-indicator | 获取指标数据 | variable | 从Flow-005获取 |
| rule-match | 风险规则匹配 | python-script | 匹配风险规则 |
| risk-alert | 风险预警生成 | python-script | 生成预警 |
| end | 结束 | end | 出口 |

**风险规则**（来自阶段一T003、T004）：

| 规则ID | 规则名称 | 触发条件 | 风险等级 |
|--------|---------|---------|---------|
| R001 | 产值确认超时 | 待确认产值 > 已确认产值 × 50% | 中 |
| R002 | 成本异常偏高 | 成本率 > 行业均值 × 120% | 高 |
| R003 | 回款率偏低 | 回款率 < 70% | 高 |
| R004 | 成本确认滞后 | 待确认成本 > 已确认成本 × 30% | 中 |

## 七、开发循环

```
1. 新建或拉取流程
   flow project pull --flow <id> --dir "<流程目录>"

2. 改代码（节点/code.sql 或 code.py）
   - 编辑代码正文，不要写进flow.json

3. 改画布配置（flow.json，连线、connectionId）
   - 必须遵守画布节点规范（200px、锚点l/t/r/b/true/false）

4. 提交节点代码
   flow node push --node <uuid> --dir "<流程目录>"

5. 单节点测试
   flow run node-exec --node <uuid> --dir "<流程目录>"

6. 查看变量
   flow variable pull --name <变量名> --dir "<流程目录>"

7. 提交画布
   flow project push --dir "<流程目录>" --canvas

8. 失败时查看错误
   _run/*.last-error.md
```

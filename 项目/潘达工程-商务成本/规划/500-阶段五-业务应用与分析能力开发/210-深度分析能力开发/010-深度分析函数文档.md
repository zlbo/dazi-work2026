## 一、函数概述

### 1.1 设计目标

基于基础查询能力，开发深度分析函数，支持复杂业务分析和决策场景，实现根因分析、预测推演、决策建议等高级功能。

### 1.2 函数分类

| 分类 | 说明 | 示例 |
|------|------|------|
| **根因分析函数** | 分析指标异常的根本原因 | analyzeRootCause |
| **趋势预测函数** | 预测未来指标走势 | predictTrend |
| **异常检测函数** | 检测异常数据和趋势 | detectAnomaly |
| **决策建议函数** | 基于分析结果提供建议 | generateSuggestion |

## 二、根因分析函数

### 2.1 analyzeRootCause

**功能**：分析指标异常的根本原因

**参数**：

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| projectId | STRING | 是 | 项目ID |
| indicatorCode | STRING | 是 | 异常指标编码 |
| reportPeriod | STRING | 否 | 报告期间 |
| threshold | DECIMAL | 否 | 异常阈值（默认10%） |

**返回值**：

| 字段 | 类型 | 说明 |
|------|------|------|
| rootCause | STRING | 根因描述 |
| confidence | DECIMAL | 置信度(%) |
| contributingFactors | LIST | 影响因素列表 |
| impactAnalysis | STRING | 影响分析 |
| suggestions | LIST | 改进建议列表 |

**示例调用**：
```
analyzeRootCause("P001", "profit_rate", "2024-01", 5)
```

**AI问答映射**：
- "为什么P001项目利润率下降了？"
- "利润率异常的根本原因是什么？"

---

## 三、趋势预测函数

### 3.1 predictTrend

**功能**：预测未来指标走势

**参数**：

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| projectId | STRING | 是 | 项目ID |
| indicatorCode | STRING | 是 | 指标编码 |
| periods | INTEGER | 否 | 预测期数（默认3） |
| method | STRING | 否 | 预测方法（ARIMA/线性回归/指数平滑） |

**返回值**：

| 字段 | 类型 | 说明 |
|------|------|------|
| predictions | LIST | 预测值列表（期间、预测值、置信区间） |
| trendDirection | STRING | 趋势方向（上升/下降/平稳） |
| confidence | DECIMAL | 预测置信度(%) |
| warningFlag | BOOLEAN | 是否预警 |
| warningMessage | STRING | 预警信息 |

**示例调用**：
```
predictTrend("P001", "output_total", 3, "ARIMA")
```

**AI问答映射**：
- "下个月产值会怎样变化？"
- "预测未来三个月的成本走势"

---

### 3.2 predictCashFlow

**参数**：

| 参数名 | 类型 | 必填 | 说明 |

## 四、异常检测函数

### 4.1 detectAnomaly

**功能**：检测异常数据

**参数**：

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| projectId | STRING | 是 | 项目ID |
| indicatorCode | STRING | 否 | 指标编码（默认全部） |
| reportPeriod | STRING | 否 | 报告期间 |
| sensitivity | DECIMAL | 否 | 检测灵敏度（0-1） |

**返回值**：

| 字段 | 类型 | 说明 |
|------|------|------|
| anomalies | LIST | 异常列表（指标、值、偏差率、严重程度） |
| totalAnomalies | INTEGER | 异常数量 |
| severityLevel | STRING | 总体严重程度 |
| summary | STRING | 异常摘要 |

**示例调用**：
```
detectAnomaly("P001", "profit_rate", "2024-01", 0.8)
```

**AI问答映射**：
- "检测P001项目的异常指标"
- "本月有哪些指标异常？"

## 五、决策建议函数

### 5.1 generateSuggestion

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|

| 字段 | 类型 | 说明 |

**示例调用**：
```
generateSuggestion("P001", "cost", "2024-01")
```

**AI问答映射**：
- "如何优化项目成本？"
- "给出利润率提升建议"

## 六、函数调用流程

### 6.1 深度分析流程

```
用户提问 → AI解析 → 函数调用 → 多维度分析 → 综合判断 → 结果返回 → 自然语言回答
```

### 6.2 函数组合调用

| 场景 | 函数组合 | 说明 |
|------|---------|------|
| 完整分析 | detectAnomaly → analyzeRootCause → generateSuggestion | 异常检测→根因分析→建议生成 |
| 预测分析 | predictTrend → detectRiskPattern → optimizePayment | 趋势预测→风险检测→优化建议 |

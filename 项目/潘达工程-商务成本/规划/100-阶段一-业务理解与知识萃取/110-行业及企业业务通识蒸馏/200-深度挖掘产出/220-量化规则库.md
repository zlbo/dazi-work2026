# 建筑工程行业商务成本管理量化规则库

## 一、成本类量化规则

【QR001】成本偏差率计算规则
- 对应隐性关系：IR003
- 量化可能性：可量化
- 量化公式：
  - 公式表达式：成本偏差率 = (实际成本 - 预算成本) / 预算成本 × 100%
  - 参数定义：实际成本来自成本台账，预算成本来自预算表
- 阈值定义：成本偏差率 > 5% 时触发预警
- 预警逻辑：当成本偏差率连续3个月超过5%时发送预警通知
- 数据来源：fact_project_cost、fact_budget
- 平台映射：派生属性（成本偏差率）+ Action Type（预警通知）

【QR002】成本节约率计算规则
- 对应隐性关系：IR001
- 量化可能性：可量化
- 量化公式：
  - 公式表达式：成本节约率 = (预算成本 - 实际成本) / 预算成本 × 100%
  - 参数定义：实际成本来自成本台账，预算成本来自预算表
- 阈值定义：成本节约率 > 5% 时视为优秀
- 预警逻辑：无（正向指标）
- 数据来源：fact_project_cost、fact_budget
- 平台映射：派生属性（成本节约率）

【QR003】材料成本占比计算规则
- 对应隐性关系：IR004
- 量化可能性：可量化
- 量化公式：
  - 公式表达式：材料成本占比 = 材料成本 / 总造价 × 100%
  - 参数定义：材料成本来自成本台账的材料费科目，总造价来自合同金额
- 阈值定义：材料成本占比 > 60% 时需关注
- 预警逻辑：材料成本占比过高可能存在风险
- 数据来源：fact_project_cost、dim_contract
- 平台映射：派生属性（材料成本占比）

## 二、资金类量化规则

【QR004】资金周转率计算规则
- 对应隐性关系：IR002
- 量化可能性：可量化
- 量化公式：
  - 公式表达式：资金周转率 = 主营业务收入 / 平均应收账款
  - 参数定义：主营业务收入来自产值确认表，平均应收账款来自财务报表
- 阈值定义：资金周转率 < 3次/年时触发预警
- 预警逻辑：资金周转率低于行业平均水平时发送预警
- 数据来源：fact_project_output、dim_finance
- 平台映射：派生属性（资金周转率）+ Rule（预警规则）

【QR005】收款率计算规则
- 对应隐性关系：IR002
- 量化可能性：可量化
- 量化公式：
  - 公式表达式：收款率 = 已收款金额 / 确认产值 × 100%
  - 参数定义：已收款金额来自收款台账，确认产值来自产值确认表
- 阈值定义：收款率 < 80% 时触发预警
- 预警逻辑：收款率低于80%时发送预警通知
- 数据来源：fact_payment、fact_project_output
- 平台映射：派生属性（收款率）+ Action Type（预警通知）

【QR006】付款比例计算规则
- 对应隐性关系：IR008
- 量化可能性：可量化
- 量化公式：
  - 公式表达式：付款比例 = 已付款金额 / 合同金额 × 100%
  - 参数定义：已付款金额来自付款台账，合同金额来自合同表
- 阈值定义：付款比例 > 合同约定比例时触发预警
- 预警逻辑：超合同约定付款需要特殊审批
- 数据来源：fact_payment、dim_contract
- 平台映射：派生属性（付款比例）+ Action Type（审批流程）

## 三、进度类量化规则

【QR007】进度偏差率计算规则
- 对应隐性关系：IR003
- 量化可能性：可量化
- 量化公式：
  - 公式表达式：进度偏差率 = (实际进度 - 计划进度) / 计划进度 × 100%
  - 参数定义：实际进度来自进度报表，计划进度来自进度计划
- 阈值定义：进度偏差率 < -10%（滞后10%）时触发预警
- 预警逻辑：进度滞后超过10%时发送预警通知
- 数据来源：fact_project_progress、dim_project_plan
- 平台映射：派生属性（进度偏差率）+ Action Type（预警通知）

【QR008】工期完成率计算规则
- 对应隐性关系：IR003
- 量化可能性：可量化
- 量化公式：
  - 公式表达式：工期完成率 = 已完成天数 / 总工期天数 × 100%
  - 参数定义：已完成天数根据开工日期和当前日期计算，总工期天数来自项目计划
- 阈值定义：工期完成率 < 进度完成率时触发预警
- 预警逻辑：工期进度落后于计划时发送预警
- 数据来源：dim_project、dim_project_plan
- 平台映射：派生属性（工期完成率）

## 四、风险类量化规则

【QR009】风险综合评分规则
- 对应隐性关系：IR024、IR025
- 量化可能性：可量化
- 量化公式：
  - 公式表达式：风险综合评分 = 发生概率 × 影响程度 × 可检测性
  - 参数定义：发生概率（0-1）、影响程度（1-5）、可检测性（1-3）
- 阈值定义：风险综合评分 > 8 时触发预警
- 预警逻辑：高风险项目需要重点关注
- 数据来源：dim_risk
- 平台映射：派生属性（风险综合评分）+ Action Type（风险预警）

【QR010】变更频率计算规则
- 对应隐性关系：IR001、IR009
- 量化可能性：可量化
- 量化公式：
  - 公式表达式：变更频率 = 变更次数 / 项目周期（月）
  - 参数定义：变更次数来自变更签证台账，项目周期根据开工日期和当前日期计算
- 阈值定义：变更频率 > 1次/月时触发预警
- 预警逻辑：变更频繁需要关注设计质量
- 数据来源：fact_change_order、dim_project
- 平台映射：派生属性（变更频率）+ Action Type（预警通知）

## 七、三算分析量化规则

【QR016】概算偏差率计算规则
- 对应隐性关系：IR003
- 量化可能性：可量化
- 量化公式：
  - 公式表达式：概算偏差率 = (预算金额 - 概算金额) / 概算金额 × 100%
  - 参数定义：概算金额来自可行性研究报告，预算金额来自预算表
- 阈值定义：概算偏差率 > 10%时触发预警
- 预警逻辑：预算超概算10%以上时发送预警通知
- 数据来源：fact_budget、dim_estimate
- 平台映射：派生属性（概算偏差率）+ Action Type（预警通知）

【QR017】预算偏差率计算规则
- 对应隐性关系：IR003
- 量化可能性：可量化
- 量化公式：
  - 公式表达式：预算偏差率 = (决算金额 - 预算金额) / 预算金额 × 100%
  - 参数定义：决算金额来自结算报告，预算金额来自预算表
- 阈值定义：预算偏差率 > 5%时触发预警
- 预警逻辑：决算超预算5%以上时发送预警通知
- 数据来源：fact_settlement、fact_budget
- 平台映射：派生属性（预算偏差率）+ Action Type（预警通知）

【QR018】三算对比差异计算规则
- 对应隐性关系：IR003
- 量化可能性：可量化
- 量化公式：
  - 公式表达式：三算对比差异 = (决算金额 - 概算金额) / 概算金额 × 100%
  - 参数定义：概算、预算、决算数据分别来自对应报表
- 阈值定义：三算对比差异 > 15%时触发预警
- 预警逻辑：三算总差异超过15%时发送预警通知
- 数据来源：dim_estimate、fact_budget、fact_settlement
- 平台映射：派生属性（三算对比差异）+ Action Type（预警通知）

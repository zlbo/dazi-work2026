## 二、报表整体结构

项目收支汇总表采用**树形层级结构**，主要分为资金余额和收款金额两大类别：

```
┌─────────────────────────────────────────────┐
│ 表头:项目名称、报告期间                      │
├─────────────────────────────────────────────┤
│ A列:代码(层级编码)                          │
│ B列:科目名称                                │
│ C列:项目金额                                │
│ D列:公司金额                                │
│ E列:合计(C+D)                              │
├─────────────────────────────────────────────┤
│ 第5行: 1 资金余额                           │
│ 第6-8行: 1.1-1.3 资金余额明细               │
│ 第9行: 2 收款金额                           │
│ 第10-305行: 2.1-2.xx 收款金额明细           │
└─────────────────────────────────────────────┘
```

## 七、业务逻辑分析

### 7.1 资金余额逻辑

```

    + 项目不含资金拆借余额(1.3)

    + 工程款(甲指分包)(2.4)

    + 公司借支余额(2.10)
    + 自筹往来余额(2.11)
    + 保证金(2.12)
    + ...
    = 收款金额(2)
```

## 九、报表特点总结

### 9.1 优势特点

1. **结构清晰** - 树形层级结构，易于理解
2. **数据完整** - 306行，涵盖所有收支科目
3. **隐藏数据源** - 为其他报表提供数据支撑
4. **分类明确** - 资金余额和收款金额两大类

### 9.2 注意事项

1. **隐藏表** - 作为数据源表，不直接展示给用户
2. **数据量大** - 306行数据，需要维护大量科目
3. **层级复杂** - 四级层级结构，需要严格遵循编码规则
4. **汇总关系** - 上级汇总依赖下级明细

### 9.3 与其他表的关系

项目收支汇总表是**核心数据源表**，为表1、表2、表3提供资金和收款数据：

```
项目收支汇总表(数据源)
    ↓
表1(资金预算批复单)
    ↓
表2(商务管理分析表)
    ↓
表3(商务管理汇总表)
```

### 9.4 核心价值

1. **统一数据源** - 集中管理所有收支数据
2. **数据一致性** - 确保各报表数据一致
3. **层级汇总** - 支持多级汇总
4. **灵活扩展** - 可以添加新的收支科目

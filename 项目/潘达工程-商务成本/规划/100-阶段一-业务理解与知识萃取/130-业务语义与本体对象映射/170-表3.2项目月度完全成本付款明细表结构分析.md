## 二、报表整体结构

表3.2采用**纵向展开+横向分组**的结构设计，主要分为五大成本类别和多个数据区域：

```
┌───────────────────────────────────────────────────────────────────────────────────┐
│ 表头:项目名称、报告期间、建筑面积、管理费率                                    │
├───────────────────────────────────────────────────────────────────────────────────┤
│ A列:序号(计算编号)                                                           │
│ B列:成本项目                                                                 │
│ C列:代码                                                                     │
│ D列:成本属性                                                                 │
│ E列:级数                                                                     │
│ F列:基本信息(合同编码、合同名称、合同内容、供应商、合同金额等)                  │
│ N列:自开工累计已发生成本(已确认、待确认、合计、人工费)                        │
│ R列:其中:本月已发生成本(已确认、待确认、合计)                                │
│ U列:成本刚性度                                                               │
│ V列:应付款(已确应付款、人工费)                                               │
│ X列:开累已付款(金额、比例)                                                  │
│ Z列:应付未付款-合约(开累应付未付、人工费、不含待确、待确、小计)              │
│ AF列:资金计划(付款方式、付款类型、项目支出计划、累计付款比例、批准金额)        │
└───────────────────────────────────────────────────────────────────────────────────┘
```

## 五、公式逻辑分析

### 5.1 成本计算公式

| 公式 | 说明 |
|------|------|
| P列 = N列 + O列 | 自开工累计合计 = 已确认 + 待确认 |
| T列 = R列 + S列 | 本月合计 = 已确认 + 待确认 |
| U列 = N列 / P列 | 成本刚性度 = 已确认 / 合计 |

### 5.2 付款计算公式

| 公式 | 说明 |
|------|------|
| Y列 = X列 / N列 | 已付款比例 = 已付款金额 / 已确认成本 |
| Z列 = V列 - X列 | 开累应付未付 = 已确应付款 - 已付款 |
| AD列 = AB列 + AC列 | 应付未付小计 = 不含待确 + 待确成本 |

### 5.3 资金计划公式

| 公式 | 说明 |
|------|------|
| AH列 = (X列 + AG列) / N列 | 累计付款比例(含本次) = (已付款 + 本次计划) / 已确认 |
| AJ列 = (X列 + AI列) / N列 | 批准累计付款比例 = (已付款 + 批准金额) / 已确认 |

## 七、业务逻辑分析

### 7.1 成本确认流程

```
1. 合同签订(基本信息录入)
    ↓
2. 成本发生(本月成本录入R、S列)
    ↓
3. 成本累计(自开工累计N、O列)
    ↓
4. 成本刚性度计算(U列)
    ↓
5. 应付款计算(V、W列)
    ↓
6. 已付款记录(X、Y列)
    ↓
7. 应付未付计算(Z-AD列)
    ↓
8. 资金计划(AG、AI列)
```

### 7.2 付款周期管理

| 阶段 | 数据内容 | 关键指标 |
|------|---------|---------|
| 合同签订 | 合同金额、付款比例 | 付款条款 |
| 成本发生 | 已确认、待确认成本 | 成本刚性度 |
| 应付款计算 | 已确应付款 | 合约应付款 |
| 已付款记录 | 开累已付金额、比例 | 付款进度 |
| 应付未付 | 未付款金额 | 合约应付未付 |
| 资金计划 | 本次计划、批准金额 | 累计付款比例 |

## 九、报表特点总结

### 9.1 优势特点

1. **结构完整** - 五大成本类别全覆盖
2. **层级清晰** - 一级、二级、三级层次分明
3. **数据详细** - 每个成本项目都有完整的付款跟踪
4. **资金管理** - 完整的付款周期管理

### 9.2 注意事项

1. **表格复杂** - 81行×43列，数据量很大
2. **手动录入多** - 大量数据需要手动填写
3. **关联性强** - 与表3、表2紧密关联
4. **维护难度大** - 需要维护大量合同信息

### 9.3 与表3的关系

表3.2是**表3完全成本区域**的数据源头：
- 表3的BA列-CN列引用表3.2的数据
- 表3.2的管理费率(P2)被表3和表2引用

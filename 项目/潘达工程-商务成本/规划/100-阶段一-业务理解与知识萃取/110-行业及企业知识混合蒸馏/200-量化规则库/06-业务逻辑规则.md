## 一、状态流转规则（BR-S）

### 【BR-S001】项目状态流转规则
- **规则编号**：BR-S001
- **规则名称**：项目状态流转规则
- **规则类别**：业务逻辑规则 → 状态流转规则
- **适用实体**：Project
- **状态流转路径**：准备 → 施工 → 竣工 → 结算 → 保修 → 完成
- **允许的状态转换**：
  - 准备 → 施工
  - 施工 → 竣工
  - 竣工 → 结算
  - 结算 → 保修
  - 保修 → 完成
  - 施工 → 准备（返工）
  - 竣工 → 施工（整改）
- **禁止的状态转换**：
  - 直接从准备到结算
  - 直接从施工到保修
  - 完成状态不能转回其他状态
- **业务依据**：项目生命周期管理规范
- **校验逻辑**：状态转换必须符合允许的流转路径
- **错误级别**：ERROR
- **错误提示**：项目状态转换不符合规范
- **企业规模适配**：年产值500-600亿企业适用
- **平台映射**：状态机约束 + 校验规则

## 二、业务约束规则（BR-C）

### 【BR-C001】付款不超过合同金额约束规则
- **规则编号**：BR-C001
- **规则名称**：付款不超过合同金额约束规则
- **规则类别**：业务逻辑规则 → 业务约束规则
- **适用实体**：Payment, Contract
- **约束关系**：累计付款 ≤ 合同金额 × 付款比例
- **公式表达**：SUM(PaymentAmount) <= ContractAmount × PaymentRatio
- **参数定义**：
  - PaymentAmount：各期付款金额
  - ContractAmount：合同金额
  - PaymentRatio：合同约定付款比例（通常≤100%）
- **业务依据**：合同付款条款约束
- **校验逻辑**：累计付款金额不能超过合同约定的可付款金额
- **错误级别**：ERROR
- **错误提示**：累计付款金额已超过合同约定金额
- **企业规模适配**：年产值500-600亿企业适用
- **平台映射**：业务规则约束 + 校验规则

### 【BR-C002】质保金暂扣约束规则
- **规则编号**：BR-C002
- **规则名称**：质保金暂扣约束规则
- **规则类别**：业务逻辑规则 → 业务约束规则
- **适用实体**：Payment, Contract
- **约束关系**：每次付款时暂扣质保金
- **公式表达**：PaymentAmount = PayableAmount - RetentionAmount
- **参数定义**：
  - PaymentAmount：实际付款金额
  - PayableAmount：应付金额
  - RetentionAmount：暂扣质保金金额
- **业务依据**：合同质保金条款
- **校验逻辑**：付款时必须暂扣合同约定比例的质保金
- **错误级别**：ERROR
- **错误提示**：未按合同约定暂扣质保金
- **企业规模适配**：年产值500-600亿企业适用
- **平台映射**：业务规则约束 + 校验规则

### 【BR-C003】分包付款约束规则
- **规则编号**：BR-C003
- **规则名称**：分包付款约束规则
- **规则类别**：业务逻辑规则 → 业务约束规则
- **适用实体**：Payment, Receivable
- **约束关系**：分包付款 ≤ 业主收款 × 付款比例
- **公式表达**：SubcontractPayment <= OwnerReceipt × PaymentRatio
- **参数定义**：
  - SubcontractPayment：分包付款金额
  - OwnerReceipt：业主收款金额
  - PaymentRatio：公司内部付款比例（通常≤80%）
- **业务依据**：背靠背付款条款
- **校验逻辑**：分包付款不能超过业主收款按比例计算的金额
- **错误级别**：ERROR
- **错误提示**：分包付款超过业主收款可支付范围
- **企业规模适配**：年产值500-600亿企业适用
- **平台映射**：业务规则约束 + 校验规则

### 【BR-C004】成本刚性约束规则

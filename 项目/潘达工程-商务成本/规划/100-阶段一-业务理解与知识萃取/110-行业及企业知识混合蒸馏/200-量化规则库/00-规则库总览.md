## 一、规则库概述

### 1.1 规则库定位

本规则库是建筑工程行业商务成本管理知识体系的核心组成部分，旨在：

| 定位维度 | 描述 |
|---------|------|
| **数据模拟支撑** | 为数据模拟提供取值范围、约束条件和生成模板 |
| **数据质量保障** | 为数据校验提供标准化规则 |
| **智能决策引擎** | 为智能决策提供可执行的业务规则 |
| **业务知识沉淀** | 将业务经验转化为可量化的规则体系 |

### 1.3 规则编码规范

```
规则编号 = 规则类别前缀 + 子类别前缀 + 3位数字序号

示例：
- QR-C001：业务运算规则-成本类第1条
- QR-F001：业务运算规则-资金类第1条
- DR-A001：数据取值规则-金额类第1条
- CR-I001：数据勾稽规则-指标间勾稽第1条
- FR-T001：数据格式规则-数据类型第1条
- IR-R001：数据完整性规则-必填字段第1条
- BR-C001：业务逻辑规则-业务约束第1条
```

### 1.4 规则描述模板

```markdown
【规则编号】QR-C001
【规则名称】成本偏差率计算规则
【规则类别】业务运算规则 → 成本类规则
【适用实体】ProjectCost, Budget
【适用场景】成本分析、成本预警
【量化公式】成本偏差率 = (实际成本 - 预算成本) / 预算成本 × 100%
【参数定义】
  - 实际成本：来自成本台账 fact_project_cost
  - 预算成本：来自预算表 fact_budget
【阈值定义】成本偏差率 > 5% 时触发预警
【预警级别】黄色预警
【业务依据】行业成本控制标准
【企业规模适配】年产值500-600亿企业适用
【数据来源】fact_project_cost, fact_budget
【平台映射】派生属性（成本偏差率）+ Action Type（预警通知）
【关联指标】成本偏差率指标
【关联关系】成本与预算的对比关系
```

## 三、企业规模基准参数

### 3.1 企业规模基准（年产值500-600亿）

| 参数 | 基准值范围 | 说明 |
|-----|-----------|------|
| 年产值 | 500-600亿元 | 央国企工程类公司规模 |
| 项目数量 | 300-500个 | 在建项目数量 |
| 员工规模 | 15000-25000人 | 含子公司、分公司 |
| 合同数量 | 2000-3000个 | 年度合同签约量 |
| 子/分公司 | 10-20个 | 区域布局 |
| 项目经理 | 300-500人 | 核心管理团队 |

### 3.2 关键比率基准

| 比率指标 | 正常范围 | 预警阈值 |
|---------|---------|---------|
| 成本刚性度 | ≥95% | <90%红色预警 |
| 产值确权比 | ≥85% | <80%黄色预警 |
| 收款率 | ≥80% | <60%红色预警 |
| 利润率 | ≥3% | <0%红色预警 |
| 资金周转率 | 3-6次/年 | <2次/年预警 |
| 资产负债率 | 50%-80% | >85%预警 |

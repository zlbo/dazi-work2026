# 本体建模落地计划

> **依据规范**：《900-方法论体系/000-核心纲领/010-项目总体规划.md》《900-方法论体系/000-核心纲领/020-本体蒸馏萃取与设计方法论.md》  
> **技术依据**：《本体规划指南》《本体命名规范》《本体分类规划与SDK扩展方案》

## 一、执行方式

本步骤调用架构师标准方法执行：
- 使用 `dazi.onto` SDK 生成初始化脚本
- 创建物理表（遵循 `dim_`/`fact_` 命名规范）
- 注册表间关系
- 注册 Cube 并定义维度/度量
- 定义 Object Type、Link Type 并绑定数据源
- 挂载平台分类

## 二、输出物

- 本体建模落地完成确认
- SDK 脚本执行日志
- 本体定义验证报告

## 三、依赖条件

- Step 4：本体设计与确认已完成
- 阶段二产出的 Object Type、Link Type 定义文档

## 四、后续步骤

本体建模落地完成后，进入 Step 5：本体脚本发布与验证

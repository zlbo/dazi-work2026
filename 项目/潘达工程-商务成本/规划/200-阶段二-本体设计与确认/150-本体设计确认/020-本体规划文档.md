## 一、本体设计概述

### 1.1 设计原则

| 原则 | 说明 | 平台规约要求 |
|------|------|-------------|
| **业务导向** | Object Type面向业务人员，而非技术实现 | 使用稳定的业务英文标识 |
| **平台对齐** | 遵循Palantir本体体系规范 | 参考《本体规划指南》 |
| **知识集成** | 整合阶段一的显性/隐性关系和行业诀窍 | 关联知识标注 |
| **映射一致** | 与阶段二的字段映射矩阵保持一致 | bind_source配置正确 |
| **可扩展性** | 支持后续业务扩展 | 命名空间规范 |

### 1.2 四层架构对应

| 层级 | 内容 | 本阶段产出 |
|------|------|-----------|
| **应用层** | AI助手、Web报表、可视化 | 后续阶段 |
| **函数层** | Function、Action | 后续阶段 |
| **本体层** | Object Type、Link Type、Action Type | 本阶段核心 |
| **Cube层** | 平台自动生成 | 本体数据架构设计文档 |

### 1.3 平台规范要求

| 规范项 | 要求 | 本项目实现 |
|--------|------|-----------|
| **Object Type命名** | 使用稳定的业务英文标识 | Project、ProjectOutput等 |
| **Link Type定义** | 两端为对象类型code，非Cube名 | belongsTo、contains、hasPart |
| **bind_source配置** | 每个Object Type必须绑定Cube | 见本章4节 |
| **qualified_name格式** | CubeName.member | ProjectCube.project_name |
| **脚本目录结构** | 项目/xxx/脚本/setup/functions/actions | 按规范创建 |

### 3.2 Link Type详细定义

#### 3.2.1 belongsTo（属于关系）

| 属性 | 值 |
|------|------|
| 语义 | 表示一个对象属于另一个对象 |
| 基数 | 多对一（Many-to-One） |
| 方向 | 从子对象指向父对象 |
| 示例 | ProjectOutput → Project（产值属于项目） |

#### 3.2.2 contains（包含关系）

| 属性 | 值 |
|------|------|
| 语义 | 表示一个对象包含另一个对象 |
| 基数 | 一对多（One-to-Many） |
| 方向 | 从父对象指向子对象 |
| 示例 | ProjectCost → Contract（成本包含合同） |

#### 3.2.3 hasPart（组成关系）

| 属性 | 值 |
|------|------|
| 语义 | 表示一个对象由多个部分组成 |
| 基数 | 一对多（One-to-Many） |
| 方向 | 从整体指向部分 |
| 示例 | Project → Section（项目包含标段） |

## 四、Action Type定义

### 4.1 预警通知Action

| 属性 | 值 |
|------|------|
| **名称** | SendRiskAlert |
| **描述** | 当风险等级达到预警阈值时，发送预警通知 |
| **触发条件** | ProjectRisk.warningLevel == '高' |
| **参数** | projectId, riskType, riskDescription, recipient |
| **执行动作** | 发送邮件/短信通知相关责任人 |

### 4.2 付款审批Action

| 属性 | 值 |
|------|------|
| **名称** | ApprovePayment |
| **描述** | 审批付款申请 |
| **触发条件** | ProjectPayment.paidAmount > threshold |
| **参数** | paymentId, projectId, amount, approver |
| **执行动作** | 创建审批工单，通知审批人 |

### 4.3 指标异常提醒Action

| 属性 | 值 |
|------|------|
| **名称** | SendIndicatorAlert |
| **描述** | 当指标超出阈值范围时发送提醒 |
| **触发条件** | ProjectIndicator.indicatorValue > targetValue * 1.1 OR < targetValue * 0.9 |
| **参数** | projectId, indicatorCode, currentValue, targetValue |
| **执行动作** | 发送异常指标提醒通知 |

## 一、设计原则

### 1.1 命名规范

> **引用**：《本体命名规范_物理表Cube与对象.md》§2.1

**命名格式**：`{类别前缀}_{业务域}_{实体}[_{粒度}]`

| 规则 | 说明 |
|------|------|
| 全小写 **snake_case** | 符合ClickHouse命名规范 |
| **类别前缀必填** | fact_、dim_、bridge_、map_、cfg_、agg_ |
| 单域实现可省略业务域 | 如 `dim_project_status` 而非 `dim_cost_project_status` |

### 1.2 表类别定义

| 前缀 | 角色 | 本项目应用 |
|------|------|-----------|
| **`dim_`** | 维度/主数据 | 项目、合同、组织、地区、科目、状态等 |
| **`fact_`** | 事实/明细/事件 | 成本、产值、付款、收款、现金流、变更、索赔等 |
| **`bridge_`** | 桥表（多对多） | 项目-分包商、合同-材料等（如需要） |
| **`map_`** | 映射/对照/码表 | 科目编码映射、状态码映射等 |
| **`cfg_`** | 配置/参数/版本 | 预算版本、指标版本等（如需要） |
| **`agg_`** | 预聚合/主题汇总 | 项目月度汇总、组织月度汇总等（可选） |

### 1.3 主键/外键约定

> **引用**：《本体命名规范_物理表Cube与对象.md》§2.4

| 对象 | 约定 | 示例 |
|------|------|------|
| 时间维 PK | **`date_key`** Int32 | `20250605`（YYYYMMDD） |
| 业务维 PK | `{entity}_id` 或 `{entity}_key` | `project_key`、`cost_subject_key` |
| 事实表 FK | 与维表 PK **同名** | `project_key`、`date_key`、`cost_subject_key` |

## 二、时间维：dim_date（强制）

> **引用**：《本体命名规范_物理表Cube与对象.md》§3「时间维 dim_date（强制）」

### 2.1 表定义

**表名**：`dim_date`

**用途**：全空间共享日历；所有事实表通过 **`date_key`** 关联。

| 字段名 | 类型 | 显示名 | 说明 | 排序键 |
|--------|------|--------|------|--------|
| **`date_key`** | Int32 | 日期键 | YYYYMMDD，主键 | ✅ |
| calendar_date | Date | 自然日 | | |
| year | Int16 | 公历年 | | |
| quarter | Int8 | 季度 | 1-4 | |
| month | Int8 | 月 | 1-12 | |
| week_of_year | Int8 | 周 | | |
| day_of_week | Int8 | 星期 | | |
| is_weekend | UInt8 | 是否周末 | 0/1 | |
| year_month | String | 年月 | 如 2025-06 | |
| fiscal_year | Int16 | 会计年度 | | |
| fiscal_period | Int8 | 会计期间 | 1-12 | |

**排序键**：`ORDER BY (date_key)`  
**引擎**：`MergeTree()`  
**分区**：`PARTITION BY toYYYYMM(calendar_date)`

### 2.2 事实表关联

```text
fact_*.date_key  →  dim_date.date_key
```

## 一、规则定义概述

### 1.1 业务规则分类体系

| 规则大类 | 规则子类 | 规则编号前缀 | 规则数量 | 说明 |
|---------|---------|-------------|---------|------|
| **计算规则** | 成本类规则 | QR-C | 7条 | 成本偏差率、节约率、结构占比等 |
| | 资金类规则 | QR-F | 8条 | 资金周转率、收款率、付款比例等 |
| | 进度类规则 | QR-S | 4条 | 进度偏差率、工期完成率等 |
| | 风险类规则 | QR-R | 4条 | 风险综合评分、变更频率等 |
| | 利润类规则 | QR-P | 4条 | 利润率、成本利润率等 |
| **勾稽关系** | 指标间勾稽 | CR-I | 20条 | 利润=产值-成本等公式关系 |
| | 层级汇总勾稽 | CR-H | 12条 | 总成本=直接成本+间接成本等 |
| | 时间序列勾稽 | CR-T | 14条 | 本期累计=上期累计+本期发生等 |
| **约束条件** | 状态流转规则 | BR-S | 8条 | 合同状态流转约束 |
| | 业务约束规则 | BR-C | 22条 | 付款比例上限、质保金约束等 |
| **数据标准** | 数据取值规则 | DR-A/R/T | 35条 | 金额、比率、时间取值范围 |
| | 数据格式规则 | FR-T/C | 30条 | 数据类型、编码规则 |
| | 数据完整性规则 | IR-R/F | 38条 | 必填字段、引用完整性 |

### 1.2 规则编码规范

```
规则编号 = 规则类别前缀 + 子类别前缀 + 3位数字序号

示例：
- QR-C001：业务运算规则-成本类第1条
- CR-I001：数据勾稽规则-指标间勾稽第1条
- BR-C001：业务逻辑规则-业务约束第1条
```

### 1.3 规则与本体映射关系

| 规则类型 | 映射到本体 | 说明 |
|---------|-----------|------|
| 计算规则 | 派生属性 | 通过计算规则生成派生指标 |
| 勾稽关系 | 数据校验 | 确保数据一致性和完整性 |
| 约束条件 | Action Type | 触发审批流程或预警通知 |
| 数据标准 | 属性定义 | 约束属性取值范围和格式 |

## 二、计算规则定义

### 2.1 成本类计算规则

#### QR-C001：成本偏差率计算规则
```yaml
规则编号: QR-C001
规则名称: 成本偏差率计算规则
适用实体: ProjectCost, Budget
适用场景: 成本分析、成本预警
量化公式: 成本偏差率 = (实际成本 - 预算成本) / 预算成本 × 100%
参数定义:
  - 实际成本: 来自成本台账 fact_project_cost
  - 预算成本: 来自预算表 fact_budget
阈值定义: 成本偏差率 > 5% 时触发预警
预警级别: 黄色预警（5%-10%）、红色预警（>10%）
平台映射: 派生属性（成本偏差率）+ Action Type（AlertTrigger）
```

#### QR-R001：风险综合评分规则
```yaml
规则编号: QR-R001
规则名称: 风险综合评分规则
适用实体: Risk
量化公式: 风险综合评分 = (发生概率 × 10) × 影响程度 / 可检测性修正系数
参数定义:
  - 发生概率: 0-1（转换为0-10分）
  - 影响程度: 1-5（1=轻微，5=严重）
  - 可检测性修正系数: 难检测=1.2，一般=1.0，易检测=0.8
阈值定义: 风险综合评分 > 25 时触发预警
预警级别: 黄色预警（25-40）、红色预警（>40）
平台映射: 派生属性（风险综合评分）+ Action Type（RiskAssessment）
```

```yaml
规则名称: 利润计算公式规则
```

## 核心设定：业务主体唯一性

> **本交接词的第一原则**：所有模拟数据的业务主体只有一个——**潘达建工集团有限公司**

### 主体架构
- **集团总部**：潘达建工集团有限公司（唯一法人主体）
- **分子公司**：5个专业子公司（房建公司一、市政公司、安装公司、基础设施公司、投资公司）
- **项目部**：100个项目部（每个项目对应一个）

### 关键区分（避免混淆）

| 概念 | 定义 | 是否业务主体 |
|------|------|-------------|
| **潘达建工集团** | 项目承包方、成本归属主体 | **是（唯一）** |
| **业主** | 项目发包方（如华润置地、地方城投等） | 否 |
| **客户** | 服务对象 | 否 |
| **分子公司** | 潘达建工内部机构 | 否（内部组织） |
| **项目部** | 潘达建工内部机构 | 否（内部组织） |

## 一、背景与目的

### 1.1 背景说明

在阶段三（本体建模与落地）的实施过程中，需要编写**灌数脚本（`panda_cost_seed_data.py`）** 来生成种子数据（seed data）。这些种子数据虽然只是基础框架数据，但它们将作为**阶段四模拟数据生成的基础**。

**关键认识**：
- 种子数据的字段名称、字段类型、数据约束、枚举值定义，将直接影响后续事实表数据的生成
- 种子数据的多样性、真实性，将决定整个模拟数据体系的基础质量
- 种子数据中如果存在不合理的设计，后续要修复成本会很高

### 1.2 目的

本交接词旨在：
1. **明确种子数据与模拟数据的关系**：种子数据是模拟数据的基础
2. **传递模拟数据设计的方法论要求**：确保种子数据生成时考虑仿真度
3. **提出具体的技术要求**：在编写灌数脚本时落实这些要求
4. **建立阶段协作机制**：避免阶段三的种子数据与阶段四的模拟数据脱节

## 三、种子数据仿真度要求

### 3.1 核心原则

> **种子数据是模拟数据的地基，必须满足"数据质量高 + 业务仿真度高 + 经营品质差异化"三大要求。**

### 3.2 数据质量要求

| 维度 | 要求 | 说明 |
|------|------|------|
| **业务规则** | 100%满足本体定义的业务规则 | 字段值必须符合业务约束 |
| **数据标准** | 100%符合数据类型、长度、精度 | 严格按物理表设计 |
| **勾稽关系** | 100%满足表间关联关系 | 外键必须存在 |
| **枚举值** | 100%在定义的枚举范围内 | 不能有未定义的值 |

### 3.3 数值精度要求

> **核心原则**：所有金额统一使用**元**作为单位，保留两位小数。

| 数据类型 | 单位 | 精度要求 | 示例 |
|---------|------|---------|------|
| 合同金额 | 元 | 保留两位小数 | 125,683,500.00元 |
| 项目金额 | 元 | 保留两位小数 | 850,000,000.00元 |
| 预算成本 | 元 | 保留两位小数 | 765,000,000.00元 |
| 比率 | - | 保留4位小数 | 0.1578 |
| 面积 | ㎡ | 保留两位小数 | 35,678.90㎡ |

**严禁**：
- 出现 100,000,000 这样的整数（应该是 100,000,000.00）
- 出现 5亿 这样的简化值（应该是 500,000,000.00元）

### 3.5 经营品质差异化要求

> **核心原则**：种子数据中要预留经营品质的差异化特征。

#### 项目经营品质分级

| 品质等级 | 数量 | 占比 | 特征 |
|---------|------|------|------|
| **A级（卓越）** | 10个 | 10% | 工期0偏差、成本0超支、无风险 |
| **B级（优良）** | 25个 | 25% | 工期<5%偏差、成本<3%超支、少量低风险 |
| **C级（一般）** | 35个 | 35% | 工期5-10%偏差、成本3-8%超支、常规问题 |

**实施要点**：
- 在项目主数据中增加 `quality_level` 字段（A/B/C/D/E）
- 在项目主数据中增加 `expected_deviation_rate` 字段（预期偏差率）
- 业主主数据中增加 `payment_behavior` 字段（付款行为：优秀/良好/一般/较差）
- 业主信用等级与项目品质等级保持相关性

#### 业主与项目品质关联

| 业主类型 | 关联项目品质 | 问题表现 |
|---------|-------------|---------|
| 央企/优质国企（AAA） | A/B级为主 | 付款及时、无拖欠 |
| 地方平台（AA+/AA） | B/C级为主 | 付款周期较长、偶有延期 |
| 优质民企（A/BBB） | C级为主 | 付款正常 |
| 问题民企（BB及以下） | D/E级为主 | 拖欠严重、可能违约 |

### 4.2 数据生成注意事项

#### 4.2.1 数值精度实现

```python
import random
from decimal import Decimal, ROUND_HALF_UP

def round_to_yuan(value):
    """将金额四舍五入到分（保留两位小数）"""
    return float(Decimal(str(value)).quantize(Decimal('0.01'), rounding=ROUND_HALF_UP))

# 示例
contract_amount = round_to_yuan(850000000 + random.uniform(-1000000, 1000000))
```

#### 4.2.2 多样性实现

```python
import random

def weighted_choice(choices):
    """加权随机选择"""
    total = sum(weight for _, weight in choices)
    r = random.uniform(0, total)
    cumulative = 0
    for value, weight in choices:
        cumulative += weight
        if r <= cumulative:
            return value
    return choices[-1][0]

# 示例：项目类型分布
project_type = weighted_choice([
    ('房建', 30),
    ('市政', 30),
    ('安装', 25),
    ('基础设施', 15)
])
```

#### 4.2.3 经营品质差异化实现

```python
def assign_quality_level(credit_level):
    """根据业主信用等级分配项目品质"""
    if credit_level == 'AAA':
        return weighted_choice([('A', 50), ('B', 40), ('C', 10)])
    elif credit_level in ['AA+', 'AA']:
        return weighted_choice([('A', 20), ('B', 40), ('C', 30), ('D', 10)])
    elif credit_level in ['A', 'BBB']:
        return weighted_choice([('B', 20), ('C', 50), ('D', 25), ('E', 5)])
    else:  # BB及以下
        return weighted_choice([('C', 20), ('D', 50), ('E', 30)])
```

## 一、业务场景与分析目标

### 1.1 业务场景
商务成本智能决策体系旨在实现：
- **成本管控**：实时监控项目成本，分析成本偏差
- **利润分析**：计算项目利润，评估盈利能力
- **风险预警**：识别潜在风险，触发三色预警
- **现金流管理**：预测现金流，保障资金安全
- **经营决策**：基于数据支持经营决策

### 1.2 分析目标
| 目标 | 描述 |
|------|------|
| 成本偏差分析 | 对比实际成本与目标成本，识别偏差原因 |
| 成本刚性分析 | 评估成本确认程度，判断成本质量 |
| 产值确权分析 | 分析产值确认比例，评估收入质量 |
| 现金流预测 | 预测未来现金流，支持资金计划 |
| 风险评估 | 综合评估项目风险等级 |

### 1.3 边界定义
- **时间范围**：项目全生命周期（立项 → 竣工 → 结算）
- **数据范围**：成本、产值、付款、收款、现金流、风险
- **组织范围**：集团及下属子公司

## 三、Cube层设计

### 3.1 Cube清单

| Cube 名 | 类别 | 事实源表 | 支撑对象类型 |
|---------|------|---------|-------------|
| ProjectCostCube | Process | fact_project_cost | ProjectCost, Project |
| ProjectOutputCube | Process | fact_project_output | ProjectOutput, Project |
| PaymentCube | Process | fact_payment | Payment, Project, Contract |
| ReceivableCube | Process | fact_receivable | Receivable, Project, Contract |
| CashFlowCube | Process | fact_cash_flow | CashFlow, Project |
| RiskCube | Process | fact_risk | Risk, Project |
| ChangeOrderCube | Process | fact_change_order | ChangeOrder, Project, Contract |
| ClaimCube | Process | fact_claim | Claim, Project, Contract |
| ProjectIndicatorCube | Subject | fact_project_indicator | Indicator, Project |

### 3.2 Cube详细设计

#### 3.2.1 ProjectCostCube
- **维度**：date_key, project_key, project_type, project_category, business_model, cost_subject_key, region_key, organization_key
- **度量**：cost_amount(SUM), cost_count(COUNT)
- **派生度量**：cost_amount_avg, cost_ratio_by_subject, cost_ratio_by_project

#### 3.2.2 ProjectOutputCube
- **维度**：date_key, project_key, project_type, project_category, business_model, output_type, confirmation_status, region_key, organization_key
- **度量**：output_amount(SUM), output_count(COUNT)
- **派生度量**：confirmed_output_amount, unconfirmed_output_amount, output_confirm_ratio

#### 3.2.3 PaymentCube
- **维度**：date_key, project_key, contract_key, payment_type, payment_status, supplier_key, region_key, organization_key
- **度量**：payment_amount(SUM), payment_count(COUNT)
- **派生度量**：approved_payment_amount, paid_payment_amount

#### 3.2.4 ReceivableCube
- **维度**：date_key, project_key, contract_key, receivable_status, region_key, organization_key
- **度量**：receivable_amount(SUM), received_amount(SUM), outstanding_amount(SUM)
- **派生度量**：receivable_ratio, overdue_receivable_amount

#### 3.2.5 CashFlowCube
- **维度**：date_key, project_key, cash_flow_type, cash_flow_category, approval_status, region_key, organization_key
- **度量**：cash_flow_amount(SUM), cash_flow_count(COUNT)
- **派生度量**：income_amount, expense_amount, net_cash_flow

#### 3.2.6 RiskCube
- **维度**：date_key, project_key, risk_type, risk_level, region_key, organization_key
- **度量**：risk_count(COUNT), risk_value_avg(AVG)
- **派生度量**：high_risk_count, medium_risk_count, low_risk_count, risk_ratio

#### 3.2.7 ChangeOrderCube
- **维度**：date_key, project_key, contract_key, change_type, approval_status, region_key, organization_key
- **度量**：change_amount(SUM), change_count(COUNT)
- **派生度量**：approved_change_amount, change_amount_avg

#### 3.2.8 ClaimCube
- **维度**：date_key, project_key, contract_key, claim_type, approval_status, region_key, organization_key
- **度量**：claim_amount(SUM), claim_count(COUNT)
- **派生度量**：approved_claim_amount, claim_amount_avg

#### 3.2.9 ProjectIndicatorCube
- **维度**：date_key, project_key, project_type, project_category, business_model, project_status, region_key, organization_key
- **度量**：total_output(SUM), total_cost(SUM), profit(SUM), profit_rate_avg(AVG), cost_rigidity_avg(AVG)
- **派生度量**：profit_margin, cost_output_ratio

## 六、与参照示例的差异说明

### 6.1 业务差异

| 对比项 | 参照示例（利润示例） | 本实现（商务成本） | 差异说明 |
|--------|-------------------|------------------|---------|
| 业务域 | 利润/GL分析 | 商务成本分析 | 聚焦施工项目成本管控 |
| 核心对象 | GL科目、成本中心 | 项目、合同、成本 | 以项目为核心单元 |
| 分析维度 | 成本中心、费用类型 | 项目类型、经营模式、地区 | 符合建筑行业特性 |
| 特有规则 | 标准GL核算 | 成本三级分类、产值双轨制、风险三色预警 | 企业特有业务规则 |

### 6.2 技术差异

| 对比项 | 参照示例 | 本实现 | 说明 |
|--------|---------|--------|------|
| Cube数量 | 3个 | 9个 | 覆盖成本、产值、付款、收款、现金流、风险等多个业务域 |
| Object Type数量 | 5个 | 11个 | 更细粒度的业务对象划分 |
| 业务函数 | 3个 | 7个 | 涵盖成本、利润、风险、现金流等分析场景 |

## 二、行业特征分析

### 2.1 石油天然气行业特征

| 特征维度 | 具体表现 | 对本体设计的影响 |
|---------|---------|-----------------|
| **业务链长** | 勘探→钻井→完井→采油→集输→处理→销售 | 需要设计跨域关联关系 |
| **专业领域多** | 地质、油藏、钻井、采油、安全等多专业 | 需要多领域本体对象 |
| **数据类型复杂** | 实时传感器、地质数据、设备数据、生产数据 | 需要支持多类型数据融合 |
| **空间分布广** | 井场分布广、地理跨度大 | 需要空间维度和区域管理 |
| **生产连续性** | 24/7不间断生产 | 需要设备健康管理和预警 |
| **安全要求高** | 高温高压、易燃易爆 | 需要QHSE本体设计 |

### 2.2 数字化水平

| 维度 | 现状 |
|------|------|
| **物联网覆盖率** | 单井覆盖率超78% |
| **数据采集频率** | 单井日均500+数据点，15秒级更新 |
| **智能分析能力** | 产量预测准确率91%+，故障响应缩短63% |
| **无人值守** | 中小型站场无人值守率86%+ |

## 三、业务链全貌

### 3.1 完整业务链条

```
地质勘探(D1) → 钻井工程(D2) → 完井改造(D3) → 采油生产(D4)
                      ↓                          ↓
                  油藏工程(D7) ←------ 增产措施(D5)
                      ↓
               地面集输(D8) → 设备资产(D9) → QHSE(D10)
                      ↓
                生产动态管理(D6)
```

### 3.2 核心实体"井"的跨域贯穿

| 业务域 | 与井的关系 | 核心属性 |
|--------|-----------|---------|
| D2 钻井工程 | 井身结构定义 | 井深、井径、钻井日期 |
| D3 完井改造 | 射孔、压裂作业 | 射孔段、压裂段数 |
| D4 采油生产 | 产量产出 | 日产液、含水率、油气比 |
| D5 增产措施 | 措施作业 | 措施类型、效果评价 |
| D7 油藏工程 | 注采关联 | 注采井组、压力监测 |
| D9 设备资产 | 装备关联 | 抽油机、电潜泵 |
| D10 QHSE | 安全监控 | 完整性状态、安全事件 |

## 五、关键KPI体系

### 5.1 生产指标

| KPI | 说明 | 计算方式 |
|-----|------|---------|
| 日产油量 | 单井/油田日产量 | 统计值 |
| 日产液量 | 油+水日产量 | 统计值 |
| 含水率 | 产出水中水量占比 | 水量/液量 × 100% |
| 油气比 | 气油产量比 | 气量/油量 |
| 生产时率 | 实际生产时间占比 | 生产时间/24小时 × 100% |
| 自然递减率 | 产量自然下降率 | (本期产量-上期产量)/上期产量 |

### 5.2 设备指标

| KPI | 说明 | 计算方式 |
|-----|------|---------|
| 设备完好率 | 完好设备占比 | 完好设备数/总设备数 × 100% |
| 故障停机时间 | 设备故障停机时长 | 统计值 |
| 维护成本 | 设备维护费用 | 统计值 |
| 泵效 | 实际排量/理论排量 | 实际排量/理论排量 × 100% |

### 5.3 油藏指标

| KPI | 说明 | 计算方式 |
|-----|------|---------|
| 地层压力 | 油藏压力 | 测量值 |
| 注采比 | 注水量/采出液量 | 注水量/采出液量 |
| 采收率 | 采出储量/地质储量 | 采出储量/地质储量 × 100% |

### 5.4 安全环保指标

| KPI | 说明 | 计算方式 |
|-----|------|---------|
| 事故发生率 | 安全事故次数 | 统计值 |
| 隐患整改率 | 隐患整改完成率 | 已整改隐患/总隐患 × 100% |
| 环保达标率 | 环保指标达标率 | 达标项/总项 × 100% |

## 七、结论

1. **业务复杂性**：新疆油田业务链长（10个业务域），核心实体"井"贯穿7个领域
2. **数据特征**：多类型数据融合，实时性要求高，空间分布广
3. **数字化基础**：物联网覆盖率高，智能分析能力成熟
4. **潘达油田建议**：以D4采油生产和D6生产动态管理为核心，优先构建单井级知识图谱

## 四、业务规则约束评估

### 4.1 核心约束清单

| 约束编号 | 约束名称 | 约束条件 | 模拟验证方法 |
|---------|---------|---------|------------|
| BC001 | 含水率范围 | 0% ≤ 含水率 ≤ 100% | ✅ 可验证 |
| BC003 | 液油勾稽 | 日产液=日产油/(1-含水率) | ✅ 可验证 |
| INJ001 | 注采比范围 | 0.8 ≤ 注采比 ≤ 1.5 | ⚠️ 需整体验证 |
| WAT001 | 泵效范围 | 0% ≤ 泵效 ≤ 100% | ✅ 可验证 |
| PRD101 | 产量非负 | 日产油 ≥ 0 | ✅ 可验证 |

### 4.2 勾稽关系验证方案

#### 层级一：单表内验证

```sql
-- 单井日产量数据勾稽验证
SELECT 
    well_key,
    date_key,
    daily_oil_production,
    daily_liquid_production,
    water_cut,
    CASE 
        WHEN ABS(daily_liquid_production * (1 - water_cut/100.0) - daily_oil_production) > 0.5 
        THEN '不一致'
        ELSE '一致'
    END as check_result
FROM fact_well_production
WHERE ABS(daily_liquid_production * (1 - water_cut/100.0) - daily_oil_production) > 0.5;
```

#### 层级二：表间勾稽验证

```sql
-- 区块月度注采比验证
WITH monthly_injection AS (
    SELECT 
        block_key,
        DATE_TRUNC('month', d.full_date) as month,
        SUM(wi.daily_water_injection) as total_injection
    FROM fact_well_injection wi
    JOIN dim_well w ON wi.well_key = w.well_key
    JOIN dim_date d ON wi.date_key = d.date_key
    GROUP BY block_key, DATE_TRUNC('month', d.full_date)
),
monthly_production AS (
    SELECT 
        block_key,
        DATE_TRUNC('month', d.full_date) as month,
        SUM(wp.daily_liquid_production) as total_liquid
    FROM fact_well_production wp
    JOIN dim_well w ON wp.well_key = w.well_key
    JOIN dim_date d ON wp.date_key = d.date_key
    GROUP BY block_key, DATE_TRUNC('month', d.full_date)
)
SELECT 
    i.block_key,
    i.month,
    i.total_injection,
    p.total_liquid,
    CASE 
        WHEN i.total_injection / p.total_liquid BETWEEN 0.8 AND 1.5 
        THEN '合理'
        ELSE '异常'
    END as ratio_check
FROM monthly_injection i
LEFT JOIN monthly_production p ON i.block_key = p.block_key AND i.month = p.month;
```

## 五、风险识别与建议

### 5.1 高风险项

| 风险编号 | 风险描述 | 风险等级 | 建议 |
|---------|---------|---------|------|
| R1 | 评价井为一次性数据，与常规井处理逻辑不同 | 高 | 增加well_category字段区分 |
| R2 | 举升方式变更历史难以模拟 | 中 | 简化为静态举升方式 |
| R3 | 新井投产时间不同，数据起始点不一致 | 中 | 使用well_start_date控制 |

### 5.2 中风险项

| 风险编号 | 风险描述 | 风险等级 | 建议 |
|---------|---------|---------|------|
| R4 | 措施作业（如压裂、酸化）对产量的影响难以精确模拟 | 中 | 基于方法论中的有效期规则简化处理 |
| R5 | 设备故障随机性难以精确模拟 | 低 | 基于故障率分布模拟 |
| R6 | 季节性产量波动与实际油田可能存在差异 | 低 | 参考方法论中的季节性系数 |

### 5.3 设计调整建议

#### 建议一：增加井分类字段

```sql
-- dim_well表增加字段
ALTER TABLE dim_well ADD COLUMN well_category VARCHAR(20); -- PR/IN/MO/EX

COMMENT ON COLUMN dim_well.well_category IS '井类别：PR-采油井, IN-注水井, MO-观察井, EX-评价井';
```

#### 建议二：增加设备与井的关联表

```sql
-- 新增 fact_well_equipment_change 表（可选）
-- 用于记录抽油机等设备的更换历史
```

#### 建议三：简化措施效果模拟

```yaml
# 措施效果简化模拟规则
stimulation_effect:
  - duration: 3-6个月
  - oil_increase_rate: 20-50%
  - decline_pattern: 线性递减
```

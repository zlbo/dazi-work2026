## 一、编码规范总则

### 1.1 编码原则

| 原则 | 说明 | 示例 |
|------|------|------|
| **唯一性** | 每个规则有唯一编码 | BC001唯一标识日产油量计算规则 |
| **可读性** | 编码能反映规则类型 | BC表示业务运算规则 |
| **可扩展性** | 编码支持新增规则 | BC026可新增第26条运算规则 |
| **规范性** | 编码格式统一 | 全部采用2字母+3数字格式 |

### 1.2 编码结构

```
编码结构：[类型前缀][序号]
示例：BC001
      ││└─── 序号（001-999）
      │└──── 类型前缀（2字母）
      └───── 规则编码
```

## 四、编码扩展规范

### 4.1 新增规则编码

```yaml
新增规则编码规范:
  步骤1: 确定规则类型（BC/BL/RE/IC/AR）
  步骤2: 确定规则类别（产量/成本/措施等）
  步骤3: 在现有编码范围内新增序号
  步骤4: 确保编码唯一性
  步骤5: 更新规则总览索引

示例:
  新增规则: 日产液量计算规则
  规则类型: 业务运算规则（BC）
  规则类别: 产量计算
  新增编码: BC026（在BC025之后）
```

## 六、编码管理规范

### 6.1 编码变更管理

```yaml
编码变更管理规范:
  变更类型:
    - 新增编码：新增规则时分配新编码
    - 修改编码：规则合并或拆分时调整编码
    - 删除编码：规则废弃时标记删除（保留编码不重用）
  变更流程:
    步骤1: 提交编码变更申请
    步骤2: 评估变更影响
    步骤3: 审批变更申请
    步骤4: 执行编码变更
    步骤5: 更新相关文档
```

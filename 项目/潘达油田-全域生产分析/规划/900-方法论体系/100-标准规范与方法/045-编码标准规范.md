## 二、编码分类与优先级

> **引用**：《000-顶层设计.md》，编码标准规范（核心定义）。

### 2.1 编码分类

| 编码类型 | 说明 | 示例 | 优先级 |
|---------|------|------|-------|
| **结构编码** | Object Type、Link Type、维度表结构等框架性编码 | `Well`、`WellProduction`、`dim_well` | 通识优先 |
| **业务编码** | 井型编码、区块编码、油藏类型编码等业务分类编码 | 井型：`PR-01-采油井`、`IN-01-注水井` | 通识优先，参考专识 |
| **属性值** | 具体的井名、区块名、设备名称等业务术语 | `Panda-01`、`North-Block-A` | 专识优先 |

### 2.2 优先级规则

| 编码类型 | 优先级 | 说明 |
|---------|-------|------|
| **结构编码** | 通识优先 | 采用行业标准编码，确保行业通用性 |
| **业务编码** | 通识优先，参考专识 | 采用行业标准编码，企业编码仅作参考 |
| **属性值** | 专识优先 | 采用企业实际业务术语，确保业务准确性 |

## 三、结构编码规则

### 3.1 Object Type 编码规则

**规则**：
- 命名格式：PascalCase，无前缀
- 长度限制：不超过50个字符
- 命名原则：反映业务实体的本质

**油田行业Object Type示例**：
```yaml
Object Type 编码示例:
  # 核心实体（10域覆盖）
  - Well: 井
  - Wellbore: 井筒
  - Reservoir: 油藏
  - Block: 区块
  - Formation: 地层
  - Fluid: 流体

  # 钻井工程域
  - DrillingRig: 钻井设备
  - DrillingBit: 钻头
  - MudSystem: 泥浆系统

  # 完井与改造域
  - WellCompletion: 完井
  - Stimulation: 措施作业
  - Fracturing: 压裂

  # 采油采气生产域
  - WellProduction: 井产量
  - ArtificialLift: 人工举升
  - Pump: 抽油泵

  # 地面集输域
  - GatheringStation: 集输站
  - Pipeline: 管线
  - Tank: 储罐

  # 设备资产域
  - Equipment: 设备
  - EquipmentMaintenance: 设备维护
```

### 3.2 Link Type 编码规则

**规则**：
- 命名格式：PascalCase，无前缀
- 长度限制：不超过50个字符
- 命名原则：反映关系的语义

**油田行业Link Type示例**：
```yaml
Link Type 编码示例:
  # 空间关系
  - locatedIn: 位于
  - contains: 包含
  - connectsTo: 连接

  # 生产关系
  - produces: 生产
  - injects: 注入
  - belongsToBlock: 属于区块

  # 工程关系
  - drilledBy: 由...钻井
  - completedBy: 由...完井
  - stimulatedBy: 由...改造

  # 技术关系
  - hasFormation: 具有地层
  - hasFluid: 含有流体
  - hasCompletion: 具有完井方式
```

### 3.3 物理表编码规则

**规则**：
- 事实表：`fact_*` 前缀 + 业务名称（snake_case）
- 维度表：`dim_*` 前缀 + 业务名称（snake_case）
- 长度限制：不超过50个字符（含前缀）

**油田行业物理表编码示例**：
```yaml
物理表编码示例:
  # 事实表
  - fact_well_production: 井产量事实表
  - fact_well_injection: 井注入事实表
  - fact_equipment_operation: 设备运行事实表
  - fact_stimulation: 措施作业事实表

  # 维度表
  - dim_well: 井维度表
  - dim_block: 区块维度表
  - dim_well_type: 井型维度表
  - dim_reservoir: 油藏维度表
  - dim_formation: 地层维度表
  - dim_fluid: 流体性质维度表
  - dim_company: 作业区维度表
  - dim_date: 日期维度表
```

### 3.4 Cube 编码规则

**规则**：
- 命名格式：PascalCase + `Cube` 后缀
- 长度限制：不超过50个字符
- 命名原则：反映分析主题

**油田行业Cube编码示例**：
```yaml
Cube 编码示例:
  - WellProductionCube: 井产量分析Cube
  - BlockProductionCube: 区块产量分析Cube
  - ArtificialLiftCube: 人工举升分析Cube
  - EquipmentOpecCube: 设备运行分析Cube
  - StimulationCube: 措施效果分析Cube
```

## 四、业务编码规则

### 4.1 层级编码规则

**规则**：
- 采用字母+数字组合或纯数字层级结构
- 编码格式：`[层级编码]-[业务含义]`
- 编码长度：根据业务需要确定

**油田行业业务编码示例**：
```yaml
# 井型编码（通识优先）
井型分类:
  L1: PR-采油井, IN-注水井, MO-观察井, EX-评价井
  L2:
    PR-01: 自喷采油井
    PR-02: 抽油井
    PR-03: 电泵井
    IN-01: 注水井
    IN-02: 注气井

# 区块编码（企业专识优先）
区块分类:
  L1: North-北, South-南, East-东, West-西
  L2: North-A(北A区), North-B(北B区), South-A(南A区)
  L3: North-A-1(北A区1), North-A-2(北A区2)

# 油藏类型编码（通识优先）
油藏类型:
  - OVL-01: 常规稠油油藏
  - OVL-02: 超稠油油藏
  - OSL-01: 常规稀油油藏
  - OGS-01: 气顶油藏
```

### 4.2 指标编码规则

**规则**：
- 采用字母+数字组合
- 编码格式：`[指标类别缩写]-[序号]`
- 编码长度：不超过10个字符

**油田行业指标编码示例**：
```yaml
指标编码示例:
  # 产量类指标
  - PRD-001: 日产油量（Daily Oil Production）
  - PRD-002: 日产液量（Daily Liquid Production）
  - PRD-003: 日产气量（Daily Gas Production）
  - PRD-004: 月度累计产油量（Monthly Cumulative Oil）

  # 注入类指标
  - INJ-001: 日注水量（Daily Water Injection）
  - INJ-002: 月度累计注水量（Monthly Cumulative Water Injection）

  # 效率类指标
  - EFF-001: 含水率（Water Cut）
  - EFF-002: 油气比（Gas-Oil Ratio）
  - EFF-003: 泵效（Pump Efficiency）

  # 设备类指标
  - EQP-001: 设备运行时率（Equipment Running Rate）
  - EQP-002: 设备故障率（Equipment Failure Rate）
```

### 4.3 设备类型编码规则

**规则**：
- 采用字母+数字组合
- 编码格式：`[设备类别缩写]-[序号]`
- 编码长度：不超过10个字符

**油田行业设备类型编码示例**：
```yaml
设备类型编码示例:
  # 举升设备
  - LFT-001: 抽油机（Rod Pump）
  - LFT-002: 电潜泵（ESP）
  - LFT-003: 螺杆泵（PCP）

  # 地面设备
  - GRD-001: 加热炉
  - GRD-002: 分离器
  - GRD-003: 储罐

  # 井下工具
  - WBT-001: 封隔器
  - WBT-002: 油管锚
  - WBT-003: 井下安全阀
```

## 五、属性值规则

### 5.1 属性值优先级

**规则**：
- 属性值采用企业实际业务术语
- 企业术语不清晰时，参考行业标准术语

**油田行业属性值示例**：
```yaml
井型属性值（专识优先）:
  PR-01 → "自喷采油井"（企业术语）
  PR-02 → "抽油井"（企业术语）
  IN-01 → "注水井"（企业术语）

区块属性值（专识优先）:
  North-A → "北A区"（企业术语）
  South-B → "南B区"（企业术语）

油藏类型属性值（专识优先）:
  OVL-01 → "常规稠油油藏"（行业标准术语）
  OSL-01 → "稀油油藏"（行业标准术语）
```

### 5.2 属性值冲突解决

**规则**：
- 企业术语优先
- 企业术语不清晰时，参考行业标准术语

**属性值冲突解决示例**：
```yaml
场景: 企业术语"抽油机井"与行业标准术语"抽油井"冲突
解决: 采用企业术语"抽油机井"，但增加备注说明对应行业标准术语"抽油井"

场景: 企业术语"电泵井"与行业标准术语"电动潜油泵井"冲突
解决: 采用企业术语"电泵井"，但增加备注说明对应行业标准术语"电动潜油泵井"
```

## 六、编码冲突解决策略

### 6.1 冲突类型

| 冲突类型 | 说明 | 示例 |
|---------|------|------|
| **结构编码冲突** | 企业编码与行业标准编码冲突 | 企业用 `WellProduction`，行业标准用 `Well_Production` |
| **业务编码冲突** | 企业编码与行业标准编码冲突 | 企业用 `W01`，行业标准用 `PR-01` |
| **属性值冲突** | 企业术语与行业标准术语冲突 | 企业用"电泵井"，行业标准用"电动潜油泵井" |

### 6.2 解决策略

| 冲突类型 | 解决策略 | 说明 |
|---------|---------|------|
| **结构编码冲突** | 采用行业标准编码 | 确保行业通用性 |
| **业务编码冲突** | 采用行业标准编码，企业编码仅作参考 | 确保行业通用性 |
| **属性值冲突** | 采用企业实际业务术语 | 确保业务准确性 |

### 6.3 冲突解决流程

```
┌─────────────────────────────────────────────────────────────┐
│                     编码冲突解决流程                         │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  步骤1：识别冲突                                             │
│  ├── 识别编码冲突类型                                       │

│  └── 评估冲突影响                                           │
│                          ↓                                  │
│  步骤2：查询行业标准                                         │
│  ├── 查询行业标准编码                                       │
│  ├── 查询行业标准术语                                       │
│  └── 评估行业标准适用性                                     │
│                          ↓                                  │
│  步骤3：确定解决策略                                         │
│  ├── 结构编码冲突 → 采用行业标准编码                        │
│  ├── 业务编码冲突 → 采用行业标准编码，企业编码仅作参考      │
│  └── 属性值冲突 → 采用企业实际业务术语                      │
│                          ↓                                  │
│  步骤4：实施编码调整                                         │

│  └── 增加备注说明                                           │
│                          ↓                                  │
│  步骤5：验证编码一致性                                       │
│  ├── 检查编码规则一致性                                     │
```

## 八、编码管理

### 8.1 编码文档化

**要求**：
- 所有编码必须文档化
- 编码文档必须包含编码规则、示例、说明
- 编码文档必须保持更新

**油田行业编码文档示例**：
```yaml
井型编码文档:
  - 编码类型: 业务编码
  - 编码规则: [类型代码]-[序号]
  - 编码示例:
    - PR-01: 自喷采油井
    - PR-02: 抽油井
    - PR-03: 电泵井
    - IN-01: 注水井
  - 编码说明: 采用石油行业井型分类标准
  - 属性值: 采用企业实际业务术语
  - 冲突解决: 企业编码与行业标准编码冲突时，采用行业标准编码

区块编码文档:
  - 编码类型: 业务编码
  - 编码规则: [方位代码]-[区块序号]
  - 编码示例:
    - North-A: 北A区
    - North-B: 北B区
    - South-A: 南A区
  - 编码说明: 企业内部区块编码规则
  - 属性值: 企业实际区块名称
```

### 8.2 编码变更管理

**要求**：
- 编码变更必须遵循变更流程
- 编码变更必须更新所有相关文档
- 编码变更必须记录变更日志

## 九、总结

### 9.1 核心原则

| 原则 | 说明 |
|------|------|
| **结构编码通识优先** | 结构编码采用行业标准编码 |
| **业务编码通识优先，参考专识** | 业务编码采用行业标准编码，企业编码仅作参考 |
| **属性值专识优先** | 属性值采用企业实际业务术语 |
| **编码冲突解决** | 明确的编码冲突解决策略 |
| **编码文档化** | 所有编码必须文档化 |

### 9.2 关键产出

| 产出 | 说明 |
|------|------|
| **编码规则** | 结构编码、业务编码、属性值的编码规则 |
| **编码示例** | 各类编码的示例 |
| **冲突解决策略** | 编码冲突的解决策略 |
| **编码实施方法** | 编码设计的流程和验证清单 |

### 9.3 下一步

编码标准规范完成后，进入本体设计阶段，将编码规则应用到 Object Type、Link Type 和维度表的设计中。

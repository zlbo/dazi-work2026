# 模拟数据方法

> **版本**：V3.0
> **创建日期**：2026年6月11日
> **更新日期**：2026年6月13日
> **适用范围**：潘达油田-全域生产分析
> **依据规范**：《000-顶层设计.md》（编码标准规范）、《040-本体设计方法.md》（业务规则定义）、《060-规则库建设规划.md》（业务规则）
> **核心定位**：本方法论定义基于本体规则的模拟数据方法，确保模拟数据具有高仿真度、可信度，支撑决策分析场景。

> **重要更新（V3.0）**：根据本项目"无实际数据，全部依赖模拟"的特点，模拟数据工作采用**早期介入策略**，分为两个阶段：
> - **阶段二（Step 5a）**：模拟数据可行性评估，验证设计能否被模拟数据支撑
> - **阶段四（Step 10）**：模拟数据实际生成与验证

## 二、模拟数据核心原则

### 2.1 两个核心维度

> **重要概念**：模拟数据需要同时满足两个核心维度。

| 维度 | 定义 | 衡量标准 |
|------|------|---------|
| **数据质量** | 数据本身的技术正确性 | 业务规则、勾稽关系、取值范围 |
| **业务真实性** | 数据反映真实的业务特征 | 产量波动、含水变化、设备状态 |

### 2.2 模拟数据来源

| 来源 | 优先级 | 说明 |
|------|-------|------|
| **本体规则** | 最高 | 基于本体定义的业务规则 |
| **规则库** | 高 | 基于油田行业规则库 |
| **新疆油田参考** | 中 | 参考新疆油田数据特征 |
| **行业标准** | 低 | 行业统计数据作为参考 |

```yaml
井产量数据模拟规则:
  # 基本参数
  - parameter: 日产油量范围(稀油)
    value: [1, 50] t/d

  - parameter: 日产油量范围(稠油)
    value: [3, 100] t/d
    description: 稠油热采井

  - parameter: 日产液量范围
    value: [10, 500] m³/d
    description: 常规油井

  - parameter: 含水率范围(稀油)
    value: [30%, 95%]
    description: 开发后期上升

  - parameter: 含水率范围(稠油)
    value: [50%, 90%]
    description: 热采开发

  # 业务规则约束
  - rule: 日产油量计算
    formula: 日产油量 = 日产液量 × (1 - 含水率/100)

  - rule: 产量非负
    constraint: 日产油量 ≥ 0
```

## 七、数据验证方法

### 7.1 业务规则验证

```sql
-- 验证1：含水率合理性
SELECT COUNT(*) as invalid_count
FROM fact_well_production
WHERE water_cut < 0 OR water_cut > 100;
-- 期望：0条

-- 验证2：产量非负
SELECT COUNT(*) as invalid_count
FROM fact_well_production
WHERE daily_oil_production < 0;
-- 期望：0条

-- 验证3：液油勾稽关系
SELECT COUNT(*) as invalid_count
FROM fact_well_production
WHERE ABS(daily_liquid_production - daily_oil_production / (1 - water_cut/100)) > 0.5;
-- 期望：0条

-- 验证4：注采比合理性
SELECT COUNT(*) as invalid_count
FROM (
    SELECT block_key, date_key,
           SUM(daily_water_injection) as total_injection,
           SUM(daily_liquid_production) as total_liquid
    FROM fact_well_injection i
    JOIN dim_well w ON i.well_key = w.well_key
    GROUP BY block_key, date_key
) t
WHERE total_injection / total_liquid NOT BETWEEN 0.8 AND 1.5;
-- 期望：0条
```

### 7.2 分布特征验证

```sql
-- 验证1：含水率分布
SELECT 
    CASE 
        WHEN water_cut < 50 THEN '低含水'
        WHEN water_cut < 80 THEN '中含水'
        ELSE '高含水'
    END as water_cut_level,
    COUNT(*) as count,
    ROUND(COUNT(*) * 100.0 / SUM(COUNT(*)) OVER(), 2) as percentage
FROM fact_well_production
GROUP BY 
    CASE 
        WHEN water_cut < 50 THEN '低含水'
        WHEN water_cut < 80 THEN '中含水'
        ELSE '高含水'
    END;

-- 期望：低含水20-30%，中含水30-40%，高含水30-40%
```

## 九、模拟数据工作时机说明（本项目定制）

> **重要说明**：本项目根据"无实际数据"的特点，将模拟数据工作分为两个阶段：

### 9.1 阶段二：模拟数据可行性评估（Step 5a）

| 工作内容 | 产出 | 目的 |
|---------|------|------|
| 验证数据体量合理性 | 评估报告 | 确认500口井、456,000条记录规模可行 |
| 验证数据分布可行性 | 评估报告 | 确认井型、举升方式分布可模拟 |
| 验证约束可达性 | 评估报告 | 确认业务规则可被满足 |
| 识别设计风险 | 风险清单 | 提出设计调整建议 |

### 9.2 阶段四：模拟数据生成与验证（Step 10）

| 工作内容 | 产出 | 依据 |
|---------|------|------|
| 基于可行性评估生成数据 | 模拟数据集 | 《模拟数据可行性评估.md》 |
| 执行数据验证 | 验证报告 | 本文档第七章验证方法 |
| 修复验证不通过项 | 修复记录 | - |

### 9.3 两个阶段的关系

```
┌─────────────────────────────────────────────────────────────┐
│                    阶段二（Step 5a）                         │
│              模拟数据可行性评估                              │
│  ┌─────────────────────────────────────────────────────┐   │
│  │ 输入：本体数据架构设计                                │   │
│  │ 处理：验证体量、分布、约束可行性                      │   │
│  │ 输出：《模拟数据可行性评估.md》                        │   │
│  └─────────────────────────────────────────────────────┘   │
│                           ↓                                 │
│  ┌─────────────────────────────────────────────────────┐   │
│  │ 如评估不通过：返回阶段二调整设计                       │   │
│  │ 如评估通过：进入阶段三                                │   │
│  └─────────────────────────────────────────────────────┘   │
└─────────────────────────────────────────────────────────────┘
                           ↓
┌─────────────────────────────────────────────────────────────┐
│                    阶段四（Step 10）                        │
│              模拟数据生成与验证                              │
│  ┌─────────────────────────────────────────────────────┐   │
│  │ 输入：《模拟数据可行性评估.md》                        │   │
│  │ 处理：生成维度表数据 → 生成事实表数据                  │   │
│  │ 输出：《模拟数据生成报告.md》                          │   │
│  └─────────────────────────────────────────────────────┘   │
└─────────────────────────────────────────────────────────────┘
```

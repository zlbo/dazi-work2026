## 一、规划概述

### 1.1 目标定位

本规划旨在建立一套**面向油气田生产企业**的量化规则体系，实现以下目标：

| 目标维度 | 具体目标 |
|---------|---------|
| **数据模拟** | 提供数据生成的约束条件和取值范围，确保模拟数据符合业务常识 |
| **数据质量** | 作为数据质量校验规则，保障业务数据的准确性和一致性 |
| **智能决策** | 为智能决策引擎提供核心规则支撑 |
| **知识沉淀** | 将业务经验转化为可执行的量化规则 |

## 三、详细规则定义

### 3.1 采油生产规则

#### 3.1.1 产量计算规则

```yaml
【规则编号】PRD001
【规则名称】日产油量计算规则
【规则类别】采油生产规则 → 产量计算规则
【适用实体】WellProduction
【适用场景】产量计算、报表生成
【量化公式】日产油量 = 日产液量 × (1 - 含水率)
【参数定义】
  - 日产液量：来自fact_well_production.daily_liquid_production
  - 含水率：来自fact_well_production.water_cut
【阈值定义】日产油量 < 0.5 吨/天 时触发低产预警
【预警级别】黄色预警
【业务依据】油田开发技术规范
【数据来源】fact_well_production
【平台映射】派生属性（计算属性）
```

```yaml
【规则编号】PRD002
【规则名称】月度累计产量计算规则
【规则类别】采油生产规则 → 产量计算规则
【适用实体】WellProduction
【适用场景】月度报表、产量统计
【量化公式】月度累计产油 = Σ(日产油量 × 当月天数)
【参数定义】
  - 日产油量：日度产量数据
  - 当月天数：月度日历天数
【阈值定义】月度产量低于配产产量90%时触发欠产预警
【预警级别】红色预警
【业务依据】油田生产计划
【数据来源】fact_well_production, dim_date
【平台映射】派生属性（聚合属性）
```

```yaml
【规则编号】PRD003
【规则名称】产量递减率计算规则
【规则类别】采油生产规则 → 产量计算规则
【适用实体】WellProduction
【适用场景】油田开发分析、储量评估
【量化公式】递减率 = (Q1 - Q2) / Q1 / Δt
【参数定义】
  - Q1：上月产量
  - Q2：本月产量
  - Δt：时间间隔（月）
【阈值定义】递减率 > 5%/月 时触发快速递减预警
【预警级别】红色预警
【业务依据】油田开发动态分析
【数据来源】fact_well_production
【平台映射】派生属性（趋势属性）
```

### 3.2 注水开发规则

#### 3.2.1 注采平衡规则

```yaml
【规则编号】INJ001
【规则名称】注采比计算规则
【规则类别】注水开发规则 → 注采平衡规则
【适用实体】Block, WellProduction
【适用场景】区块开发分析、注水效果评价
【量化公式】注采比 = 月度注水量 / (月度产液量 × 地层体积系数)
【参数定义】
  - 月度注水量：Σ(各注水井日注量 × 当月天数)
  - 月度产液量：Σ(各油井日产液量 × 当月天数)
  - 地层体积系数：通常取1.0-1.2
【阈值定义】
  - 注采比 < 0.8：欠注预警（红色）
  - 注采比 > 1.5：超注预警（黄色）
【预警级别】根据阈值确定
【业务依据】注水开发技术规范
【数据来源】fact_well_injection, fact_well_production, dim_block
【平台映射】派生属性（计算属性）
```

```yaml
【规则编号】INJ002
【规则名称】注水井配注量计算规则
【规则类别】注水开发规则 → 注入量计算规则
【适用实体】WellInjection
【适用场景】注水井管理、配注方案制定
【量化公式】配注量 = 区块目标注采比 × 区块产液量 / 注水井数
【参数定义】
  - 区块目标注采比：0.8-1.2
  - 区块产液量：区块内所有油井产液量之和
  - 注水井数：区块内注水井数量
【阈值定义】实际注水量偏离配注量 ±20% 时触发调整预警
【预警级别】黄色预警
【业务依据】注水开发方案
【数据来源】dim_block, fact_well_injection
【平台映射】派生属性（计算属性）
```

#### 3.2.2 注入压力规则

```yaml
【规则编号】INJ003
【规则名称】注入压力合理性检验规则
【规则类别】注水开发规则 → 压力控制规则
【适用实体】WellInjection
【适用场景】数据质量校验、压力监控
【量化公式】校验：0 ≤ 注入压力 ≤ 地层破裂压力
【参数定义】
  - 注入压力：来自fact_well_injection.injection_pressure
  - 地层破裂压力：根据区块地质条件确定
【阈值定义】
  - 注入压力 < 0 MPa：数据异常
  - 注入压力 > 地层破裂压力：超压预警（红色）
【预警级别】红色预警
【业务依据】注水安全操作规程
【数据来源】fact_well_injection, dim_block
【平台映射】数据质量规则 + 业务约束规则
```

### 3.3 增产措施规则

#### 3.3.1 措施选井规则

```yaml
【规则编号】STK001
【规则名称】压裂选井条件规则
【规则类别】增产措施规则 → 措施选井规则
【适用实体】Well, WellProduction
【适用场景】压裂措施方案制定
【量化公式】综合评价指数 = w1×含水率 + w2×累计产量 + w3×地层系数
【参数定义】
  - 含水率：当前含水率（目标 < 70%）
  - 累计产量：历史累计产油量（目标 > 5000吨）
  - 地层系数：储层渗透率×有效厚度（目标 > 50md·m）
  - w1, w2, w3：权重系数
【阈值定义】综合评价指数 < 0.5 时不推荐压裂
【预警级别】信息提示
【业务依据】压裂选井技术规范
【数据来源】fact_well_production, dim_well, dim_block
【平台映射】智能推荐引擎
```

```yaml
【规则编号】STK002
【规则名称】酸化选井条件规则
【规则类别】增产措施规则 → 措施选井规则
【适用实体】Well, WellProduction

【量化公式】酸化适应性指数 = w1×酸敏性 + w2×孔隙度 + w3×渗透率
【参数定义】

  - 孔隙度：储层孔隙度（目标 > 10%）

【阈值定义】酸化适应性指数 < 0.4 时不推荐酸化
【预警级别】信息提示
【业务依据】酸化选井技术规范
【数据来源】dim_well, dim_block

【规则编号】EQP001

【预警级别】根据阈值确定

#### 3.4.2 设备维护周期规则

【规则编号】MNT001
【规则名称】抽油机保养到期预警规则
【规则类别】设备运行规则 → 设备维护周期规则
【适用实体】Equipment, EquipmentMaintenance
【适用场景】设备预防性维护
【量化公式】保养到期天数 = 计划保养日期 - 当前日期
【参数定义】
  - 计划保养日期：根据上次保养日期+保养周期计算
  - 当前日期：系统当前日期
【阈值定义】
  - 保养到期天数 < 0：已过期（红色）
  - 保养到期天数 0-7：即将到期（黄色）
  - 保养到期天数 7-30：计划提醒（蓝色）
【预警级别】根据阈值确定
【业务依据】设备维护保养规程
【数据来源】dim_equipment, fact_equipment_operation
【平台映射】派生属性（时间属性）+ Action Type（预警通知）
```

### 4.1 产量勾稽规则

```yaml
【规则编号】CR001
【规则名称】日产液量与日产油量勾稽规则
【规则类别】数据勾稽规则 → 指标间勾稽
【适用实体】WellProduction
【适用场景】数据一致性校验
【量化公式】日产液量 = 日产油量 / (1 - 含水率)
【容差范围】±0.5%
【验证方法】比较计算值与实际记录值的偏差
【预警级别】偏差 > 0.5% 时触发数据异常预警
【业务依据】生产数据逻辑关系
【数据来源】fact_well_production
【平台映射】数据质量规则
```

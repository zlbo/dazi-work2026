## 二、项目背景

### 2.1 业务背景

潘达油田需要建立一套**本体驱动的智能决策体系**，用于全域生产数据分析。该体系将：
- 构建油田生产领域的完整本体网络，展现业务全景
- 支持基于本体的灵活问数和智能问答
- 提供生产监控可视化看板和驾驶舱
- 支撑产量分析、设备预警、油藏管理等深度分析场景

### 2.2 平台架构说明

**重要说明**：本项目基于**本体驱动的智能决策平台**构建，该平台内置了以下核心能力：

| 平台能力         | 说明                                    | 对本项目的影响                |
| ------------ | ------------------------------------- | ---------------------- |
| **自动Cube生成** | 平台可将数据表自动转换为Cube分析视图                  | 无需手动设计Cube，阶段一输出转换规则即可 |
| **本体层管理**    | 支持Object Type、Link Type、Action Type定义 | 阶段二专注于本体语义设计           |
| **函数引擎**     | 支持Function和Action的开发和注册               | 阶段三专注于业务逻辑封装           |
| **智能问答**     | 支持基于本体的自然语言问答                         | 最终交付的AI能力              |

**四层架构**：

```
┌──────────────────────────────────────────────────────────────┐
│                         应用层（面向用户）                     │
│  AI助手问答 │ Web报表 │ 可视化看板 │ 驾驶舱 │ 分析报告          │
├──────────────────────────────────────────────────────────────┤
│                         函数层（Function/Action）              │
```

## 三、核心策略：相对理想型

**这是本项目的核心指导思想，必须贯穿所有阶段**：

| 维度      | 说明                                      |
| ------- | --------------------------------------- |
| **理想化** | 本体架构和数据架构应该支撑油田全域生产分析的**完整、科学、面向未来**的状态 |
| **相对**  | 理想架构必须基于**现实可获得的数据源**，不能脱离实际            |
| **平衡**  | 在理想设计与现实约束之间寻求平衡                        |

### 3.1 油田行业适配原则

> **引用**：《000-顶层设计.md》§1.4「油田行业特征适配原则」

| 适配维度 | 说明 |
|---------|------|
| **业务链长** | 勘探→钻井→完井→采油→集输，需设计跨域关联关系 |
| **核心实体贯穿** | "井"作为核心实体贯穿7个业务域 |
| **数据类型多样** | 实时传感器、地质数据、设备数据、生产数据需融合 |
| **空间分布广** | 需要空间维度和区域管理 |

## 1. 开发前准备（必须做）

### 1.1 确认三个维度的信息

| 维度 | 关键信息 | 从哪里查 |
|------|---------|---------|
| **数据源** | 源表名（如 `tb_project_income_outcome_summary`）、字段清单、`connectionId`（如 `duckdb__建工成本01`） | `资源/datasources/<连接名>/<表名>.md` |
| **本体目标** | 数据空间 ID（如 `space__panda_construction`）、目标表名（如 `fact_project_output`）、字段清单 | `资源/dataspaces/<空间名>/<表名>.md` |
| **命名规范** | 流程名格式：`flow-<业务>数据同步`（如 `flow-付款数据同步`）、节点语义ID规范 | 查看现有流程（如 `flow-成本数据同步`） |

### 1.2 查阅已完成的相似流程

**强烈推荐**：从已成功运行的流程中「复制」结构，不要从零创建！

## 2. 流程创建与初始化（标准步骤）

### 2.1 正确的流程创建顺序

```powershell
# 1. 平台新建流程（不要手动创建目录！）
dazi flow flows create --name "付款数据同步"

# 2. 记下返回的 flowId（如 137），拉取到本地
dazi flow project pull --flow 137 --dir "d:\GitHub\dazi-work\项目\潘达工程-商务成本\流程\flows\flow-付款数据同步"

# 3. 进入流程目录，用已有流程的 flow.json 替换（调整节点ID）
#    保持 start/end 节点不变，把中间节点替换成自己需要的
```

### 2.2 标准流程节点链

**本体数据接入流程的节点链固定为以下顺序**：

```mermaid
start --> database-source --> python-script(数据转换) --> python-script(质量检查) --> dataspace-sink --> end
```

各节点语义ID命名规范：
- `database-source-<业务>`（如 `database-source-payment`）
- `python-transform-<业务>`（如 `python-transform-payment`）
- `python-quality-<业务>`（如 `python-quality-payment`）
- `dataspace-sink-<业务>`（如 `dataspace-sink-payment`）

## 3. 节点配置与代码编写（核心规范）

### 3.1 database-source（读取源数据）

#### 配置要点（绝对不能错！）

| 字段 | 值规范 | 错误示例 | 正确示例 |
|------|--------|---------|---------|
| `data.type` | 固定为 `"database-source"` | `dataSource` | `"database-source"` |

### 3.2 python-script（数据转换）

#### 核心原则（必守）

1. **只消费上游一个变量**：`df = get_variable("<上游变量名>")`
2. **所有值转换为 Python 原生类型**（不要留 numpy 类型！）
3. **逐行处理，避免 `df.apply` 等整体操作的类型问题**
4. **`result_df[col] = df_result[col].tolist()`**：最后强制转换所有列
5. **目标字段严格对应本体表字段顺序和名称**

#### 代码模板（复制即用）

```python
# -*- coding: utf-8 -*-
# <业务>同步 - 数据转换节点
# output_variable_name = <业务>_data

import pandas as pd
import numpy as np

output.print("[<业务>同步] 开始数据转换")

# 1. 获取上游数据
df = get_variable("<上游变量名>_raw_data")
output.print(f"输入数据 shape={df.shape}")
output.print(f"输入列: {list(df.columns)}")

# 2. 定义目标表字段（严格按本体表顺序）
target_fields = [
    "payment_id", "date_key", "contract_id", "contract_name",
    "contract_amount", "settlement_status", "supplier_id",
    "report_period", "payable_confirmed", "payable_unconfirmed",
    "labor_payable", "approval_status", "approval_amount",
    "paid_amount", "tax_rate", "payment_ratio", "project_id",
    "created_at", "supplier_name", "contract_code"
]

# 3. 创建结果列表，逐行处理（关键！保证类型正确）
result_rows = []
current_time = pd.Timestamp.now().to_pydatetime()

for _, row in df.iterrows():
    # 计算派生字段（如 date_key、report_period）
    try:
        year = int(row["year"]) if pd.notnull(row["year"]) else 2025
        month = int(row["month"]) if pd.notnull(row["month"]) else 1
        day = 31 if month in [1,3,5,7,8,10,12] else (29 if (month==2 and (year%4==0 and year%100!=0 or year%400==0)) else (28 if month==2 else 30))
        date_key = year * 10000 + month * 100 + day
        report_period = f"{int(year)}-{int(month):02d}"
    except:
        date_key = 0
        report_period = ""

    # 确保所有值为 Python 原生类型（关键！）
    payment_id = str(row["id"]) if pd.notnull(row["id"]) else ""
    project_id = str(row["project_id"]) if pd.notnull(row["project_id"]) else ""
    contract_id = str(row["contract_id"]) if pd.notnull(row["contract_id"]) else ""
    contract_name = str(row["base_contract_name"]) if pd.notnull(row["base_contract_name"]) else ""
    contract_amount = float(row["base_contract_amount"]) if pd.notnull(row["base_contract_amount"]) else 0.0
    paid_amount = float(row["pay_amount_acc"]) if pd.notnull(row["pay_amount_acc"]) else 0.0

    # 构建行
    result_row = {
        "payment_id": payment_id,
        "date_key": int(date_key),
        "contract_id": contract_id,
        "contract_name": contract_name,
        "contract_amount": contract_amount,
        "settlement_status": "",
        "supplier_id": "",
        "report_period": report_period,
        "payable_confirmed": paid_amount,
        "payable_unconfirmed": 0.0,
        "labor_payable": 0.0,
        "approval_status": "",
        "approval_amount": 0.0,
        "paid_amount": paid_amount,
        "tax_rate": 0.09,
        "payment_ratio": 0.0,
        "project_id": project_id,
        "created_at": current_time,
        "supplier_name": "",
        "contract_code": str(row["base_contract_code"]) if pd.notnull(row["base_contract_code"]) else ""
    }
    result_rows.append(result_row)

# 4. 转换为 DataFrame
df_result = pd.DataFrame(result_rows, columns=target_fields)

# 5. 强制转换所有列为 Python 原生类型（关键！）
result_df = pd.DataFrame()
for col in df_result.columns:
    result_df[col] = df_result[col].tolist()

output.print(f"输出 shape={result_df.shape}")
output.print(f"输出列: {list(result_df.columns)}")
output.print("[<业务>同步] 数据转换完成")
```

### 3.3 python-script（质量检查）

#### 核心原则

1. **必填字段校验**：主键、业务关键字段（如 `project_id`、`report_period`）非空
2. **格式校验**：`date_key` 必须是 8 位数字
3. **同样最后强制转换为 Python 原生类型**
4. **即使只有警告也让数据通过，在日志中记录**

#### 代码模板

```python
# -*- coding: utf-8 -*-
# <业务>同步 - 质量检查节点
# output_variable_name = validated_<业务>_data

import pandas as pd

output.print("[<业务>同步] 开始数据质量检查")

# 获取上游数据
df = get_variable("<业务>_data")
output.print(f"输入数据 shape={df.shape}")

errors = []
warnings = []

# 1. 必填字段检查
required_fields = ["<业务>_id", "project_id", "report_period"]
for field in required_fields:
    if field in df.columns:
        missing = df[field].isnull().sum()
        if missing > 0:
            errors.append(f"必填字段 {field} 有 {missing} 个空值")
            df = df[~df[field].isnull()]

# 2. 日期格式检查
if "date_key" in df.columns:
    invalid_date = df[~df["date_key"].astype(str).str.match(r"^\d{8}$")]
    if len(invalid_date) > 0:
        errors.append(f"date_key 格式错误: {len(invalid_date)} 条记录")

# 3. 完整性统计
total_input = len(df)
total_output = len(df)
rejection_rate = 0.0

# 4. 输出报告
output.print("=" * 50)
output.print("数据质量检查报告")
output.print("=" * 50)
output.print(f"输入记录数: {total_input}")
output.print(f"输出记录数: {total_output}")
output.print(f"剔除记录数: {total_input - total_output}")
output.print(f"剔除率: {rejection_rate:.2f}%")
if errors:
    output.print("错误列表:")
    for error in errors:
        output.print(f"  - {error}")
if warnings:
    output.print("警告列表:")
    for warning in warnings:
        output.print(f"  - {warning}")
if not errors and not warnings:
    output.print("✓ 所有数据质量检查通过")
output.print("=" * 50)

# 5. 再次确保输出是 Python 原生类型
result_rows = []
for _, row in df.iterrows():
    result_row = {}
    for col in df.columns:
        val = row[col]
        if hasattr(val, 'dtype'):
            if val.dtype == 'float64':
                result_row[col] = float(val)
            elif val.dtype == 'int64':
                result_row[col] = int(val)
            else:
                result_row[col] = val
        elif isinstance(val, float):
            result_row[col] = float(val)
        elif isinstance(val, int):
            result_row[col] = int(val)
        elif pd.isnull(val):
            if col in ["date_key"]:
                result_row[col] = 0
            elif col.endswith(("_id", "_name", "_status", "_code", "_period")):
                result_row[col] = ""
            else:
                result_row[col] = 0.0
        else:
            result_row[col] = val
    result_rows.append(result_row)

df_result = pd.DataFrame(result_rows)
result_df = pd.DataFrame()
for col in df_result.columns:
    result_df[col] = df_result[col].tolist()

output.print("[<业务>同步] 数据质量检查完成")
```

## 4. 测试与验证（一次性通过检查点）

### 4.1 修改后的步骤（绝对顺序！）

```
改 code.sql / code.py / flow.json
    ↓
先 repair-meta
    ↓
然后 project push --canvas（如果改了 flow.json）
    ↓
然后 node push（如果改了 code.*）
    ↓
然后 node-exec（平台执行）
    ↓
然后 variable pull（验证输出）
```

### 4.2 CLI 命令速查

```powershell
# 0. 进入流程目录
cd "d:\GitHub\dazi-work\项目\潘达工程-商务成本\流程\flows\flow-付款数据同步"

# 2. 状态检查
```

### 4.3 一次性通过检查清单

| 检查项 | ✅ 通过条件 |
|--------|-----------|
| **0. 开发前** | 已查源表文档、已查目标表文档、已看相似流程模板 |
| **1. flow.json** | 节点顺序正确、`connectionId` 是字符串、`spaceId` 正确、`output_variable_name` 链式传递 |
| **2. database-source** | `code.sql` 用的是真实源表名、字段列表与源表匹配、有 `LIMIT 200000` |
| **3. python-script** | 用 `get_variable()` 消费上游、最后用 `tolist()` 转换所有列、字段严格对应目标表 |

| **5. node-exec** | 三个节点（读取、转换、质检）都返回 `success: true` |
| **6. variable pull** | `变量/<名>.json` 存在、`status: "ready"`、`row_count` 合理 |

## 5. 常见错误与避坑指南

### 5.1 配置错误类（占比 60%）

| 错误现象 | 原因 | 解决办法 |
|---------|------|---------|
| `404 Not Found GET /.../flows/1003/debug-run` | 流程 ID 在平台不存在、旧流程被删后还在用旧目录 | **重新按正确流程创建**：`flows create` → `project pull` 得到新目录 |
| `Catalog Error: Table with name xxx does not exist!` | `code.sql` 里表名写错了、或者源表名确实不是这个 | **查 `资源/datasources/<连接>/` 下的真实表名** |
| `connectionId 相关错误` | 用了 `dataSourceId: 11`（数字） | **改成 `connectionId: "duckdb__xxx"`（字符串）** |

### 5.2 类型错误类（占比 30%）

| 错误现象 | 原因 | 解决办法 |
|---------|------|---------|
| `'numpy.float64' object has no attribute...` 或写入数据空间失败 | Python 节点输出里有 numpy 原生类型 | **严格执行 `tolist()` 转换！**（看§3.2模板第4步） |
| `date_key` 写入失败 | 不是 8 位整数格式 | 确保代码中 `date_key: int(date_key)` |

### 5.3 流程错误类（占比 10%）

| 错误现象 | 原因 | 解决办法 |
|---------|------|---------|
| `node-exec` 测不到刚改的代码 | `node-exec` 执行的是**平台代码**，本地没 `push` | **改 code 后必须先 `node push`**！ |
| `flow.meta.json` 索引不对 | 目录结构变了但没更新 | **每次增删节点、重命名目录后运行 `repair-meta`** |

## 总结：一次性交付成功的 7 个关键点

1. **不要手工创建目录/流程**，用 `dazi flow flows create` + `project pull`
2. **复制已成功流程**作为模板，不要从零开始
3. **绝对不要混用 `dataSourceId`（数字）和 `connectionId`（字符串）**
4. **Python 节点最后必须 `tolist()` 强制转换所有列**
5. **严格按 `repair-meta` → `push` → `node-exec` → `variable pull` 顺序**
6. **测试阶段**：先单独测读取，再测转换，再测质检，最后跑整流程
7. **随时查阅文档**：本文档 + `local-files-spec.md` + `node-code-guide.md`

**祝你下一个流程一次性通过！** 🚀

### 1.1 业务背景

**搭子建设集团**工程项目利润分析场景：在已有「项目成本 + 产值（营收代理）」数据基础上，建立 **损益科目体系** 与 **利润分析本体**，以业务语言回答：

- **营收**：项目产值/合同收入是多少？按片区、组织、项目如何分布？
- **成本与科目**：成本按 **作业成本科目**（人工/材料/…）与 **损益会计科目**（收入/营业成本/期间费用）如何展开？
- **利润**：项目毛利、组织利润、集团利润；利润率走势
- **预算执行**：成本预算、营收预算与实际对比，执行率与偏差

### 1.2 分析目标（核心问题）

| 问题 | 说明 |
| --- | --- |
| 利润是多少 | 期间/项目/组织的营收 − 成本 = 毛利/利润 |
| 营收从哪来 | 项目产值、片区、组织贡献 |
| 成本结构如何 | 成本科目树 + 映射到损益科目 |
| 预算执行如何 | 成本/营收预算 vs 实际，偏差与执行率 |
| 哪些项目/组织异常 | Top N 利润/亏损项目，超预算预警 |
| 趋势如何 | 按月利润、营收、成本同比/环比 |

### 1.3 规划范围

**复用表（ensure，不重复建表）**：

| --- | --- | --- |

| `fact_cost` | 事实表 | 项目成本流水（含行级 `budget_amount`） |
| `fact_output` | 事实表 | 项目产值流水（**营收口径**） |

**本域新建表**：

| 表名 | 平台分类 | 显示名 | 说明 |
| --- | --- | --- | --- |

| 维度 | 内置 [利润示例](../../../../../../资源/examples/onto/利润示例/plans/规划示例_利润分析本体方案.md) | 同空间 [引擎测试](../引擎测试/) | **本方案（利润分析01）** |

### 3.5 表间关系设计（物理层）

**关系模型**：

```text
                         dim_date
                        (date_key PK)
           ┌──────────────┼──────────────┐
           │              │              │
    fact_output      fact_cost     fact_pl_budget
    (产值/营收)       (成本)          (利润预算)
```

## 四、Cube 层设计

### 4.0 Cube 清单

| Cube | 平台分类 | 类别 | 事实源 | 时间维 | 支撑对象 |
| --- | --- | --- | --- | --- | --- |
| **OutputCube** | 流程型 | Process | fact_output | dim_date | RevenueRecord, ProfitAnalysis |
| **CostCube** | 流程型 | Process | fact_cost | dim_date | CostRecord, ProfitAnalysis |
| **AccountCostCube** | 主体型 | Subject | fact_cost + bridge + dim_account | dim_date | Account |
| **BudgetPlCube** | 流程型 | Process | fact_pl_budget | dim_date + fiscal_* | BudgetLine |
| **ProjectProfitCube** | 对比型 | Comparison | fact_output + fact_cost（同 project×period JOIN） | dim_date | ProfitAnalysis, Project |
| **BudgetVsActualCube** | 对比型 | Comparison | fact_pl_budget + 实际聚合 | fiscal_year/period | BudgetAnalysis |

> **OutputCube / CostCube**：若引擎测试已注册，本 init **ensure 度量与维度一致** 或 skip 重复注册；规划验收以侧栏 Cube 列表可查询为准。

### 4.1 OutputCube（营收 · Process）

**事实源**：`fact_output`

**维度**：date_key, year, month, project_id, project_name, region, org_id, org_name

**度量**：

| 度量名 | 列名 | 聚合 | 说明 |
| --- | --- | --- | --- |
| revenue_total | output_amount | sum | 营收合计 |
| output_count | output_id | count | 产值行数 |

**派生度量**：

| 度量名 | 表达式（示意） | 说明 |
| --- | --- | --- |
| avg_revenue | revenue_total / output_count | 单行均值 |

### 4.2 CostCube（成本 · Process）

**事实源**：`fact_cost`

**维度**：date_key, year, month, project_id, project_name, region, org_id, org_name, cost_type_id, cost_type_name, parent_cost_type_id

**度量**：

| 度量名 | 列名 | 聚合 | 说明 |
| --- | --- | --- | --- |
| cost_total | cost_amount | sum | 成本合计 |
| cost_budget_total | budget_amount | sum | 成本行级预算 |
| cost_count | cost_id | count | 成本行数 |

**派生度量**：

| 度量名 | 表达式（示意） | 说明 |
| --- | --- | --- |
| cost_budget_variance | cost_total - cost_budget_total | 成本预算偏差 |
| cost_exec_rate | if(cost_budget_total>0, cost_total/cost_budget_total, 0) | 成本执行率 |

### 4.3 AccountCostCube（损益科目 · Subject）

**目的**：`Account` 对象 bind_source；通过 bridge 将成本汇总到损益科目。

**维度**：account_id, account_code, account_name, account_type, pl_category, account_level, project_id, org_id, region, year, month

**度量**：cost_total（sum cost_amount）, cost_count

**派生度量**：按 account_type 切片 `cost`, `expense`（费用类预留）

### 4.4 BudgetPlCube（利润预算 · Process）

**事实源**：`fact_pl_budget`

**维度**：budget_version, fiscal_year, fiscal_period, date_key, project_id, org_id, region, account_id, account_code, account_type, pl_category

**度量**：budget_amount（sum）, budget_lines（count）

**派生度量**：

| 度量名 | 表达式（示意） | 说明 |
| --- | --- | --- |
| budget_revenue | sumIf(budget_amount, account_type='收入') | 营收预算 |
| budget_cost | sumIf(budget_amount, account_type='成本') | 成本预算 |
| budget_expense | sumIf(budget_amount, account_type='费用') | 费用预算 |

### 4.5 ProjectProfitCube（项目利润 · Comparison）

**目的**：`ProfitAnalysis` 主读模型；同项目同期间对齐营收与成本。

**维度**：date_key, year, month, fiscal_year, fiscal_period, project_id, project_name, region, org_id, org_name

**度量**（函数/SQL 或 seed 物化视图实现）：

| 度量名 | 说明 |
| --- | --- |
| revenue | 来自 OutputCube 聚合 |
| cost | 来自 CostCube 聚合 |
| gross_profit | revenue - cost |
| profit_margin | if(revenue>0, gross_profit/revenue, 0) |

### 4.6 BudgetVsActualCube（预实对比 · Comparison）

**维度**：fiscal_year, fiscal_period, budget_version, project_id, org_id, account_id, pl_category

**度量**：budget_amount, actual_amount, variance, execution_rate

**actual_amount 口径**：

- 收入类：`fact_output` 按项目×期间汇总
- 成本类：`fact_cost` 经 bridge 汇总到 account_id
- 费用类：首版可为 0 或 fact_pl_budget 对应 actual 扩展

**函数 ID 前缀**：`profit01.fn.*`（与本体域 code 一致）

### 6.1 函数清单

| 函数 ID | 平台分类 | 功能 | 主要参数 |
| --- | --- | --- | --- |
| profit01.fn.get_summary | 总览分析 | 利润总览（营收/成本/毛利/利润率） | start_date, end_date, org_id（可选） |
| profit01.fn.project_profit | 结构分析 | 项目利润排行 | start_date, end_date, limit, region（可选） |
| profit01.fn.account_breakdown | 结构分析 | **损益科目结构** | start_date, end_date, account_level, pl_category（可选） |
| profit01.fn.cost_type_breakdown | 结构分析 | **成本科目结构** | start_date, end_date, project_id（可选） |
| profit01.fn.budget_vs_actual | 预实分析 | 预算 vs 实际 | fiscal_year, fiscal_period, budget_version, project_id（可选） |
| profit01.fn.region_profit | 组织分析 | 片区利润对比 | start_date, end_date |
| profit01.fn.org_profit | 组织分析 | 组织利润树 | start_date, end_date, org_level（可选） |

### 6.2 函数示例：profit01.fn.get_summary

**功能**：期间损益总览

**返回值**：

| 字段名 | 类型 | 说明 |
| --- | --- | --- |
| total_revenue | Float64 | 总营收（产值合计） |
| total_cost | Float64 | 总成本 |
| gross_profit | Float64 | 毛利 |
| profit_margin | Float64 | 利润率 |
| project_count | Int | 有数据项目数 |

**SQL 要点**：`sum(fact_output.output_amount)` − `sum(fact_cost.cost_amount)`，按 `date_key` 范围过滤。

### 6.3 函数示例：profit01.fn.account_breakdown

**功能**：按损益科目层级展开成本/预算结构

**返回值**：account_code, account_name, account_level, pl_category, cost_amount, budget_amount, share_pct

### 6.4 函数示例：profit01.fn.budget_vs_actual

**参数**：fiscal_year, fiscal_period（0=全年）, budget_version, project_id（可选）

**返回值**：project_name, account_code, pl_category, budget_amount, actual_amount, variance, execution_rate

## 七、实施计划

### 7.1 阶段一：物理层

| 步骤 | 任务 |
| --- | --- |
| 1 | ensure `dim_date` / `dim_org` / `dim_project` / `dim_cost_type` / `fact_cost` / `fact_output` |
| 2 | 新建 `dim_account`、`bridge_cost_type_account`、`fact_pl_budget` |
| 3 | `TABLE_REGISTRY` + `register_with_meta` |
| 4 | 注册 **7 条** 新表间关系（§3.5） |

### 7.2 阶段二：Cube 层

| 步骤 | 任务 |
| --- | --- |
| 5 | ensure/register OutputCube、CostCube |
| 6 | 注册 AccountCostCube、BudgetPlCube、ProjectProfitCube、BudgetVsActualCube |
| 7 | 派生度量：gross_profit、profit_margin、budget_revenue/cost、variance |

### 7.3 阶段三：本体层

| 步骤 | 任务 |
| --- | --- |
| 8 | define_object_type（增量/新建 8–9 种） |
| 9 | bind_source → Cube |
| 10 | define_property + define_link_type（10 链接） |
| 11 | sync_metric_refs |

### 7.4 阶段四：灌数、函数与分类

| 步骤 | 任务 |
| --- | --- |
| 12 | seed：dim_account + bridge + fact_pl_budget（与现有 output/cost 同项目同期间） |
| 13 | 7 个 `profit01_fn_*.py` publish + `--register-function-id` |
| 14 | `function run` + `save-test-arguments` |
| 15 | `profit01_category_mount.py` 最后执行 |

**seed 数据建议**：

- 损益科目：一级 3–4 个，二级 8–12 个
- bridge：覆盖现有 cost_type 全集
- fact_pl_budget：6 项目 × 12 期 × 主要科目；营收预算与 output 同量级，成本预算与 fact_cost.budget_amount 可勾稽

## 八、风险与验收

### 8.1 风险

| 等级 | 风险 | 缓解 |
| --- | --- | --- |
| 中 | 与引擎测试对象 code 冲突 | 域隔离；增量 define；实施前 `onto space get` 核对 |
| 中 | 营收口径（产值≠财务收入） | 文档与属性 alias 标明；后续可接 GL |
| 低 | ProjectProfitCube 跨 fact JOIN | 函数内 SQL 聚合；必要时 seed 物化宽表 |
| 低 | 期间费用无 actual 源 | 首版预算侧预留；v2 扩展 fact_gl_entry |

### 8.2 验收标准

| 验收项 | 标准 |
| --- | --- |
| dim_account | 科目树可解析 |
| bridge_cost_type_account | 成本科目 100% 映射 |
| fact_pl_budget | 与 output/cost 同维可 JOIN |
| 表间关系 | 新建 **7 条** + 复用表可 JOIN |
| Cube | **6 个**（含 2 ensure + 4 新建） |
| 对象 + 链接 | ≥8 对象，10 链接 |
| 函数 | 7 个 `profit01.fn.*` 可 run |
| 数据 | get_summary 毛利 = 营收 − 成本，与手工 SQL 一致 |

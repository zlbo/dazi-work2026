## 二、需求分析

### 2.1 业务背景

围绕**库存与仓储**域构建本体，将 SKU 主数据、仓库、库存快照与出入库流水统一为可查询、可分析、可对话的业务对象，支撑：

- 按 SKU、品类、仓库、批次/库位等多维度库存查询与汇总
- 现存量、可用量、在途量、安全库存与缺货预警
- 出入库流水追溯与周转率、库龄分析
- 呆滞/超储识别与补货建议（规则 + 函数输出）

### 2.2 业务域与边界

| 范围内                               | 范围外（本期不做）           |
| ------------------------------------ | ---------------------------- |
| SKU/物料主数据（可与销售产品对齐）   | 采购订单审批、供应商结算     |
| 仓库/库区/库位主数据                 | WMS 拣货路径、波次调度       |
| 库存快照（日/时点）与实时余额逻辑    | 财务存货计价、总账凭证       |
| 出入库流水（入/出/调拨/盘点调整）    | 生产 BOM 展开、MES 工单明细  |
| 安全库存、预警、周转与 ABC 分析      | 销售订单履约（属销售本体）   |
| 与生产 KPI 的**弱关联**（可选指标）  | 深度 MRP/APS 排程            |

> **与同空间其他本体的分工**：`onto_本体项目01` 的销售本体明确**不包含**库存明细；本方案专注库存域，通过共享 `product_id` / `sku` 与销售、生产域**逻辑对齐**，本体链接**不强制**跨项目建链。

### 2.3 核心业务问题

| 问题                 | 说明                                   |
| -------------------- | -------------------------------------- |
| 现在有多少           | 各仓、各 SKU 现存量与可用量总览        |
| 哪些要补货/已缺货    | 低于安全库存、零库存、负可用预警         |
| 库存怎么变的         | 按流水类型、时间追溯入出库与调拨         |
| 哪些占资金/周转慢    | 库龄、周转天数、呆滞 SKU 识别            |
| 仓库结构是否合理     | 各仓库存占比、品类分布、超储 SKU         |
| 和生产是否匹配       | 可选：对照 `pcc_kpi_daily` 产量与出库节奏 |

### 2.4 分析场景（用户故事）

| 场景 ID | 角色     | 场景描述                                           |
| ------- | -------- | -------------------------------------------------- |
| INV-01  | 仓储主管 | 打开总览：总 SKU 数、总库存量、预警 SKU 数、各仓占比 |
| INV-02  | 计划员   | 按品类查看低于安全库存的 SKU 清单并导出补货建议    |
| INV-03  | 财务分析 | 按仓库查看库存金额结构（需 `unit_cost`）           |
| INV-04  | 运营     | 近 30 天周转最慢 Top N SKU                         |
| INV-05  | 审计     | 某 SKU 在某仓的出入库流水明细                      |

## 三、物理层设计

### 3.1 设计原则

1. **事件/明细优先**：`inventory_transaction`（流水）+ `inventory_balance_snapshot`（快照）为分析主数据；宽表仅作可选派生。
2. **主数据复用**：`product_master` 可与销售本体共用表结构；若销售表未建，本方案自带最小 SKU 维表字段。
3. **命名稳定**：表名面向存储，对象类型 `code` 面向业务（见第五节），二者刻意区分。

### 3.2 表清单

| 表名                         | 类型   | 说明                         |
| ---------------------------- | ------ | ---------------------------- |
| `product_master`             | 维度表 | SKU/物料主数据（可共享）     |
| `warehouse_dimension`        | 维度表 | 仓库主数据                   |
| `inventory_balance_snapshot` | 事实表 | 库存余额快照（日/时点）      |
| `inventory_transaction`      | 事实表 | 出入库/调拨/盘点流水         |

> 字段可与侧栏 **数据资源 → 数据空间0519** 实际建表对齐；下表为推荐初始结构。

### 3.3 表结构

#### 3.3.1 维度表：product_master（物料/SKU，可共享）

**用途**：库存分析所需的 SKU 身份与品类；与销售本体同表时可共用 `product_id`。

| 字段名              | 类型     | 说明                     | 主键/索引 |
| ------------------- | -------- | ------------------------ | --------- |
| product_id          | String   | 产品/SKU ID              | PK        |
| product_code        | String   | 编码                     |           |
| product_name        | String   | 名称                     |           |
| product_category    | String   | 大类（原料/半成品/成品） |           |
| unit                | String   | 计量单位                 |           |
| unit_cost           | Float64  | 标准成本单价（可选）     |           |
| safety_stock_global | Float64  | 全局默认安全库存（可选） |           |
| status              | String   | 状态（在用/停用）        |           |
| updated_at          | DateTime | 更新时间                 |           |

**排序键**：`ORDER BY (product_id)`

#### 3.3.2 维度表：warehouse_dimension（仓库）

| 字段名         | 类型   | 说明                           |
| -------------- | ------ | ------------------------------ |
| warehouse_id   | String | 仓库 ID                        |
| warehouse_code | String | 仓库编码                       |
| warehouse_name | String | 仓库名称                       |
| warehouse_type | String | 类型（原料仓/成品仓/虚拟仓等） |
| region         | String | 所属区域/工厂                  |
| status         | String | 启用/停用                      |

**排序键**：`ORDER BY (warehouse_id)`

#### 3.3.3 事实表：inventory_balance_snapshot（库存快照）

**用途**：按「日期 × 仓库 × SKU」记录期末/时点库存，支撑现存量分析与预警。

| 字段名           | 类型    | 说明                         | 主键/索引 |
| ---------------- | ------- | ---------------------------- | --------- |
| snapshot_id      | String  | 快照行 ID                    | PK        |
| snapshot_date    | Date    | 快照日期                     | 分区/排序 |
| product_id       | String  | SKU ID                       |           |
| warehouse_id     | String  | 仓库 ID                      |           |
| on_hand_qty      | Float64 | 现存量                       |           |
| available_qty    | Float64 | 可用量（扣除预留）           |           |
| in_transit_qty   | Float64 | 在途量                       |           |
| reserved_qty     | Float64 | 预留量                       |           |
| safety_stock     | Float64 | 该仓该 SKU 安全库存          |           |
| unit_cost        | Float64 | 成本单价（冗余，便于金额）   |           |
| inventory_amount | Float64 | 库存金额（可选，或派生计算） |           |

**排序键**：`ORDER BY (snapshot_date, warehouse_id, product_id)`  
**分区建议**：`PARTITION BY toYYYYMM(snapshot_date)`

**业务规则**：

- `available_qty` ≤ `on_hand_qty`（允许系统暂态误差，ETL 告警）
- `inventory_amount` ≈ `on_hand_qty * unit_cost`（有成本时）
- 预警：`available_qty < safety_stock` 或 `on_hand_qty <= 0`

#### 3.3.4 事实表：inventory_transaction（库存流水）

**用途**：记录每一次库存变动，支撑追溯与周转分析。

| 字段名            | 类型     | 说明                                      | 主键/索引 |
| ----------------- | -------- | ----------------------------------------- | --------- |
| transaction_id    | String   | 流水 ID                                   | PK        |
| transaction_time  | DateTime | 业务时间                                  | 排序      |
| transaction_date  | Date     | 业务日期（冗余）                          | 分区      |
| product_id        | String   | SKU ID                                    |           |
| warehouse_id      | String   | 仓库 ID                                   |           |
| to_warehouse_id   | String   | 调入仓（调拨时）                          |           |
| transaction_type  | String   | 类型：IN/OUT/TRANSFER/ADJUST/COUNT        |           |
| source_doc_type   | String   | 来源单据类型（采购入库/销售出库/生产领料等） |        |
| source_doc_id     | String   | 来源单号                                  |           |
| quantity          | Float64  | 变动数量（入为正、出为负，或分正负列）    |           |
| unit_cost         | Float64  | 单价                                      |           |
| amount            | Float64  | 金额                                      |           |
| batch_no          | String   | 批次号（可选）                            |           |
| operator          | String   | 操作人（可选）                            |           |

**排序键**：`ORDER BY (transaction_date, warehouse_id, product_id, transaction_id)`  
**引擎**：`MergeTree()`

**业务规则**：

- `transaction_type = 'OUT'` 时 `quantity` 为负，或统一用 `direction` 字段（规划实施时二选一并文档化）
- 调拨：同一 `source_doc_id` 下 OUT + IN 成对出现

### 3.4 与现有表关系

| 现有表          | 关系说明                                                                 |
| --------------- | ------------------------------------------------------------------------ |
| `pcc_kpi_daily` | 生产 KPI（产量、合格率等）；**不做强制外键链接**；函数层可选只读对照产量 |
| 销售本体表      | 若已建 `product_master`、`sales_order_fact`，SKU 以 `product_id` 对齐；销售出库流水可写入 `inventory_transaction`（`source_doc_type=销售出库`） |

## 四、Cube 层设计

### 4.1 InventoryCube（库存分析主 Cube）

**目的**：全局库存快照分析、预警汇总。

**数据源**：`inventory_balance_snapshot`（取最新 `snapshot_date` 或参数化日期）

**维度**：

| 维度名           | 列名             | 类型   | 说明     |
| ---------------- | ---------------- | ------ | -------- |
| snapshot_date    | snapshot_date    | date   | 快照日期 |
| product_id       | product_id       | string | SKU ID   |
| warehouse_id     | warehouse_id     | string | 仓库 ID  |

**度量**：

| 度量名         | 列名           | 聚合 | 说明         |
| -------------- | -------------- | ---- | ------------ |
| on_hand_qty    | on_hand_qty    | sum  | 现存量       |
| available_qty  | available_qty  | sum  | 可用量       |
| in_transit_qty | in_transit_qty | sum  | 在途量       |
| inventory_amount | inventory_amount | sum | 库存金额   |
| sku_count      | product_id     | uniq | SKU 数       |
| warehouse_count| warehouse_id   | uniq | 涉及仓库数   |

**派生度量**：

| 度量名              | 表达式 / 逻辑                                      | 说明           |
| ------------------- | -------------------------------------------------- | -------------- |
| stock_shortage_flag | available_qty < safety_stock                       | 缺货标记（0/1）|
| stockout_flag       | on_hand_qty <= 0                                   | 零库存标记     |
| fill_rate           | if(safety_stock>0, available_qty/safety_stock, null) | 安全库存满足率 |

### 4.2 WarehouseInventoryCube（仓库库存 Cube）

**目的**：按仓库聚合库存结构与占比。

**维度**：`warehouse_id`（join `warehouse_dimension` 补充 `warehouse_name`、`warehouse_type`）  
**度量**：`on_hand_qty`, `available_qty`, `inventory_amount`, `sku_count`  
**派生**：各仓 `inventory_amount` 占全空间比例（查询层或函数层计算 `share_pct`）

### 4.3 SkuInventoryCube（SKU 库存 Cube）

**目的**：按 SKU/品类聚合，支撑 Top 呆滞、品类结构。

**维度**：`product_id`, `product_category`（来自 `product_master` join）  
**度量**：`on_hand_qty`, `available_qty`, `inventory_amount`  
**派生**：`days_of_supply`（需结合出库量，见 MovementCube）

### 4.4 InventoryMovementCube（库存流水 Cube）

**目的**：出入库汇总、周转分析。

**数据源**：`inventory_transaction`

**维度**：

| 维度名           | 列名             | 类型   | 说明       |
| ---------------- | ---------------- | ------ | ---------- |
| transaction_date | transaction_date | date   | 业务日期   |
| product_id       | product_id       | string | SKU        |
| warehouse_id     | warehouse_id     | string | 仓库       |
| transaction_type | transaction_type | string | 流水类型   |
| source_doc_type  | source_doc_type  | string | 来源类型   |

**度量**：

| 度量名        | 列名     | 聚合 | 说明           |
| ------------- | -------- | ---- | -------------- |
| in_qty        | quantity | sum  | 入库量（过滤 IN）|
| out_qty       | quantity | sum  | 出库量（过滤 OUT，取绝对值）|
| transfer_qty  | quantity | sum  | 调拨量         |
| txn_count     | transaction_id | uniq | 流水笔数   |

**派生度量**（函数层亦可实现）：

| 度量名           | 说明                                   |
| ---------------- | -------------------------------------- |
| turnover_days    | 平均库存 / 日均出库（需快照+出库窗口） |
| outbound_per_day | 窗口内出库量 / 天数                    |

### 4.5 TimeInventoryCube（时间维度 Cube）

**目的**：库存趋势、月末对比。

**维度**：`snapshot_date`, `year_month`, `quarter`  
**度量**：`on_hand_qty`, `inventory_amount`, `sku_count`

## 六、本体函数设计

### 6.1 函数清单

| 函数 ID                              | 功能             | 主要参数                                      |
| ------------------------------------ | ---------------- | --------------------------------------------- |
| inventory.fn.get_summary             | 库存总览         | snapshot_date（可选，默认最新）               |
| inventory.fn.low_stock_alerts        | 低库存/缺货清单  | warehouse_id, category, limit, snapshot_date  |
| inventory.fn.warehouse_breakdown     | 分仓库存结构     | snapshot_date, metric（qty/amount）           |
| inventory.fn.movement_trace          | SKU 流水追溯     | product_id, warehouse_id, start_date, end_date |
| inventory.fn.turnover_rank           | 周转排行         | window_days, limit, direction（slow/fast）    |
| inventory.fn.abc_classification      | ABC 分类         | snapshot_date, metric（amount/qty）           |
| inventory.fn.compare_production_kpi  | 产量与出库对照   | start_date, end_date（只读 pcc_kpi_daily）    |

### 6.2 函数详细设计（节选）

#### 6.2.1 inventory.fn.get_summary（库存总览）

**返回值**：

| 字段名            | 类型    | 说明                 |
| ----------------- | ------- | -------------------- |
| total_on_hand_qty | Float64 | 总现存量             |
| total_available   | Float64 | 总可用量             |
| total_amount      | Float64 | 总库存金额           |
| sku_count         | Int     | 有库存 SKU 数        |
| alert_sku_count   | Int     | 低于安全库存 SKU 数  |
| stockout_count    | Int     | 零库存 SKU 数        |
| snapshot_date     | Date    | 数据快照日期         |

#### 6.2.2 inventory.fn.low_stock_alerts（低库存预警）

**参数**：`warehouse_id`（可选）、`category`（可选）、`limit`（默认 50）、`snapshot_date`

**返回值**：`rank`, `product_id`, `product_name`, `warehouse_id`, `warehouse_name`, `available_qty`, `safety_stock`, `gap_qty`, `severity`

**severity 规则（建议）**：

- `critical`：`available_qty <= 0`
- `warning`：`0 < available_qty < safety_stock`

#### 6.2.3 inventory.fn.turnover_rank（周转排行）

**逻辑**：窗口内日均出库 = `sum(out_qty) / window_days`；平均库存取窗口期快照均值；`turnover_days = avg_inventory / daily_out`（出库为 0 时标记为呆滞）

**返回值**：`rank`, `product_id`, `product_name`, `turnover_days`, `avg_inventory`, `daily_outbound`, `tag`（fast/normal/slow/dead）

## 九、风险与验收

### 9.1 风险识别

| 风险等级 | 风险描述                         | 缓解措施                                   |
| -------- | -------------------------------- | ------------------------------------------ |
| 中       | 与销售本体 `product_master` 分叉 | 约定单一主数据维护方；ID 映射表              |
| 中       | 快照与流水口径不一致导致周转失真 | 统一日切规则；函数层注明统计窗口             |
| 中       | 安全库存维护不全导致误报         | 区分 global / 仓级 safety_stock；规则过滤停用 SKU |
| 低       | 与 `pcc_kpi_daily` 语义混用      | 独立函数、不做本体链接；文档标明可选对照     |
| 低       | 成本缺失导致金额分析不可用       | 金额类指标降级为仅数量分析                   |

### 9.2 验收标准

| 验收项   | 验收标准                                           |
| -------- | -------------------------------------------------- |
| 物理表   | 4 张表可查询，演示数据可关联                       |

| 对象类型 | 6 种对象类型与属性映射正确                         |

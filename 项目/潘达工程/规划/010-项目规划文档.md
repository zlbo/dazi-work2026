# 潘达建工商务成本智能决策体系 - 项目规划文档

## 一、项目背景

### 1.1 业务背景
潘达建工集团需要建立一套**本体驱动的智能决策体系**，用于商务成本管理分析。该体系将：

- 替代传统Excel台账，提供Web版报表展示
- 支持AI助手问答，实现指标查询、规律探寻、根因分析
- 提供可视化图表看板和驾驶舱
- 生成分析报告和决策建议

### 1.2 现有资源分析

当前可用资源包括：

#### （1）Excel报表模板
`潘达建工（房建）商务成本报表模板.xlsx`包含以下核心报表：
| 报表名称 | 表编号 | 内容概述 |
|---------|-------|---------|
| 项目月度资金预算批复单 | 表1 | 资金情况、利润状况、指标分析（7项核心指标） |
| 项目月度商务管理分析表 | 表2 | 项目评价、工程概况、风险清单、主控费用 |
| 项目月度商务管理汇总表 | 表3 | 业主产值、收款、成本等核心数据汇总 |
| 业主产值详表 | 表3.1 | 产值明细（已确认/待确认） |
| 业主待确产值详表 | 表3.1.1 | 待确认产值明细 |
| 月度成本、资金详表 | 表3.2 | 成本支出明细 |
| 已发生待确成本计算详表 | 表3.2.1 | 待确认成本明细 |

#### （2）数据库表（建工成本01数据源）
| 表名 | 业务含义 | 关键字段 |
|-----|---------|---------|
| `tb_cost_report_cell` | 报表单元格数据 | row_id, row_num, column_num, value, formula |
| `tb_cost_report_row` | 报表行结构 | report_id, sheet_id, level, parent_id |
| `tb_cost_report_view_model_definition_item` | 视图模型定义 | title, code, binding_type, meta |
| `tb_project_cost_pay_detail` | 成本付款明细 | project_id, cost_code, cost_actual_confirmed_acc |
| `tb_project_cost_unconfirmed_analyse` | 待确认成本分析 | contract_code, estimate_amount, reason |
| `tb_project_income_outcome_summary` | 收支汇总 | code, name, project_amount, company_amount |
| `tb_project_indicator` | 项目指标 | code, title, value, remark(表关联) |
| `tb_project_major_indicator_analyse` | 核心指标分析 | code, name, rule, value, warning |
| `tb_project_major_cost` | 主控费用 | code, name, target_cost_amount, incurred_cost_amount |
| `tb_project_risk_list_analyse` | 风险清单分析 | risk_type, code, name, value, warning |

#### （3）资源关联发现
- 数据库字段的`remark`字段包含表编号引用（如"表3(6.4)"、"表3.1(1.8)"），直接映射到Excel报表
- `project_id` + `year` + `month` 是核心关联键
- Excel公式中的跨Sheet引用与数据库中的视图模型定义存在逻辑对应

## 二、最终目标

### 2.1 总体架构目标
构建**本体驱动的商务成本智能决策体系**，包含以下层次：

```
┌─────────────────────────────────────────────────────────────┐
│                    应用层（面向用户）                         │
│  AI助手问答 │ Web报表 │ 可视化看板 │ 分析报告 │ 决策建议       │
├─────────────────────────────────────────────────────────────┤
│                    服务层（业务逻辑）                         │
│  指标计算引擎 │ 数据分析引擎 │ 根因分析引擎 │ 报表生成引擎     │
├─────────────────────────────────────────────────────────────┤
│                    数据层（本体与数据）                       │
│  本体模型 │ 本体网络 │ 指标库 │ 事实库 │ 规则库              │
├─────────────────────────────────────────────────────────────┤
│                    数据源层                                  │
│  建工成本01 │ 其他业务系统 │ 外部数据                         │
└─────────────────────────────────────────────────────────────┘
```

### 2.2 核心能力目标

| 能力域 | 具体能力 | 描述 |
|-------|---------|------|
| **指标查询** | AI自然语言问答 | 支持"本月利润率是多少？"等自然语言查询 |
| **规律探寻** | 趋势分析 | 自动发现成本变动趋势、异常波动 |
| **根因分析** | 因果推断 | 定位成本超支、利润率下降的根本原因 |
| **报表重现** | Web报表 | 在线展示Excel中的所有报表内容 |
| **可视化** | 驾驶舱看板 | 多维度图表展示关键指标 |
| **决策支持** | 智能建议 | 基于分析结果提供优化建议 |

## 三、阶段目标规划

### 阶段一：数据与报表映射分析（当前阶段）
**周期**：2-3周  
**目标**：完成数据库表与Excel报表的深度映射分析

**核心任务**：
1. 建立Excel报表与数据库表的完整映射矩阵
2. 解析报表间的公式依赖关系
3. 提取核心指标的业务计算公式
4. 输出《数据血缘分析报告》

**交付物**：
- 报表-数据表映射矩阵
- 指标计算逻辑文档
- 数据血缘图

### 阶段二：本体模型构建
**周期**：3-4周  
**目标**：构建商务成本领域本体模型

**核心任务**：
1. 定义本体概念（项目、成本、收入、指标等）
2. 定义概念间关系（包含、属于、计算等）
3. 构建指标计算规则
4. 建立本体网络

**交付物**：
- 本体模型文档（OWL/RDF格式）
- 本体关系图谱
- 指标规则库

### 阶段三：数据对接与ETL开发
**周期**：3-4周  
**目标**：实现数据库到本体的映射与同步

**核心任务**：
1. 开发ETL流程
2. 实现数据清洗与转换
3. 建立事实库填充机制
4. 开发增量同步机制

**交付物**：
- ETL流程设计文档
- ETL代码实现
- 数据质量报告

### 阶段四：智能问答与分析引擎开发
**周期**：4-6周  
**目标**：实现AI助手问答和分析能力

**核心任务**：
1. 开发自然语言理解模块
2. 实现指标查询引擎
3. 开发根因分析引擎
4. 集成大语言模型

**交付物**：
- 问答API接口
- 分析引擎核心代码
- 测试用例与结果

### 阶段五：可视化报表与看板开发
**周期**：3-4周  
**目标**：实现Web报表和可视化看板

**核心任务**：
1. 重现Excel报表的Web版本
2. 开发驾驶舱仪表盘
3. 实现交互式图表
4. 开发报告生成模块

**交付物**：
- Web报表页面
- 可视化看板组件
- 报告模板

## 四、当前阶段计划（阶段一）

### 4.1 阶段目标
完成数据库表与Excel报表的深度映射分析，明确数据流向和计算逻辑。

### 4.2 详细任务清单

| 任务编号 | 任务名称 | 任务描述 | 预计耗时 | 依赖 |
|---------|---------|---------|---------|-----|
| 1.1 | 报表结构分析 | 详细分析Excel各Sheet的结构、单元格含义、公式逻辑 | 2天 | 原始Excel模板 |
| 1.2 | 数据库表深度分析 | 分析每个数据库表的业务含义、字段语义、数据样例 | 2天 | 数据表文档 |
| 1.3 | 映射关系建立 | 建立Excel单元格与数据库字段的映射关系 | 3天 | 1.1, 1.2 |
| 1.4 | 指标计算逻辑提取 | 解析Excel公式，提取核心指标的计算规则 | 3天 | 1.1 |
| 1.5 | 数据血缘分析 | 追踪指标从原始数据到报表展示的完整流向 | 2天 | 1.3, 1.4 |

### 4.3 关键产出物

| 产出物名称 | 格式 | 内容概述 |
|-----------|------|---------|
| 报表-数据表映射矩阵 | Excel/MD | 列：Excel单元格，行：数据库字段，标注映射关系 |
| 指标计算逻辑文档 | MD | 核心指标的计算公式、依赖关系、业务含义 |
| 数据血缘图 | SVG/PNG | 从数据源到报表的完整数据流可视化 |
| 数据血缘分析报告 | MD/PDF | 综合分析报告，含问题清单和建议 |

### 4.4 成功标准

1. 完成所有核心报表（表1-表3.2.1）的字段映射，映射覆盖率≥90%
2. 提取至少50个核心指标的计算逻辑
3. 绘制完整的数据血缘图
4. 输出可用于指导后续本体构建的分析报告

## 五、当前资源与约束

### 5.1 可用资源
- **Excel模板**：潘达建工商务成本报表模板（含公式）
- **数据库文档**：9张核心数据表的结构和样例数据
- **技术团队**：前端、后端、AI工程师

### 5.2 约束条件
- 当前无实时数据库连接，仅能基于文档分析
- Excel模板无真实数据，需依赖数据库样例推断
- 部分字段含义需要业务专家确认

### 5.3 风险与应对

| 风险 | 影响 | 应对措施 |
|-----|------|---------|
| 字段含义不明确 | 映射分析不准确 | 列出疑问清单，与业务专家确认 |
| 公式逻辑复杂 | 计算规则提取困难 | 重点分析核心指标，逐步扩展 |
| 数据缺失 | 验证困难 | 基于现有样例数据进行推断 |

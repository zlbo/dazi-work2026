## 二、报表整体结构

表3.1采用**横向分组**的结构设计，主要分为六大区域：

```
┌───────────────────────────────────────────────────────────────────────────┐
│ 表头:项目名称、报告期间、单位                                            │
├───────────────────────────────────────────────────────────────────────────┤
│ A列:序号                                                                 │
│ B列:日期                                                                 │
│ C列:本期产值核定期间                                                     │
│ D-F列:开工至上年度产值(已确认、待确认、合计)                             │
│ G-I列:本年度累计实际完成产值(已确认、待确认、合计)                        │
│ J-L列:其中:本月实际完成产值(已确认、待确认、合计)                         │
│ M-O列:自开工累计实际完成产值(已确认、待确认、合计)                        │
│ P-Q列:二、应收款(合约)                                                  │
│ R-T列:资金收款计划(业主收款、往来收入、小计)                             │
│ U列:备注                                                                 │
│ ...后续列(可能包含更多明细)...                                           │
└───────────────────────────────────────────────────────────────────────────┘
```

## 七、业务逻辑分析

### 7.1 产值确认流程

```
1. 录入本月产值(J6、K6)
    ↓
2. 累计到自开工累计(M6、N6)
    ↓
3. 计算本年度累计(G6、H6)
    ↓
4. 计算各项合计(F6、I6、L6、O6)
    ↓
5. 计算应收款(P6)
    ↓
6. 记录实际收款(R6、S6)
```

### 7.2 应收款计算逻辑

应收款计算需要考虑多个因素：
1. **安全文明施工费预付款** - 按合同约定比例
2. **业主审核产值** - 按合同约定付款比例
3. **预付款扣回** - 按合同约定扣回
4. **其他扣款** - 质保金、罚款等

示例计算式：
```
应收款 = 50[安全文明施工费预付款] 
       + 100[业主审核产值] * 70%[合同约定付款比例] 
       - 20[预、其他扣款]
```

### 7.3 时间段关系

```
自开工累计产值 = 开工至上年度产值 + 本年度累计产值
本年度累计产值 = 自开工累计产值 - 开工至上年度产值
本月产值 ⊂ 本年度累计产值
```

## 九、报表特点总结

### 9.1 优势特点

1. **时间维度清晰** - 上年度、本年度、本月、自开工累计四个时间维度
2. **确认状态明确** - 已确认、待确认、合计三种状态
3. **计算逻辑严谨** - 时间段关系、合计关系自动计算
4. **应收款详细** - 提供详细计算式，便于审核

### 9.2 注意事项

1. **表格过宽** - 260+列，操作不便
2. **手动录入多** - 大部分数据需要手动填写
3. **附件要求高** - 需要提供产值单扫描件等附件
4. **依赖子表** - 待确认金额依赖表3.1.1

### 9.3 维护建议

1. **数据校验** - 增加时间段逻辑校验
2. **附件管理** - 建立附件上传和管理机制
3. **计算辅助** - 提供应收款计算辅助工具
4. **历史对比** - 建立历史产值对比功能

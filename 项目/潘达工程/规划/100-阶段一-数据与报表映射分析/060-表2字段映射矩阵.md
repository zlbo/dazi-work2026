# 060-表2字段映射矩阵

## 一、映射关系概述

表2的数据主要来源于:
1. **表3-商务管理汇总表**(78%的引用)
2. **表3.2-完全成本付款明细表**(4%的引用)
3. **本表计算**(18%的引用)

由于表3的数据最终来源于数据库表,因此表2与数据库的映射需要通过表3作为中间层。

## 三、数据库表详细说明

### 3.1 tb_project_indicator(项目指标表)

**用途**: 存储项目各类指标数据

**关键字段**:
- project_id: 项目ID
- year: 年份
- month: 月份
- code: 指标编码(用于区分不同指标)
- title: 指标标题
- value: 指标值
- value_type: 值类型(1=文本,2=数值,3=日期)

**表2相关code**:
- project_name: 项目名称
- owner_name: 建设单位名称
- project_area: 工程规模(建筑面积)
- project_contract_amount_exclude: 合同价款(不含暂列金)
- management_fee_rate: 管理费率
- management_fee_paid_acc: 已缴管理费累计
- output_actual_acc: 开累产值(已确+待确)
- output_actual_cyear_acc: 本年度产值累计
- output_actual_confirmed_acc: 已确权产值累计
- full_cost_confirmed_acc: 已确权成本累计
- full_cost_unconfirmed_acc: 待确权成本累计
- profit_acc: 利润额(已确+待确)
- profit_rate_acc: 利润率(已确+待确)
- profit_confirmed_acc: 利润额(已确)
- profit_rate_confirmed: 利润率(已确)

## 七、特殊情况说明

### 7.1 空值处理

以下情况可能出现空值:
- D27、D28: 当已确权产值为0时,利润率计算会出现除零错误
- D22、D24: 当预估结算金额或本年度目标产值为0时,完成率计算会出现除零错误
- 所有单方造价: 当建筑面积为0时,会出现除零错误

**建议处理方式**:
- 在Excel公式中增加IF判断,避免除零错误
- 示例: `=IF(J10=0,0,D13/J10)`

### 7.2 数据更新时机

4. 建立数据审核机制,确保数据质量

## 八、总结

### 8.1 映射特点

1. **间接映射为主**: 表2数据主要通过表3间接获取,减少直接数据库访问
2. **计算字段较多**: 24%的字段需要计算,体现报表的分析功能
3. **手动填写必要**: 工程概况、风险分析等需要人工判断和填写
4. **多表关联**: 涉及3个主要数据库表,数据来源多样化

### 8.2 应用建议

1. **优先使用公式**: 能用公式计算的尽量不用手动填写
2. **建立数据字典**: 明确各code的含义和取值范围
3. **加强数据验证**: 在数据录入时进行完整性校验
4. **定期数据审核**: 确保数据质量和一致性

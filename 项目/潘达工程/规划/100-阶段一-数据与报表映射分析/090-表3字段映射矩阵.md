# 090-表3字段映射矩阵

## 一、映射关系概述

表3是整个报表体系的**数据核心枢纽**,采用"一行多列"的独特设计。数据流向为:
- **数据来源**: 表3.1、表3.2、项目收支汇总表
- **数据去向**: 表1、表2引用
- **数据库映射**: 通过多个专门表存储数据

## 二、核心数据库表说明

### 2.1 数据存储架构

| 数据库表 | 作用 | 相关表3区域 |
|---------|------|-----------|
| tb_project_indicator | 项目指标存储 | 基本信息、利润指标、核心指标 |
| tb_project_income_outcome_summary | 收支汇总存储 | 往来资金区域 |
| tb_project_major_indicator_analyse | 主要指标分析 | 7项核心指标 |
| tb_project_cost_pay_detail | 成本付款明细 | 完全成本区域 |
| tb_project_risk_list_analyse | 风险清单 | 风险相关 |
| tb_project_major_cost | 主要成本 | 主控费用 |
| tb_cost_report_row | 报表行定义 | 通用报表结构 |
| tb_cost_report_cell | 报表单元格 | 通用报表结构 |
| tb_cost_report_view_model_definition_item | 视图模型定义 | 字段绑定规则 |

### 2.2 tb_cost_report_view_model_definition_item详解

这个表是字段绑定的**关键配置表**,定义了每个字段如何绑定:

| 字段 | 说明 | 示例 |
|-----|------|-----|
| code | 字段编码 | project_name |
| title | 字段标题 | 项目名称 |
| binding_type | 绑定类型 | 1=数据绑定,2=可编辑 |
| meta | 元数据 | {"dataType":"number"} |
| editable | 是否可编辑 | 1=可编辑,2=只读 |
| role_codes | 可操作角色 | gscw,fgscw,gssw |
| remark | 备注 | 表3(位置说明) |

## 五、映射类型详细说明

### 5.1 直接映射

Excel单元格直接对应数据库字段,通过code关联:

```
Excel:表3!B7
  ↓
code: 'project_name'
  ↓
Database: tb_project_indicator where code='project_name'
  ↓
提取: value字段
```

### 5.2 间接映射(通过子表)

Excel从子表引用,子表再映射到数据库:

```
Excel:表3!V7
  ↓
引用:表3.1!D6
  ↓
子表内部映射到数据库
```

### 5.3 计算映射

通过公式计算得出:

```
X7 = V7 + W7
AG7 = AE7 + AF7
```

### 5.4 LOOKUP映射

通过查找项目收支汇总表:

```
F7 = LOOKUP(1,0/(项目收支汇总表!A5:A306="5.1"),项目收支汇总表!C5:C306)
```

## 六、数据存储设计分析

### 6.1 存储策略

表3采用**混合存储策略**:
1. **直接字段** - 用tb_project_indicator按code存储
2. **计算指标** - 用tb_project_major_indicator_analyse按编号存储
3. **视图定义** - 用tb_cost_report_view_model_definition_item定义绑定规则

### 6.2 扩展性设计

通过tb_cost_report_view_model_definition_item的设计,可以:
- 动态添加新字段
- 控制编辑权限
- 定义数据类型
- 绑定到报表位置

## 七、验证规则与检查清单

### 7.1 数据完整性检查

| 检查项 | SQL示例 | 预期结果 |
|------|---------|---------|
| 基本信息完整性 | SELECT * FROM tb_project_indicator WHERE code='project_name' AND project_id=? | 有记录 |
| 产值指标完整性 | SELECT * FROM tb_project_indicator WHERE code LIKE 'output%' | 多条记录 |
| 成本指标完整性 | SELECT * FROM tb_project_indicator WHERE code LIKE 'cost%' | 多条记录 |
| 7项指标完整性 | SELECT * FROM tb_project_major_indicator_analyse WHERE code BETWEEN '10' AND '70' | 7条记录 |

### 7.2 数据一致性检查

| 检查项 | 验证逻辑 |
|------|---------|
| 合计正确性 | 合计 = 已确认 + 待确认 |
| 累计合理性 | 累计 ≥ 上年度 + 本年度 |
| 利润率计算 | 利润率 = 利润 / 产值(处理除零) |

## 八、实施建议

### 8.1 数据库同步流程

```
1. 表3.1、表3.2数据录入
   ↓
2. 表3公式自动计算
   ↓
3. 提取表3关键数据
   ↓
4. 写入相关数据库表(tb_project_indicator等)
   ↓
5. 表1、表2自动更新显示
```

### 8.2 字段映射配置

建议使用tb_cost_report_view_model_definition_item表进行配置:
- code: 唯一编码
- title: 显示标题
- binding_type: 绑定类型
- meta: JSON格式元数据
- editable: 编辑权限
- role_codes: 角色权限
- remark: 位置备注(表3(位置))
